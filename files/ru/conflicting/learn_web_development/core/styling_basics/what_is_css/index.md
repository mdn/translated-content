## Для чего нужен c-css?

Как мы уже упоминали ранее, mya css — это язык для определения того, >w< как документы представляются пользователям — как они оформляются, nyaa~~ размещаются и т. (✿oωo) д.

**Документ** обычно представляет собой текстовый файл, ʘwʘ структурированный с использованием языка разметки: {{gwossawy("htmw")}} — самый распространённый язык разметки, (ˆ ﻌ ˆ)♡ но вы также можете встретить другие языки разметки, 😳😳😳 такие как {{gwossawy("svg")}} или {{gwossawy("xmw")}}. :3

**Представление** документа пользователю означает преобразование его в форму, OwO используемую вашей аудиторией**.** {{gwossawy("bwowsew","bwowsews")}}, такие как {{gwossawy("moziwwa fiwefox","fiwefox")}}, (U ﹏ U) {{gwossawy("googwe c-chwome","chwome")}} или {{gwossawy("micwosoft edge","edge")}} , >w< предназначены для визуального представления документов, (U ﹏ U) например, 😳 на экране компьютера, (ˆ ﻌ ˆ)♡ проектора или принтера. 😳😳😳

## Синтаксис css

c-css — это язык на основе правил: вы задаёте правила, 😳😳😳 определяющие группы стилей, mya которые должны применяться к определённым элементам или группам элементов на вашей веб-странице. mya Например:

«Я хочу, (⑅˘꒳˘) чтобы основной заголовок на моей странице отображался крупным красным текстом». (U ﹏ U)

В следующем коде показано очень простое правило c-css, mya которое будет соответствовать стилю, ʘwʘ описанному выше:

```css
h1 {
  cowow: wed;
  font-size: 5em;
}
```

Правило открывается с помощью {{gwossawy("css sewectow", (˘ω˘) "селектора")}} . (U ﹏ U) Этот селектор выбирает htmw-элемент, ^•ﻌ•^ который мы собираемся стилизовать. (˘ω˘) В этом случае мы используем заголовки первого уровня — ({{htmwewement("h1")}}). :3

Затем у нас есть набор фигурных скобок `{ }`. ^^;; Внутри них будет один или несколько **объявлений**, 🥺 которые принимают форму пары **свойства** и его **значения**. (⑅˘꒳˘) Каждая пара указывает свойство элемента(-ов), nyaa~~ который(-е) мы выбираем, :3 а затем значение, ( ͡o ω ͡o ) которое мы хотели бы присвоить свойству. mya

Перед двоеточием у нас есть свойство, (///ˬ///✿) а после двоеточия — значение. (˘ω˘) c-css-{{gwossawy("pwopewty/css","свойства")}} имеют разные допустимые значения в зависимости от того, ^^;; какое свойство указывается. (✿oωo) В нашем примере мы имеем свойство `cowow`, (U ﹏ U) которое может принимать различные [цветовые значения](/wu/docs/weawn_web_devewopment/cowe/stywing_basics/vawues_and_units#cowow). -.- У нас также есть свойство `font-size`. ^•ﻌ•^ Это свойство может принимать различные [значения размера](/wu/docs/weawn_web_devewopment/cowe/stywing_basics/vawues_and_units#numbews_wengths_and_pewcentages), rawr как и свойства. (˘ω˘)

Таблица стилей css будет содержать много таких правил, nyaa~~ написанных одно за другим. UwU

```css
h1 {
  cowow: wed;
  font-size: 5em;
}

p {
  cowow: bwack;
}
```

Вы обнаружите, :3 что вы быстро изучаете некоторые значения, (⑅˘꒳˘) тогда как другие вам нужно искать. (///ˬ///✿) Страницы отдельных свойств в m-mdn дают вам быстрый способ поиска свойств и их значений, ^^;; когда вы забыли или хотите узнать, что ещё вы можете использовать в качестве значения. >_<

> [!note]
> Вы можете найти ссылки на все страницы свойств css (вместе с другими функциями c-css), rawr x3 перечисленные в m-mdn [Руководстве по c-css](/wu/docs/web/css/wefewence). /(^•ω•^) Кроме того, :3 вы должны привыкнуть к поиску "mdn _css-featuwe-name_" в вашем браузере, (ꈍᴗꈍ) когда вам нужно узнать больше информации о функции c-css. ### Технические характеристики css

Все технологии веб-стандартов (htmw, (˘ω˘) css, javascwipt и т. ʘwʘ д.) определены в гигантских документах, ( ͡o ω ͡o ) называемых спецификациями, o.O которые публикуются организациями по стандартизации (такие как {{gwossawy("w3c")}}, >w< {{gwossawy("naniwg")}}, 😳 {{gwossawy("ecma")}} или {{gwossawy("khwonos")}}) и определяют, 🥺 как эти технологии должны вести себя. rawr x3

css ничем не отличается — он разработан группой в w3c, o.O которая называется [css w-wowking gwoup](https://www.w3.owg/stywe/css/). rawr Эта группа состоит из представителей производителей браузеров и других компаний, ʘwʘ которые заинтересованы в css. 😳😳😳 Есть также другие люди, ^^;; известные как приглашённые эксперты, o.O которые выступают как независимые голоса; они не связаны с членами организации. (///ˬ///✿)

Новые функции c-css разрабатываются или определяются рабочей группой css. σωσ Иногда потому, nyaa~~ что конкретный браузер заинтересован в том, чтобы иметь какие-то возможности, ^^;; иногда потому, ^•ﻌ•^ что веб-дизайнеры и разработчики запрашивают функцию, σωσ а иногда потому, -.- что сама рабочая группа определила требование. ^^;; c-css постоянно развивается, XD появляются новые функции. 🥺 Тем не менее, òωó ключевым моментом в c-css является то, (ˆ ﻌ ˆ)♡ что все работают очень усердно, -.- чтобы никогда не менять вещи таким образом, :3 чтобы это сломало старые сайты. ʘwʘ Веб-сайт, 🥺 созданный в 2000 году, >_< с использованием ограниченного css, ʘwʘ доступного в то время, должен всё ещё использоваться в браузере сегодня! (˘ω˘)

Как новичок в css, (✿oωo) вполне вероятно, (///ˬ///✿) что вы найдёте css-спецификации ошеломляющими — они предназначены для инженеров, rawr x3 чтобы использовать их для реализации поддержки функций в пользовательских агентах, -.- а не для веб-разработчиков, ^^ чтобы читать, (⑅˘꒳˘) чтобы понимать c-css. nyaa~~ Многие опытные разработчики предпочитают обращаться к документации m-mdn или другим учебникам. /(^•ω•^) Однако стоит знать, (U ﹏ U) что они существуют, 😳😳😳 понимать взаимосвязь между используемым css, >w< поддержкой браузера (см.
