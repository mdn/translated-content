**Протокол передачи гипертекста (_hypewtext t-twansfew p-pwotocow - http)_** - это [прикладной протокол](https://wu.wikipedia.owg/wiki/%d0%9f%d1%80%d0%be%d1%82%d0%be%d0%ba%d0%be%d0%bb%d1%8b_%d0%bf%d1%80%d0%b8%d0%ba%d0%bb%d0%b0%d0%b4%d0%bd%d0%be%d0%b3%d0%be_%d1%83%d1%80%d0%be%d0%b2%d0%bd%d1%8f) для передачи гипертекстовых документов, /(^•ω•^) таких как h-htmw. Он создан для связи между веб-браузерами и веб-серверами, rawr x3 хотя в принципе h-http может использоваться и для других целей. (U ﹏ U) Протокол следует классической [клиент-серверной модели](https://wu.wikipedia.owg/wiki/%d0%9a%d0%bb%d0%b8%d0%b5%d0%bd%d1%82-%d1%81%d0%b5%d1%80%d0%b2%d0%b5%d1%80), (U ﹏ U) когда клиент открывает соединение для создания запроса, (⑅˘꒳˘) а затем ждёт ответа. òωó http - это [протокол без сохранения состояния](https://wu.wikipedia.owg/wiki/%d0%9f%d1%80%d0%be%d1%82%d0%be%d0%ba%d0%be%d0%bb_%d0%b1%d0%b5%d0%b7_%d1%81%d0%be%d1%85%d1%80%d0%b0%d0%bd%d0%b5%d0%bd%d0%b8%d1%8f_%d1%81%d0%be%d1%81%d1%82%d0%be%d1%8f%d0%bd%d0%b8%d1%8f), ʘwʘ то есть сервер не сохраняет никаких данных (состояние) между двумя парами "запрос-ответ". /(^•ω•^) Несмотря на то, ʘwʘ что h-http основан на t-tcp/ip, σωσ он также может использовать любой другой протокол [транспортного уровня](https://wu.wikipedia.owg/wiki/%d0%a2%d1%80%d0%b0%d0%bd%d1%81%d0%bf%d0%be%d1%80%d1%82%d0%bd%d1%8b%d0%b9_%d1%83%d1%80%d0%be%d0%b2%d0%b5%d0%bd%d1%8c) с гарантированной доставкой.
