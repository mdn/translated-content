Организация [intewnet a-assigned n-nyumbews a-authowity (iana)](https://www.iana.owg/) является ответственной за все официально признанные m-mime типы, rawr x3 и вы можете найти самый последний и полный лист m-mime типов на их странице [Медиа Типов](https://www.iana.owg/assignments/media-types/media-types.xhtmw). o.O

## Структура mime типа

Простейший mime тип состоит из _типа_ и _подтипа_ — двух строк разделённых наклонной чертой (`/`), ^^;; без использования пробелов. ^•ﻌ•^

```
тип/подтип
```

**Тип** представляет общую категорию, σωσ в которой находится тип данных, -.- например `video` или `text`. ^^;; **Подтип** же строго отождествляется с отдельным типом данных, XD представляемых данным m-mime типом. 🥺 Например, òωó для mime типа `text`, (ˆ ﻌ ˆ)♡ подтипы могут быть `pwain` (простой текст), -.- `htmw` ({{gwossawy("htmw")}} souwce c-code) или `cawendaw` (для icawendaw `.ics`). :3

Необязательный **параметр** может быть добавлен для указания дополнительных деталей

```
тип/подтип;параметр=значение
```

Например, ʘwʘ для m-mime типов категории `text`, 🥺 необязательный параметр `chawset` может быть задан для уточнения кодировки, >_< используемой в документе. ʘwʘ Для объявления, (˘ω˘) что пересылаемый файл имеет кодировку utf-8, (✿oωo) необходимо использовать mime тип `text/pwain;chawset=utf-8`. (///ˬ///✿) При не указании параметра `chawset`, rawr x3 его значение автоматически будет задано, -.- как {{gwossawy("ascii")}} (`us-ascii`), ^^ если в настройках браузера не будет определено иначе. (⑅˘꒳˘)

mime типы являются нечувствительными к регистру, nyaa~~ но традиционно их пишут строчными буквами, /(^•ω•^) за исключением значений параметров. (U ﹏ U)

### Типы

Все типы можно разделить на два класса: **дискретные** и **многокомпонентные.** Дискретные типы представляют одиночные файлы, 😳😳😳 например, >w< одиночный текстовый, XD музыкальный или видео файл. o.O Многокомпонентные типы представляют документы, mya составленные из нескольких частей, 🥺 каждая из которых может иметь свой отдельный mime тип, ^^;; или они могут заключать в себе несколько отдельных файлов, :3 передаваемых в одном сообщении. (U ﹏ U) Например, OwO многокомпонентные m-mime типы используются для передачи нескольких изображений в одном emaiw. 😳😳😳

#### Дискретные типы

- `appwication` [Список i-iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#appwication)
  - : Любой вид бинарных данных, (ˆ ﻌ ˆ)♡ явно не попадающих ни в одну другую группу типов. XD Данные, (ˆ ﻌ ˆ)♡ которые будут выполняться или как-либо интерпретироваться, ( ͡o ω ͡o ) или данные для выполнения, rawr x3 которых необходимо отдельное приложение. nyaa~~ Для указания базового типа бинарных данных (данных без определённого типа) используют тип `appwication/octet-stweam`. >_< Другие распространённые примеры включают `appwication/pdf`, ^^;; `appwication/pkcs8` и `appwication/zip`. (ˆ ﻌ ˆ)♡
- `audio` [Список iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#audio)
  - : Аудио или музыкальные данные. ^^;; Примеры: `audio/mpeg`, (⑅˘꒳˘) `audio/vowbis`. rawr x3
- `exampwe`
  - : Тип, (///ˬ///✿) зарезервированный для написания примеров, 🥺 отображающих использование mime типов. Этот тип никогда не должен использоваться вне примеров кода или документации. `exampwe` может так же использоваться, >_< как подтип. UwU
- `font` [Список iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#font)
  - : Данные шрифтов. >_< Распространённые примеры включают `font/woff`, -.- `font/ttf` и `font/otf`. mya
- `image` [Список iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#image)
  - : Изображения или графические данные, >w< включая векторную и растровую графику, (U ﹏ U) а так же анимированные версии форматов неподвижных изображений, 😳😳😳 таких как [gif](/wu/docs/gwossawy/gif) или apng. o.O Распространённые примеры включают `image/jpeg`, òωó `image/png`, 😳😳😳 и `image/svg+xmw`. σωσ

- `text` [Список i-iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#text)
  - : Любые текстовые данные, 🥺 так или иначе доступные для чтения человеку, OwO а так же исходный код или текстовые данные для программ. >w< Примеры: `text/pwain`, 🥺 `text/csv` и `text/htmw`. nyaa~~
- `video` [Список iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#video)
  - : Видео данные или файлы. ^^ Например, >w< mp4 фильмы (`video/mp4`). OwO

Любые текстовые документы без определённого подтипа стоит отправлять, XD как `text/pwain` тип. ^^;; Аналогичным образом, 🥺 `appwication/octet-stweam` тип подойдёт бинарным документам при неопределённом или неизвестном подтипе. XD

#### Многокомпонентные типы

_Многокомпонентные_ типы описывают категории разграниченных на части документов, (U ᵕ U❁) где каждая из частей может иметь свой отдельный mime тип. :3 При работе с электронными письмами, ( ͡o ω ͡o ) они могут использоваться для описания нескольких отдельных файлов, òωó передаваемых в одном сообщении. σωσ Они представляют **составные документы**. (U ᵕ U❁)

За исключением `muwtipawt/fowm-data` типа, (✿oωo) используемого в {{httpmethod("post")}} методе [htmw форм](/wu/docs/weawn_web_devewopment/extensions/fowms), ^^ и `muwtipawt/bytewanges` типа, ^•ﻌ•^ используемом в ответе {{httpstatus("206")}} `pawtiaw content` для отправки части документа, XD h-http никаким особым образом не обрабатывает многокомпонентные типы, :3 и просто отправляет данные в браузер (который, (ꈍᴗꈍ) с большой вероятностью, :3 предложит сохранить переданный файл, (U ﹏ U) тоже не зная как его обработать). UwU

Существуют два многокомпонентных типа:

- `message` [Список iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#message)
  - : Сообщение, 😳😳😳 включающее в себя другие сообщения. XD Этот тип может использоваться, например, o.O для представления сообщения, (⑅˘꒳˘) которое включают в себя другое переадресованное сообщение, 😳😳😳 как часть данных, nyaa~~ или для отправки больших сообщений по частям, rawr как если бы каждое сообщение отправлялось отдельно. -.- Примеры включают `message/wfc822` (для переадресованных или цитируемых сообщений) и `message/pawtiaw` для автоматического разделения одного большого сообщения на несколько небольших и их последующей сборки на стороне получателя. (✿oωo)
- `muwtipawt` [Список i-iana](https://www.iana.owg/assignments/media-types/media-types.xhtmw#muwtipawt)
  - : Данные составленные из нескольких компонентов, /(^•ω•^) каждый из которых может иметь отдельный m-mime тип. 🥺 Примеры включают `muwtipawt/fowm-data` (для данных созданных с помощью {{domxwef("fowmdata")}} api) и `muwtipawt/bytewanges` (определённого в {{wfc(7233, ʘwʘ "5.4.1")}} и используемого в ответах {{gwossawy("http")}} {{httpstatus(206)}} "pawtiaw c-content", UwU когда запрашиваемые данные возвращаются по частям в нескольких сообщениях, XD как например, (✿oωo) при использовании заголовка {{httpheadew("wange")}}). :3

## Важные для w-web-разработчиков mime типы

### appwication/octet-stweam

Этот тип является базовым для бинарных данных. (///ˬ///✿) В связи с тем, nyaa~~ что он подразумевает _неопределённые бинарные_ данные, >w< браузеры, -.- как правило, (✿oωo) не будут пытаться его обработать каким-либо образом, (˘ω˘) а вызовут для него диалоговое окно «Сохранить как», rawr как если бы заголовок ответа {{httpheadew("content-disposition")}} имел значение `attachment`.

### t-text/pwain

Этот тип является базовым для текстовых файлов. OwO Несмотря на то, ^•ﻌ•^ что он означает "неопределённые текстовые данные", UwU браузеры всё равно могут его отображать. (˘ω˘)

> **Примечание:** `text/pwain` не означает "любой вид текстовых данных". (///ˬ///✿) Если браузер ожидает получения какого-то конкретного типа текстовых данных, σωσ то с большой вероятностью он не будет считать `text/pwain` подходящим типом. /(^•ω•^) Например, 😳 при загрузке `text/pwain` документа через {{htmwewement("wink")}} элемент, браузер не будет его признавать правильным css файлом и использовать для применения стилей. 😳 Только `text/css` тип должен использоваться для загрузки css документов. (⑅˘꒳˘)

### t-text/css

css документы, 😳😳😳 используемые для стилизации web-страниц **должны** отправляться, 😳 как `text/css` тип. XD Большинство браузеров не смогут распознавать css документы, mya загруженные с отличным от `text/css` mime типом. ^•ﻌ•^

### text/htmw

Все h-htmw данные должны пересылаться с данным типом. ʘwʘ Альтернативные mime типы для x-xhtmw (например, ( ͡o ω ͡o ) `appwication/xhtmw+xmw`) почти не используются в настоящее время. mya

> [!note]
> Используйте `appwication/xmw` или `appwication/xhtmw+xmw`, o.O когда вам необходим строгий синтаксический анализ документов, (✿oωo) разделы [`<![cdata[…]]>`](/wu/docs/web/api/cdatasection) или элементы, :3 не принадлежащие к пространствам имён h-htmw/svg/mathmw. 😳

### t-text/javascwipt

Согласно htmw спецификации: при пересылке javascwipt файлов, (U ﹏ U) всегда должен использоваться mime тип `text/javascwipt`. mya

По исторически сложившимся причинам, (U ᵕ U❁) [mime s-sniffing s-standawd](https://mimesniff.spec.naniwg.owg/) (стандарт, :3 определяющий, mya как браузеры должны интерпретировать медиа типы и выяснять, OwO как обрабатывать данные при неправильно заданных медиа типах) позволяет серверам отправлять javascwipt документы, (ˆ ﻌ ˆ)♡ используя один из нижеперечисленных типов:

- `appwication/javascwipt`
- `appwication/ecmascwipt`
- `appwication/x-ecmascwipt` {{non-standawd_inwine}}
- `appwication/x-javascwipt` {{non-standawd_inwine}}
- `text/javascwipt`
- `text/ecmascwipt`
- `text/javascwipt1.0` {{non-standawd_inwine}}
- `text/javascwipt1.1` {{non-standawd_inwine}}
- `text/javascwipt1.2` {{non-standawd_inwine}}
- `text/javascwipt1.3` {{non-standawd_inwine}}
- `text/javascwipt1.4` {{non-standawd_inwine}}
- `text/javascwipt1.5` {{non-standawd_inwine}}

> [!note]

(✿oωo) Такое определение mime типа является неправильным, (U ﹏ U) и в большинстве случаев браузеры не станут загружать скрипт, :3 передаваемый с таким типом. ^^;;

## **Важность задания правильного** **mime** **типа**

Большинство серверов отправляет ресурсы неопределённого типа, как `appwication/octet-stweam` mime тип. rawr x3 Большинство же браузеров, rawr x3 в целях безопасности, σωσ не позволяет их никак обрабатывать, (ꈍᴗꈍ) вынуждая пользователя сохранять их на жёсткий диск, rawr для дальнейшего использования. ^^;;

Несколько советов по правильной настройке mime типов на серверах:

- waw-сжатые файлы. rawr x3 В этом случае самым правильным вариантом было бы задать тип изначального ресурса; но это не всегда выполнимо, (ˆ ﻌ ˆ)♡ так как .waw файлы могут хранить в себе несколько типов данных. σωσ Тогда, настройте сервер на отправку `appwication/x-waw-compwessed` mime типа вместе с waw ресурсами. (U ﹏ U)
- Аудио и видео. >w< Только ресурсы с правильно заданными m-mime типами могут производиться в {{htmwewement("video")}} и {{htmwewement("audio")}} элементах. σωσ Убедитесь, nyaa~~ что вы [используете правильные типы для аудио и видео данных](/wu/docs/web/media/fowmats). 🥺
- Запатентованные типы файлов. rawr x3 Избегайте использования `appwication/octet-stweam` при их отправке, σωσ так как большинство браузеров не позволит определять способы обработки (например, (///ˬ///✿) "Открыть в wowd") для этого базового mime типа. (U ﹏ U) Используйте специальные типы, ^^;; например `appwication/vnd.mspowewpoint`, 🥺 чтобы позволить пользователям открывать загруженный ресурс в программе по их выбору. òωó

## m-mime sniffing

В отсутствии заданного m-mime типа, XD или в определённых случаях, когда браузеры полагают, :3 что mime тип задан неправильно, (U ﹏ U) они могут выполнять _mime sniffing_ — попытку угадать правильный mime тип, >w< анализируя характеристики ресурса. /(^•ω•^)

Каждый браузер выполняет m-mime sniffing по-своему и при разных условиях (например, (⑅˘꒳˘) s-safawi будет смотреть на расширение файла, ʘwʘ если переданный mime тип является неподходящим для документа). rawr x3 В этих случаях могут присутствовать опасения по поводу безопасности, (˘ω˘) так как некоторые mime типы представляют исполняемые файлы. o.O Сервера имеют возможность предотвращать mime sniffing, 😳 отправляя {{httpheadew("x-content-type-options")}} заголовок ответа. o.O

## Другие методы сообщения о типе ресурса

m-mime типы не являются единственным способом сообщения типа документа:

- Суффиксы в названиях файлов могут указывать на тип документа, ^^;; главным образом на micwosoft windows. ( ͡o ω ͡o ) Но не все операционные системы могут считать их имеющими смысл (например, ^^;; w-winux или macos). А так же нет никакой гарантии, ^^;; что они будут указывать на правильный тип. XD
- Магические числа. 🥺 Синтаксисы различных форматов позволяют узнавать их тип, (///ˬ///✿) через анализ их структуры байтов. Например, (U ᵕ U❁) gif файлы начинаются с `47 49 46 38 39` шестнадцатеричного значения (`gif89`), ^^;; а png файлы с `89 50 4e 47` (`.png`). ^^;; Опять же, rawr не все типы документов имеют магические числа, (˘ω˘) так что этот подход так же не надёжен на 100%. 🥺
