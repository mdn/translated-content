Элемент {{htmlelement("meta")}} используется для предоставления метаданных документа в формате "имя-значение". Атрибут [`name`](/ru/docs/Web/HTML/Element/meta#name) задает имя метаданных, а атрибут [`content`](/ru/docs/Web/HTML/Element/meta#content) определяет их значение.

### Стандартные имена метаданных, определенные в спецификации HTML

Спецификация HTML определяет следующий набор стандартных имен метаданных:

- `application-name`: имя приложения, функционирующего на веб-странице.

  > [!NOTE]
  >
  > - Браузеры могут использовать это значение для идентификации приложения. Это отличается от элемента {{HTMLElement("title")}}, который обычно включает имя приложения, а также дополнительную информацию, например, название документа или его статус.
  > - Простые веб-страницы не должны использовать `application-name`.

- `author`: имя автора документа.
- `description`: краткое и точное описание содержания страницы. Поисковые системы, такие как [Google](https://developers.google.com/search/docs/appearance/snippet#meta-descriptions), используют это поле для формирования внешнего вида страницы в результатах поиска.
- `generator`: название программного обеспечения, которое сгенерировало страницу.
- `keywords`: ключевые слова, относящиеся к содержанию страницы, перечисленные через запятую.
- `referrer`: определяет, каким образом HTTP-заголовок {{httpheader("Referer")}} должен отправляться при запросах с этой страницы:

  <table class="standard-table">
    <caption>Значения атрибута <code>content</code> для <code>&#x3C;meta name="referrer"></code></caption>
    <tbody>
      <tr>
        <td><code>no-referrer</code></td>
        <td>Не отправлять HTTP-заголовок {{httpheader("Referer")}}.</td>
      </tr>
      <tr>
        <td><code>origin</code></td>
        <td>Отправлять только {{glossary("origin")}} документа.</td>
      </tr>
      <tr>
        <td><code>no-referrer-when-downgrade</code></td>
        <td>
          Отправлять полный URL, если целевая страница не менее безопасна, чем текущая (например, HTTPS → HTTPS). Не отправлять реферер для менее безопасных переходов (например, HTTPS → HTTP). Это поведение по умолчанию.
        </td>
      </tr>
      <tr>
        <td><code>origin-when-cross-origin</code></td>
        <td>Отправлять полный URL (без параметров) для запросов того же источника, но только origin для других случаев.</td>
      </tr>
      <tr>
        <td><code>same-origin</code></td>
        <td>Отправлять полный URL (без параметров) только для запросов того же источника. Межсайтовые запросы не содержат заголовок реферера.</td>
      </tr>
      <tr>
        <td><code>strict-origin</code></td>
        <td>
          Отправлять origin только для запросов к более безопасным целевым страницам (например, HTTP → HTTPS). Не отправлять реферер для менее безопасных переходов (например, HTTPS → HTTP).
        </td>
      </tr>
      <tr>
        <td><code>strict-origin-when-cross-origin</code></td>
        <td>
          Отправлять полный URL (без параметров) для запросов того же источника. Для межсайтовых запросов отправлять только origin, если целевая страница не менее безопасна, чем текущая (например, HTTPS → HTTPS). Иначе — не отправлять реферер.
        </td>
      </tr>
      <tr>
        <td><code>unsafe-URL</code></td>
        <td>Отправлять полный URL (без параметров) независимо от источника запроса.</td>
      </tr>
    </tbody>
  </table>

  > [!NOTE]
  >
  > - Динамическое добавление `<meta name="referrer">` (например, с использованием {{domxref("Document.write", "document.write()")}} или {{domxref("Node.appendChild", "appendChild()")}}) может привести к непредсказуемому поведению заголовка реферера.
  > - При наличии нескольких конфликтующих значений применяется политика `no-referrer`.

- [`theme-color`](/ru/docs/Web/HTML/Element/meta/name/theme-color): указывает рекомендуемый цвет, который браузеры могут использовать для оформления страницы или интерфейса пользователя. Атрибут `content` принимает валидное CSS-значение {{cssxref("&lt;color&gt;")}}. Также может быть добавлен атрибут `media` для применения цвета в зависимости от медиазапросов.

- `color-scheme`: указывает одну или несколько цветовых схем, совместимых с документом.

  Браузеры используют эту информацию совместно с настройками пользователя для определения цветовой палитры, включая фон, передний план, элементы управления формой и скроллбары. Основное назначение `<meta name="color-scheme">` — указать совместимость и предпочтения, такие как поддержка светлого и темного режимов.

  Значения атрибута [`content`](/ru/docs/Web/HTML/Element/meta#content) для `color-scheme` могут быть следующими:

  - `normal`
    - : Документ не поддерживает цветовые схемы и должен отображаться с использованием стандартной палитры.
  - `light`, `dark`, `light dark`, `dark light`
    - : Указывает одну или несколько поддерживаемых цветовых схем. Первая из них считается предпочтительной.
  - `only light`
    - : Указывает, что документ поддерживает _только_ светлый режим.

  Например, чтобы указать предпочтение темного режима, но оставить возможность отображения в светлом режиме:

  ```html
  <meta name="color-scheme" content="dark light" />
  ```

  Это работает на уровне документа так же, как CSS-свойство {{cssxref("color-scheme")}} позволяет отдельным элементам указывать свои предпочтительные и допустимые цветовые схемы. Ваши стили могут адаптироваться к текущей цветовой схеме с использованием CSS-медиафункции {{cssxref("@media/prefers-color-scheme", "prefers-color-scheme")}}.

### Стандартные имена метаданных, определенные в других спецификациях

Спецификация CSS Device Adaptation определяет следующее имя метаданных:

- `viewport`: дает подсказки о начальном размере для {{glossary("viewport")}}.

  <table class="fullwidth-table">
    <caption>
      Значения для содержимого
      <code>&#x3C;meta name="viewport"></code>
    </caption>
    <thead>
      <tr>
        <th scope="col">Значение</th>
        <th scope="col">Возможные подзначения</th>
        <th scope="col">Описание</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td><code>width</code></td>
        <td>Положительное целое число или текст <code>device-width</code></td>
        <td>Определяет ширину viewport, при которой должна отображаться веб-страница.</td>
      </tr>
      <tr>
        <td><code>height</code></td>
        <td>Положительное целое число или текст <code>device-height</code></td>
        <td>Определяет высоту viewport. Не используется никаким браузером.</td>
      </tr>
      <tr>
        <td><code>initial-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет соотношение между шириной устройства (<code>device-width</code> в портретной ориентации или <code>device-height</code> в альбомной) и размером viewport.</td>
      </tr>
      <tr>
        <td><code>maximum-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет максимальный уровень масштабирования страницы. Должен быть больше или равен <code>minimum-scale</code>, иначе поведение не определено. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>minimum-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет минимальный уровень масштабирования страницы. Должен быть меньше или равен <code>maximum-scale</code>, иначе поведение не определено. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>user-scalable</code></td>
        <td><code>yes</code> или <code>no</code></td>
        <td>Если установлено <code>no</code>, пользователю запрещено увеличивать веб-страницу. По умолчанию установлено значение <code>yes</code>. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>viewport-fit</code></td>
        <td><code>auto</code>, <code>contain</code> или <code>cover</code></td>
        <td>
          <p>Значение <code>auto</code> не влияет на начальный viewport, и вся веб-страница доступна для просмотра.</p>
          <p>Значение <code>contain</code> означает, что viewport масштабируется для размещения наибольшего прямоугольника, вписанного в дисплей.</p>
          <p>Значение <code>cover</code> означает, что viewport масштабируется для заполнения экрана устройства. Настоятельно рекомендуется использовать переменные <a href="/ru/docs/Web/CSS/env">safe area inset</a> для обеспечения видимости важного содержимого.</p>
        </td>
      </tr>
    </tbody>
  </table>

  > [!WARNING]
  >
  > Отключение возможности масштабирования, установив `user-scalable` в `no`, не дает пользователям с нарушениями зрения читать и понимать содержимое страницы.
  >
  > - [MDN Understanding WCAG, Guideline 1.4 explanations](/ru/docs/Web/Accessibility/Understanding_WCAG/Perceivable#guideline_1.4_make_it_easier_for_users_to_see_and_hear_content_including_separating_foreground_from_background)
  > - [Understanding Success Criterion 1.4.4 | W3C Understanding WCAG 2.0](https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-scale.html)

### Другие имена метаданных

[Страница MetaExtensions на Wiki WHATWG](https://wiki.whatwg.org/wiki/MetaExtensions) содержит большой набор нестандартных имен метаданных, которые еще не были официально приняты; однако некоторые из имен, представленных там, уже широко используются на практике — включая следующие:

- `creator`: имя создателя документа, например, организации или учреждения. Если создателей несколько, следует использовать несколько элементов {{HTMLElement("meta")}}.
- `googlebot`: синоним `robots`, используется только Googlebot (индексирующим роботом Google).
- `publisher`: имя издателя документа.
- `robots`: поведение, которое должны использовать поисковые сканеры, или "роботы", на странице. Это список значений, разделенных запятыми:

  | Значение       | Описание                                                                                           | Используется                                                                                                                                                                                                                                             |
  | -------------- | -------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | `index`        | Позволяет роботу индексировать страницу (по умолчанию).                                            | Все                                                                                                                                                                                                                                                      |
  | `noindex`      | Просит робота не индексировать страницу.                                                           | Все                                                                                                                                                                                                                                                      |
  | `follow`       | Позволяет роботу следовать по ссылкам на странице (по умолчанию).                                  | Все                                                                                                                                                                                                                                                      |
  | `nofollow`     | Просит робота не следовать по ссылкам на странице.                                                 | Все                                                                                                                                                                                                                                                      |
  | `all`          | Эквивалентно `index, follow`                                                                       | [Google](https://developers.google.com/search/docs/crawling-indexing/special-tags?visit_id=637855965067987211-415685194&rd=1)                                                                                                                            |
  | `none`         | Эквивалентно `noindex, nofollow`                                                                   | [Google](https://developers.google.com/search/docs/crawling-indexing/special-tags?visit_id=637855965074074862-574753619&rd=1)                                                                                                                            |
  | `noarchive`    | Просит поисковую систему не кешировать содержимое страницы.                                        | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag), [Yahoo](https://help.yahoo.com/kb/search-for-desktop/SLN2213.html), [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240) |
  | `nosnippet`    | Запрещает отображение любого описания страницы в результатах поиска.                               | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag), [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240)                                                                     |
  | `noimageindex` | Просит, чтобы эта страница не появлялась как ссылающаяся страница для индексированных изображений. | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag)                                                                                                                                                                    |
  | `nocache`      | Синоним `noarchive`.                                                                               | [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240)                                                                                                                                                            |

  > [!NOTE]
  >
  > - Только поисковые роботы следуют этим правилам. Не рассчитывайте на то, что они предотвратят сбор адресов электронной почты.
  > - Роботу все равно нужно получить доступ к странице, чтобы прочитать эти правила. Чтобы не терять трафик, рассмотрите использование _{{Glossary("robots.txt")}}_ файла.
  > - Тег `robots` `<meta>` и файл `robots.txt` служат разным целям: `robots.txt` контролирует сканирование страниц, но не влияет на индексацию или другие действия, регулируемые мета-тегом `robots`. Страница, которая не может быть отсканирована, все равно может быть индексирована, если на нее ссылается другой документ.
  > - Если вы хотите удалить страницу, `noindex` сработает, но только после того, как робот снова посетит эту страницу. Убедитесь, что файл `robots.txt` не препятствует повторным посещениям.
  > - Некоторые значения исключают друг друга, например, `index` и `noindex`, или `follow` и `nofollow`. В таких случаях поведение роботов не определено и может варьироваться.
  > - Некоторые роботы, такие как Google, Yahoo и Bing, поддерживают аналогичные значения для HTTP-заголовка `X-Robots-Tag`. Это позволяет применять эти правила к не-HTML-документам, например изображениям.
