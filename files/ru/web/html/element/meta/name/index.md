Элемент {{htmlelement("meta")}} может использоваться для предоставления метаданных документа в виде пар "имя-значение", где атрибут [`name`](/ru/docs/Web/HTML/Element/meta#name) задает имя метаданных, а атрибут [`content`](/ru/docs/Web/HTML/Element/meta#content) задает значение.

### Стандартные имена метаданных, определенные в спецификации HTML

Спецификация HTML определяет следующий набор стандартных имен метаданных:

- `application-name`: имя приложения, работающего на веб-странице.

  > [!NOTE]
  >
  > - Браузеры могут использовать это для идентификации приложения. Это отличается от элемента {{HTMLElement("title")}}, который обычно содержит имя приложения, но также может включать информацию, такую как имя документа или статус.
  > - Простым веб-страницам не следует определять `application-name`.

- `author`: имя автора документа.
- `description`: краткое и точное описание содержимого страницы. Поисковые системы, такие как [Google](https://developers.google.com/search/docs/appearance/snippet#meta-descriptions), могут использовать это поле для управления внешним видом веб-страницы в результатах поиска.
- `generator`: идентификатор программного обеспечения, которое создало страницу.
- `keywords`: слова, относящиеся к содержимому страницы, разделенные запятыми.
- `referrer`: управляет HTTP-заголовком {{httpheader("Referer")}} для запросов, отправленных с документа:

          Отправить origin, если пункт назначения по крайней мере такой же

  > [!NOTE]
  >
  > - Динамическое добавление `<meta name="referrer">` (с помощью {{domxref("Document.write", "document.write()")}} или {{domxref("Node.appendChild", "appendChild()")}}) делает поведение реферера непредсказуемым.
  > - Когда определено несколько конфликтующих политик, применяется политика `no-referrer`.

- [`theme-color`](/en-US/docs/Web/HTML/Element/meta/name/theme-color): указывает рекомендуемый цвет, который юзер-агенты () должны использовать для настройки отображения страницы или окружающего интерфейса пользователя. Атрибут `content` содержит валидный CSS {{cssxref("&lt;color&gt;")}}. Атрибут `media` с допустимым списком медиазапросов может быть включен для установки медиа, к которым применяется цветовая схема.
- <a id="color-scheme" href="#color-scheme">`color-scheme`</a>: указывает одну или несколько цветовых схем, с которыми совместим документ.

  Браузер будет использовать эту информацию в тандеме с настройками браузера или устройства пользователя, чтобы определить, какие цвета использовать для всего, от фона и переднего плана до элементов управления формой и скроллбаров. Основное использование `<meta name="color-scheme">` заключается в указании совместимости и предпочтительности светлого и темного режимов.

  Значение свойства [`content`](/ru/docs/Web/HTML/Element/meta#content) для `color-scheme` может быть следующим:

  - `normal`
    - : Документ не знает о цветовых схемах и должен отображаться с использованием дефолтной цветовой палитры.
  - `light`, `dark`, `light dark`, `dark light`
    - : Одна или несколько цветовых схем, поддерживаемых документом. Повторное указание одной и той же цветовой схемы имеет тот же эффект, что и указание ее только один раз. Указание нескольких цветовых схем указывает, что первая схема предпочтительна для документа, но вторая также допустима, если пользователь предпочитает ее.
  - `only light`
    - : Указывает, что документ поддерживает _только_ светлый режим, со светлым фоном и темным цветом переднего плана. По спецификации `only dark` _недействителен_, так как принудительное отображение документа в темном режиме, если он не совместим с ним, может привести к нечитабельному содержимому; все основные браузеры по умолчанию используют светлый режим, если не указано иное.

  Например, чтобы указать, что документ предпочитает темный режим, но также функционально отображается в светлом режиме:

  ```html
  <meta name="color-scheme" content="dark light" />
  ```

  Это работает на уровне документа так же, как CSS свойство {{cssxref("color-scheme")}} позволяет отдельным элементам указывать свои предпочтительные и допустимые цветовые схемы. Ваши стили могут адаптироваться к текущей цветовой схеме с использованием CSS-медиафункции {{cssxref("@media/prefers-color-scheme", "prefers-color-scheme")}}.

### Стандартные имена метаданных, определенные в других спецификациях

Спецификация CSS Device Adaptation определяет следующее имя метаданных:

- `viewport`: дает подсказки о начальном размере для {{glossary("viewport")}}.

  <table class="fullwidth-table">
    <caption>
      Значения для содержимого
      <code>&#x3C;meta name="viewport"></code>
    </caption>
    <thead>
      <tr>
        <th scope="col">Значение</th>
        <th scope="col">Возможные подзначения</th>
        <th scope="col">Описание</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td><code>width</code></td>
        <td>Положительное целое число или текст <code>device-width</code></td>
        <td>Определяет ширину viewport, при которой должна отображаться веб-страница.</td>
      </tr>
      <tr>
        <td><code>height</code></td>
        <td>Положительное целое число или текст <code>device-height</code></td>
        <td>Определяет высоту viewport. Не используется никаким браузером.</td>
      </tr>
      <tr>
        <td><code>initial-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет соотношение между шириной устройства (<code>device-width</code> в портретной ориентации или <code>device-height</code> в альбомной) и размером viewport.</td>
      </tr>
      <tr>
        <td><code>maximum-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет максимальный уровень масштабирования страницы. Должен быть больше или равен <code>minimum-scale</code>, иначе поведение не определено. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>minimum-scale</code></td>
        <td>Положительное число от <code>0.0</code> до <code>10.0</code></td>
        <td>Определяет минимальный уровень масштабирования страницы. Должен быть меньше или равен <code>maximum-scale</code>, иначе поведение не определено. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>user-scalable</code></td>
        <td><code>yes</code> или <code>no</code></td>
        <td>Если установлено <code>no</code>, пользователю запрещено увеличивать веб-страницу. По умолчанию установлено значение <code>yes</code>. Настройки браузера могут игнорировать это правило, а iOS10+ игнорирует его по умолчанию.</td>
      </tr>
      <tr>
        <td><code>viewport-fit</code></td>
        <td><code>auto</code>, <code>contain</code> или <code>cover</code></td>
        <td>
          <p>Значение <code>auto</code> не влияет на начальный viewport, и вся веб-страница доступна для просмотра.</p>
          <p>Значение <code>contain</code> означает, что viewport масштабируется для размещения наибольшего прямоугольника, вписанного в дисплей.</p>
          <p>Значение <code>cover</code> означает, что viewport масштабируется для заполнения экрана устройства. Настоятельно рекомендуется использовать переменные <a href="/en-US/docs/Web/CSS/env">safe area inset</a> для обеспечения видимости важного содержимого.</p>
        </td>
      </tr>
    </tbody>
  </table>

  > [!WARNING]
  >
  > Отключение возможности масштабирования, установив `user-scalable` в `no`, не дает пользователям с нарушениями зрения читать и понимать содержимое страницы.
  >
  > - [MDN Understanding WCAG, Guideline 1.4 explanations](/en-US/docs/Web/Accessibility/Understanding_WCAG/Perceivable#guideline_1.4_make_it_easier_for_users_to_see_and_hear_content_including_separating_foreground_from_background)
  > - [Understanding Success Criterion 1.4.4 | W3C Understanding WCAG 2.0](https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-scale.html)

### Другие имена метаданных

[Страница MetaExtensions на Wiki WHATWG](https://wiki.whatwg.org/wiki/MetaExtensions) содержит большой набор нестандартных имен метаданных, которые еще не были официально приняты; однако некоторые из имен, представленных там, уже широко используются на практике — включая следующие:

- `creator`: имя создателя документа, например, организации или учреждения. Если создателей несколько, следует использовать несколько элементов {{HTMLElement("meta")}}.
- `googlebot`: синоним `robots`, используется только Googlebot (индексирующим роботом Google).
- `publisher`: имя издателя документа.
- `robots`: поведение, которое должны использовать поисковые сканеры, или "роботы", на странице. Это список значений, разделенных запятыми:

  | Значение       | Описание                                                                                           | Используется                                                                                                                                                                                                                                             |
  |----------------|----------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
  | `index`        | Позволяет роботу индексировать страницу (по умолчанию).                                            | Все                                                                                                                                                                                                                                                      |
  | `noindex`      | Просит робота не индексировать страницу.                                                           | Все                                                                                                                                                                                                                                                      |
  | `follow`       | Позволяет роботу следовать по ссылкам на странице (по умолчанию).                                  | Все                                                                                                                                                                                                                                                      |
  | `nofollow`     | Просит робота не следовать по ссылкам на странице.                                                 | Все                                                                                                                                                                                                                                                      |
  | `all`          | Эквивалентно `index, follow`                                                                       | [Google](https://developers.google.com/search/docs/crawling-indexing/special-tags?visit_id=637855965067987211-415685194&rd=1)                                                                                                                            |
  | `none`         | Эквивалентно `noindex, nofollow`                                                                   | [Google](https://developers.google.com/search/docs/crawling-indexing/special-tags?visit_id=637855965074074862-574753619&rd=1)                                                                                                                            |
  | `noarchive`    | Просит поисковую систему не кешировать содержимое страницы.                                        | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag), [Yahoo](https://help.yahoo.com/kb/search-for-desktop/SLN2213.html), [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240) |
  | `nosnippet`    | Запрещает отображение любого описания страницы в результатах поиска.                               | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag), [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240)                                                                     |
  | `noimageindex` | Просит, чтобы эта страница не появлялась как ссылающаяся страница для индексированных изображений. | [Google](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag)                                                                                                                                                                    |
  | `nocache`      | Синоним `noarchive`.                                                                               | [Bing](https://www.bing.com/webmasters/help/which-robots-metatags-does-bing-support-5198d240)                                                                                                                                                            |

  > [!NOTE]
  >
  > - Только поисковые роботы следуют этим правилам. Не рассчитывайте на предотвращение сбора адресов электронной почты с их помощью.
  > - Роботу все равно нужно получить доступ к странице, чтобы прочитать эти правила. Чтобы избежать пропуск трафика, рассмотрите возможность использования _{{Glossary("robots.txt")}}_ файла.
  > - Тег `robots` `<meta>` и файл `robots.txt` служат разным целям: `robots.txt` контролирует сканирование страниц, но не влияет на индексацию или другие действия, регулируемые мета-тегом `robots`. Страница, которая не может быть отсканирована, все равно может быть индексирована, если на нее ссылается другой документ.
  > - Если вы хотите удалить страницу, `noindex` сработает, но только после того, как робот снова посетит страницу. Убедитесь, что файл `robots.txt` не препятствует повторным посещениям.
  > - Некоторые значения взаимно исключают друг друга, например, `index` и `noindex`, или `follow` и `nofollow`. В таких случаях поведение робота не определено и может варьироваться.
  > - Некоторые роботы, такие как Google, Yahoo и Bing, поддерживают те же значения для HTTP-заголовка `X-Robots-Tag`; это позволяет использовать эти правила для не-HTML документов, таких как изображения.
