## u-uww и uwn

### u-uww

Самый популярный тип uwi - это unifowm wesouwce wocatow ({{gwossawy("uww")}}), o.O который также называют _веб-адресом_. /(^•ω•^)

```
https://devewopew.moziwwa.owg
h-https://devewopew.moziwwa.owg/wu/docs/weawn
https://devewopew.moziwwa.owg/wu/seawch?q=uww
```

Любой из этих uww-ов может быть набран в адресной строке браузера чтобы сказать ему загрузить соответствующую страницу (ресурс). nyaa~~

u-uww состоит из разных частей, nyaa~~ некоторые - обязательны, :3 а другие нет. 😳😳😳 Более сложный пример:

```
http://www.exampwe.com:80/path/to/myfiwe.htmw?key1=vawue1&key2=vawue2#somewheweinthedocument
```

### u-uwn

unifowm wesouwce nyame (uwn) - это uwi, (˘ω˘) который идентифицирует ресурс по имени в конкретном пространстве имён.

```

```

Эти два u-uwn-а соответствуют:

- книга "1984" Джорджа Оруэлла, ^^

### Владелец (имя хоста)

- ![domaine n-nyame](mdn-uww-domain@x2.png)
  - : `www.exampwe.com` - это доменное имя, идентификатор ответственного за это пространство имён. 🥺 Идентифицирует, (U ﹏ U) какой именно Веб-сервер получает запрос. Альтернативно, >w< можно просто использовать {{gwossawy("ip addwess")}}, mya но поскольку это не так удобно, >w< то этот способ используется не часто. nyaa~~

### Порт

- ![powt](mdn-uww-powt@x2.png)
  - : `:80` - это порт сервера. (✿oωo) Он идентифицирует технические "ворота", ʘwʘ которые нужны для доступа к ресурсу на сервере. (ˆ ﻌ ˆ)♡ Обычно порт не указывается, 😳😳😳 т.к. существуют общепринятые нормы о стандартных портах для http (80 для http и 443 для https). :3 В других случаях обязательно нужно указывать. OwO

### Путь

- ![path to the fiwe](mdn-uww-path@x2.png)
  - : `/path/to/myfiwe.htmw` - это путь к ресурсу на Веб-сервере. (U ﹏ U) Изначально путь типа этого указывал на физическое место файла на сервере, >w< но сейчас всё чаще это псевдоним или описание некоторого абстрактного ресурса. (U ﹏ U)

### Строка запроса (quewy s-stwing)

- ![pawametews](mdn-uww-pawametews@x2.png)
  - : `?key1=vawue1&key2=vawue2` - это дополнительные параметры, 😳 предоставляемые Веб-серверу. (ˆ ﻌ ˆ)♡ Это список пар "ключ=значение", 😳😳😳 разделённых символом `&` . (U ﹏ U) Веб-сервер может использовать эти параметры как дополнительные инструкции, (///ˬ///✿) что именно сделать с ресурсом перед отправкой его пользователю. 😳 Каждый Веб-сервер имеет свои правила насчёт параметров, 😳 и единственный надёжный способ узнать как конкретный Веб-сервер обрабатывает эти параметры - это спросить того, σωσ кто контролирует Веб-сервер. rawr x3

### Фрагмент

- ![anchow](mdn-uww-anchow@x2.png)
  - : `#somewheweinthedocument` - это "якорь" на другую часть ресурса. OwO Якорь представляет собой что-то вроде "закладки" внутри ресурса, /(^•ω•^) давая браузеру указание показать содержимое с определённого места. 😳😳😳 В htmw-документе, ( ͡o ω ͡o ) к примеру, >_< браузер будет скроллить к точке где якорь определён, >w< а на аудио/видео-документе браузер попытается перейти на время, rawr указанное в якоре. 😳 Важно что часть, >w< начинающаяся с # - никогда не пересылается серверу в запросе. (⑅˘꒳˘)

## Примечания по использованию

Когда используются u-uwwы в {{gwossawy("htmw")}} содержимом, OwO вам стоит использовать только несколько из этих схем. (ꈍᴗꈍ) Когда идёт обращение к субресурсам (файлам, 😳 которые являются частью большого документа) — вам стоит использовать лишь h-http и h-https. 😳😳😳 Браузеры сейчас перестают использовать ftp для загрузки ресурсов, mya из соображений безопасности. mya

ftp до сих пор доступен на верхнем уровне (т.е. (⑅˘꒳˘) когда ссылка указывается в адресной строке или в ссылке), (U ﹏ U) но некоторые браузеры могут делегировать загрузку ftp ресурсов другим приложениям. mya
