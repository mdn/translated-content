Возможности веба для представления, создания и управления аудио, видео и другим медиаконтентом со временем значительно расширились. Сегодня существует множество API, HTML-элементов, интерфейсов DOM и других функций, которые предоставляют гибкие инструменты для работы с медиа. Эта статья содержит руководства и справочные материалы по различным функциям, которые можно использовать при интеграции медиа в ваши проекты.

## Руководства

- [Передача аудио и видео](/ru/docs/Web/Media/Guides/Audio_and_video_delivery)
  - : Мы можем передавать аудио и видео в вебе разными способами: от «статичных» медиафайлов до адаптивной потоковой передачи в реальном времени. Эта статья служит отправной точкой для изучения различных механизмов доставки медиа и их совместимости с популярными браузерами.
- [Обработка аудио и видео](/ru/docs/Web/Media/Guides/Audio_and_video_manipulation)
  - : Нативная поддержка аудио и видео в браузере позволяет использовать эти потоки данных с такими технологиями, как {{htmlelement("canvas")}}, [WebGL](/ru/docs/Web/API/WebGL_API) или [Web Audio API](/ru/docs/Web/API/Web_Audio_API) для непосредственного изменения, например добавления эффектов реверберации/компрессии к аудио или применения фильтров в оттенках серого/сепии к видео.
- [Руководство по автоматическому воспроизведению для медиа и Web Audio APIs](/ru/docs/Web/Media/Guides/Autoplay)
  - : Неожиданное автоматическое воспроизведение медиа или аудио может неприятно удивить пользователей. Хотя автоматическое воспроизведение может быть полезным, его следует использовать осторожно. Чтобы пользователи могли сами контролировать воспроизведение, многие браузеры теперь предлагают механизмы блокировки автозапуска. В этом руководстве даются советы о том, когда и как использовать автозапуск и как корректно работать с его блокировкой в браузерах.
- [Адаптивная потоковая передача (DASH) для HTML 5 Video](/ru/docs/Web/API/Media_Source_Extensions_API/DASH_Adaptive_Streaming)
  - : DASH (Dynamic Adaptive Streaming over HTTP) — это протокол адаптивной потоковой передачи, который позволяет переключать качество видео в зависимости от состояния сети для обеспечения непрерывного воспроизведения.
- [Потоковая передача аудио и видео](/ru/docs/Web/Media/Guides/Streaming)
  - : Руководство, охватывающее методы и технологии потоковой передачи аудио и видео, а также способы обеспечения наилучшего качества и производительности ваших потоков.
- [Типы и форматы медиа](/ru/docs/Web/Media/Guides/Formats)
  - : Руководство по типам файлов и кодекам для изображений, аудио и видео в вебе. В нём даются рекомендации по выбору подходящих форматов для разных видов контента, рассматриваются лучшие практики (в том числе использование резервных вариантов), а также приводится информация о поддержке форматов различными браузерами.
- [Использование изображений в HTML](/ru/docs/Web/Media/Guides/Images)
  - : Руководство по добавлению изображений на сайты с учётом адаптивности, доступности и производительности.

## Справочные материалы

### HTML

Описание HTML-элементов, используемых для работы с медиа:

- {{HTMLElement("audio")}}
  - : Элемент `<audio>` используется для воспроизведения аудио. Его можно использовать незаметно, как целевой элемент для более сложных медиа-приложений, или с видимыми элементами управления для воспроизведения аудиофайлов пользователем. Доступен из JavaScript как объект {{domxref("HTMLAudioElement")}}.
- {{HTMLElement("video")}}
  - : Элемент `<video>` используется для воспроизведения видеофайлов. Его можно использовать для показа видеоконтента или как получатель потокового видео. `<video>` также может использоваться для связи медиа-API с другими HTML- и DOM-технологиями, включая {{HTMLElement("canvas")}} (например, для захвата и обработки кадров). Доступен из JavaScript как объект {{domxref("HTMLVideoElement")}}.
- {{HTMLElement("track")}}
  - : Элемент HTML `<track>` может быть размещён внутри элемента {{HTMLElement("audio")}} или {{HTMLElement("video")}} для указания ссылки на дорожку субтитров или титров в формате [WebVTT](/ru/docs/Web/API/WebVTT_API), которая будет использоваться при воспроизведении медиафайла. Доступен из JavaScript как объект {{domxref("HTMLTrackElement")}}.
- {{HTMLElement("source")}}
  - : Элемент HTML `<source>` используется внутри элемента {{HTMLElement("audio")}} или {{HTMLElement("video")}} для указания исходного медиафайла для воспроизведения. Можно указать несколько источников, чтобы предоставить медиа в разных форматах, размерах или разрешениях. Доступен из JavaScript как объект {{domxref("HTMLSourceElement")}}.

### APIs

- [Media Capabilities API](/ru/docs/Web/API/Media_Capabilities_API)
  - : API Media Capabilities позволяет определить возможности устройства по кодированию и декодированию медиа. Это даёт возможность принимать решения в реальном времени о том, какие форматы использовать и когда.
- [Media Capture and Streams API](/ru/docs/Web/API/Media_Capture_and_Streams_API)
  - : Справочник по API, который позволяет транслировать, записывать и обрабатывать медиа как локально, так и через сеть. Например, использовать камеры и микрофоны для захвата видео, аудио и фотографий.
- [Media Session API](/ru/docs/Web/API/Media_Session_API)
  - : Media Session API предоставляет способ настраивать медиа-уведомления. Для этого передаются метаданные, которые пользовательский агент (User Agent) может отображать для медиа, воспроизводимого вашим веб-приложением. Также API предоставляет обработчики действий, которые позволяют браузеру реагировать на медиа-клавиши устройства — как аппаратные (на клавиатурах, гарнитурах, пультах), так и программные (в областях уведомлений и на экранах блокировки мобильных устройств).
- [MediaStream Recording API](/ru/docs/Web/API/MediaStream_Recording_API)
  - : MediaStream Recording API позволяет захватывать медиапотоки для последующей обработки, фильтрации данных или записи на диск.
- [Web Audio API](/ru/docs/Web/API/Web_Audio_API)
  - : Web Audio API позволяет генерировать, фильтровать и обрабатывать звуковые данные как в реальном времени, так и на предварительно записанном материале, а затем выводить звук, например, через элемент `<audio>`, медиапоток или на диск.
- [WebRTC](/ru/docs/Web/API/WebRTC_API)
  - : WebRTC (Web Real-Time Communication) позволяет передавать потоковое аудио и видео в реальном времени, а также произвольные данные между двумя узлами (пирами) через Интернет без необходимости в промежуточном сервере.

## Related topics

Темы, которые могут быть интересны, так как они могут использоваться вместе с медиа API различными полезными способами.

- [Доступный мультимедиа-контент](/ru/docs/Learn_web_development/Core/Accessibility/Multimedia)
  - : В этом руководстве описаны способы, с помощью которых веб-дизайнеры и разработчики могут создавать контент, доступный для людей с разными возможностями. Это включает использование атрибута [`alt`](/ru/docs/Web/HTML/Reference/Elements/img#alt) для элементов {{HTMLElement("img")}} добавление субтитров и тегирование медиа для экранных читалок.
- [Canvas API](/ru/docs/Web/API/Canvas_API)
  - : Canvas API позволяет рисовать внутри элемента {{HTMLElement("canvas")}},изменяя и обрабатывая содержимое изображения. Это может быть использовано с медиа, например, для вывода видео или съёмки с камеры в элемент `<canvas>` с возможностью захвата и обработки видеокадров.
- [WebGL](/ru/docs/Web/API/WebGL_API)
  - : WebGL предоставляет API, совместимый с OpenGL ES, поверх Canvas API, позволяя реализовать высокопроизводительную 3D-графику в браузере. Через `<canvas>` это позволяет добавлять 3D-графику в медиаконтент.
- [WebXR](/ru/docs/Web/API/WebXR_Device_API)
  - : WebXR, заменивший устаревший API WebVR, — это технология для создания контента в виртуальной (VR) и дополненной (AR) реальности. Такой смешанный контент может отображаться на экране устройства или с помощью гарнитур.
- [WebVR](/ru/docs/Web/API/WebVR_API) {{deprecated_inline}}
  - : API Web Virtual Reality поддерживает устройства виртуальной реальности (VR), такие как Oculus Rift и HTC Vive, позволяя разработчикам передавать положение и движение пользователя в перемещения внутри 3D-сцены. WebVR устарел и был заменён WebXR; его поддержка будет удалена из браузеров в ближайшее время.
- [Web Audio spatialization basics](/ru/docs/Web/API/Web_Audio_API/Web_audio_spatialization_basics)
  - : В 3D-средах (будь то отрисованные сцены на экране или VR/AR-опыт через гарнитуру) важно, чтобы звук воспроизводился с учётом направления источника. В этом руководстве объясняется, как этого достичь.
