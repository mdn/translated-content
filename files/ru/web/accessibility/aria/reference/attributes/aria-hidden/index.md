`aria-hidden` — состояние, которое указывает, доступен ли элемент для API доступности.

## Описание

Атрибут `aria-hidden` может использоваться для скрытия неинтерактивного содержимого от API доступности.

Добавление `aria-hidden="true"` к элементу удаляет этот элемент и все его дочерние элементы из дерева доступности (accessibility tree). Это может улучшить опыт для пользователей вспомогательных технологий, скрывая:

*   Сугубо декоративный контент, такой как иконки или изображения.
*   Продублированный контент, например, повторяющийся текст.
*   Содержимое, находящееся за пределами видимой области экрана или свёрнутое (например, меню).

Наличие атрибута `aria-hidden` скрывает контент от вспомогательных технологий, но визуально ничего не скрывает.

`aria-hidden="true"` не следует использовать на элементах, которые могут получать фокус. Кроме того, поскольку этот атрибут наследуется дочерними элементами, его не следует добавлять к родительскому или вышестоящему элементу, содержащему фокусируемый элемент.

> [!WARNING]
> Не используйте `aria-hidden="true"` на фокусируемых элементах.

Статус скрытия элемента определяется тем, отображается ли он. Обычно отображением управляет CSS. Например, элемент, у которого свойство `display` установлено в `none` через CSS, не отображается. Элемент считается скрытым, если он или любой из его предков не отображается или имеет значение атрибута `aria-hidden`, установленное в `true`. Обратите внимание, что элемент и его дочерние элементы, у которых указано `aria-hidden="true"`, останутся видимыми, если они также не скрыты с помощью CSS.

Соблюдайте осторожность при использовании `aria-hidden` для скрытия визуально отображаемого контента от вспомогательных технологий. Не следует скрывать видимый контент, кроме тех случаев, когда это улучшает опыт для пользователей вспомогательных технологий, удаляя избыточный или посторонний контент. Удаление видимого контента из API доступности можно считать оправданным только в том случае, если идентичное или эквивалентное значение и функциональность предоставляются вспомогательным технологиям другим способом.

> [!NOTE]
> Принимайте во внимание все виды ограничений возможностей, когда скрываете визуально отображаемый контент от вспомогательных технологий. Не все пользователи вспомогательных технологий имеют нарушения зрения. Если видимый контент не соответствует текстовому содержимому в API доступности, пользовательский опыт для зрячих пользователей будет нарушен.

На первый взгляд, `aria-hidden="true"` и `role="presentation"` с его синонимом `role="none"` могут показаться похожими, но их назначение различается.

*   `aria-hidden="true"` полностью удаляет элемент из API доступности.
*   `role="presentation"` и `role="none"` удаляют семантическое значение элемента, но при этом всё же предоставляют сам элемент и его содержимое вспомогательным технологиям.

`aria-hidden="true"` не нужно добавлять, когда:

*   Присутствует HTML-атрибут [`hidden`](/ru/docs/Web/HTML/Reference/Global_attributes/hidden).
*   Элемент или его предок скрыт с помощью [`display: none`](/ru/docs/Web/CSS/display).
*   Элемент или его предок скрыт с помощью [`visibility: hidden`](/ru/docs/Web/CSS/visibility).

Во всех трёх случаях добавлять атрибут необязательно, потому что элемент уже удалён из дерева доступности. Визуальное скрытие элементов с помощью `display` или `visibility` скрывает контент как с экрана, так и от вспомогательных технологий.

Использование `aria-hidden="false"` не приведёт к повторному предоставлению элемента вспомогательным технологиям, если любой из его родительских элементов имеет `aria-hidden="true"`.

## Пример

Добавление `aria-hidden="true"` к иконке скрывает символ иконки, чтобы он не включался в доступное имя (accessible name).

```html
<button>
  <span class="fa fa-tweet" aria-hidden="true"></span>
  <span class="label"> Твитнуть </span>
</button>
```

У нас есть кнопка с [иконкой Font Awesome](https://fontawesome.com/). Мы скрываем иконку от вспомогательных технологий с помощью `aria-hidden="true"`, поскольку предоставление иконки вспомогательным технологиям может привести к избыточности или, если содержание иконки не совпадает с видимым текстом, к путанице.

## Значения

*   `false`
    *   Элемент предоставляется API доступности так, как если бы он был отображён.
*   `true`
    *   Элемент скрыт от API доступности.
*   `undefined` (по умолчанию)
    *   Состояние скрытия элемента определяется пользовательским агентом на основе того, отображается ли элемент.
