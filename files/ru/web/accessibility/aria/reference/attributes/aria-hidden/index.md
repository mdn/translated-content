Состояние `aria-hidden` указывает, доступен ли элемент для API специальных возможностей.

## Описание

Атрибут `aria-hidden` можно использовать для скрытия неинтерактивного контента от API специальных возможностей.

Добавление `aria-hidden="true"` к элементу удаляет этот элемент и всех его потомков из дерева доступности. Это может улучшить взаимодействие с пользователями вспомогательных технологий, скрывая:

- Чисто декоративный контент, такой как иконки или изображения;
- Дублированный контент, например повторяющийся текст;
- Контент за пределами экрана или свёрнутый контент, такой как меню.

Наличие атрибута `aria-hidden` скрывает контент от вспомогательных технологий, но не скрывает его визуально.

`aria-hidden="true"` не следует использовать на элементах, которые могут получать фокус. Кроме того, поскольку этот атрибут наследуется дочерними элементами, его не следует добавлять к родителю или предку фокусируемого элемента.

> [!WARNING]
> Не используйте `aria-hidden="true"` на элементах, которые могут получать фокус.

Скрытое состояние элемента определяется тем, отображается ли он. Отображение обычно управляется CSS. Например, элемент, свойство `display` которого установлено в `none` через CSS, не отображается. Элемент считается скрытым, если он сам или любой из его предков не отображается либо имеет значение атрибута `aria-hidden`, равное `true`. Обратите внимание: элемент и его потомки, для которых указано `aria-hidden="true"`, всё ещё будут видимы, если только они не скрыты также с помощью CSS.

С осторожностью используйте `aria-hidden` для сокрытия визуально отображаемого контента от вспомогательных технологий. Не следует скрывать видимый контент, если только это не улучшает взаимодействие с пользователями вспомогательных технологий за счёт удаления избыточного или постороннего контента. Только когда идентичное или эквивалентное значение и функциональность предоставляются вспомогательным технологиям, удаление видимого контента из API специальных возможностей может считаться допустимым.

> [!NOTE]
> При сокрытии визуально отображаемого контента от вспомогательных технологий учитывайте все виды инвалидности. Не все пользователи вспомогательных технологий имеют нарушения зрения. Если видимый контент не соответствует текстовому содержимому в API специальных возможностей, это негативно скажется на пользовательском опыте для зрячих пользователей.

На первый взгляд `aria-hidden="true"` и `role="presentation"` (и его синоним `role="none"`) могут показаться похожими, но цель у них разная.

- `aria-hidden="true"` полностью удаляет элемент из API специальных возможностей.
- `role="presentation"` и `role="none"` удаляют семантическое значение элемента, но оставляют сам элемент и его содержимое доступными для вспомогательных технологий.

`aria-hidden="true"` не следует добавлять, если:

- присутствует HTML-атрибут [`hidden`](/ru/docs/Web/HTML/Reference/Global_attributes/hidden);
- элемент или его предок скрыт с помощью [`display: none`](/ru/docs/Web/CSS/Reference/Properties/display);
- элемент или его предок скрыт с помощью [`visibility: hidden`](/ru/docs/Web/CSS/Reference/Properties/visibility).

Во всех трёх случаях атрибут добавлять не нужно, так как элемент уже удалён из дерева доступности. Визуальное скрытие элементов с помощью `display` или `visibility` скрывает содержимое как с экрана, так и от вспомогательных технологий.

Использование `aria-hidden="false"` не сделает элемент снова доступным для вспомогательных технологий, если какой-либо из его родителей имеет `aria-hidden="true"`.

## Пример

Добавление `aria-hidden="true"` к иконке скрывает символ иконки из доступного имени.

```html
<button>
  <span class="fa fa-tweet" aria-hidden="true"></span>
  <span class="label">Твитнуть</span>
</button>
```

У нас есть кнопка с [иконкой Font Awesome](https://fontawesome.com/). Мы скрываем иконку от вспомогательных технологий с помощью `aria-hidden="true"`, поскольку её раскрытие могло бы привести к избыточности или, если содержимое иконки не совпадает с видимым текстом, к путанице.

## Значения

- `false`
  - : Элемент доступен для API специальных возможностей, как если бы он отображался.
- `true`
  - : Элемент скрыт от API специальных возможностей.
- `undefined` (по умолчанию)
  - : Скрытое состояние элемента определяется пользовательским агентом на основе того, отображается ли он.
