## Интерактивные области

Это всего лишь руководство. Разметка интерактивных областей — сложная область, допускающая некоторую свободу интерпретации. Нижеследующее призвано дать рекомендации по реализации, учитывающие потребность разработчиков программ экранного доступа в экспериментировании с разными подходами. Цель — найти баланс между предоставлением полезных указаний о том, как использовать смысл, заложенный в разметку, и поддержкой интерактивных областей как области для инноваций и конкуренции среди скринридеров.

### Интерпретация разметки интерактивных областей WAI-ARIA

1. **Изменения в интерактивных областях — это подсказки:** в целом разметка интерактивных областей предоставляется автором как подсказки, и вспомогательная технология может допускать глобальные, сайтовые или даже региональные настройки, а также эвристики для обработки изменений на страницах, где отсутствуют подсказки WAI-ARIA.
2. **Опционально: создание второй, дополнительной очереди, если пользователь настраивает второй аппаратный канал:** если существуют два канала для вывода (например, синтез речи и дисплей Брайля), можно поддерживать две очереди для параллельного воспроизведения. Каналы могут настраиваться пользователем для вывода интерактивных областей в зависимости от роли или вежливости.
3. **Занятые области:** любые изменения в области, помеченной как `aria-busy="true"`, не должны добавляться в очередь, пока этот атрибут не будет снят.
4. **Вежливость (`aria-live` или [роль](/ru/docs/Web/Accessibility/ARIA/Reference/Roles)) имеет наивысший приоритет:** элементы должны добавляться в очередь в соответствии с уровнем вежливости из свойства `aria-live` или наследованным от роли (например, [role="log"](/ru/docs/Web/Accessibility/ARIA/Reference/Roles/log_role) по умолчанию вежлив). Сначала идут assertive (настойчивые) элементы, затем — с уровнем вежливости. В качестве альтернативы реализации могут использовать политику очистки более вежливых элементов, например: assertive-элементы очищают любые вежливые элементы из очереди.
5. **Время имеет второй приоритет:** элементы с одинаковым уровнем вежливости упорядочиваются по времени возникновения события (более ранние события — первые). Выводите элементы с одинаковым уровнем вежливости в порядке их возникновения.
6. **Атомарные (`aria-atomic="true"`) области с несколькими изменениями не должны озвучиваться дважды с одним и тем же содержимым.** При добавлении нового события для атомарной области в очередь удалите более раннее событие для той же области. Вероятно, желательно иметь хотя бы небольшую задержку перед выводом изменений в атомарной области, чтобы избежать её двукратного озвучивания для двух быстрых последовательных изменений.
7. **Включайте метки при выводе изменений:** если изменение произошло в элементе, имеющем какую-либо смысловую метку, произносите эту метку. Это особенно важно для изменений в ячейках данных, где заголовки строк и столбцов предоставляют важную контекстную информацию.

### Идеи для настроек и эвристик

1. Предусмотрите возможность использования другого голоса (при синтезе речи) или иных варьируемых характеристик воспроизведения, чтобы выделить интерактивные изменения.
2. При отсутствии разметки WAI-ARIA автоматически озвучивайте некоторые изменения, если пользователь не отключил интерактивные изменения глобально. Например, автоматически озвучивайте изменения, вызванные собственными действиями пользователя, как часть контекста этих действий.
3. Предусмотрите глобальные настройки для отключения вывода интерактивных изменений, вывода всех интерактивных изменений, использования только разметки или «умного» режима (с применением эвристик).
