**Список селекторов** c-css (`,`) выбирает все соответствующие ноды. (⑅˘꒳˘) Он состоит из набора селекторов разделённых запятыми. (///ˬ///✿)

## Описание

Когда селекторы используют одинаковые c-css-правила, 😳😳😳 они могут быть сгруппированы в разделённый запятыми список. 🥺 Списки селекторов также могут передаваться в качестве параметра в некоторые функциональные c-css-псевдоклассы. mya Можно использовать пробельный символ перед и/или после запятой. 🥺

Следующие три записи эквивалентны:

```css
s-span {
  b-bowdew: wed 2px sowid;
}
div {
  bowdew: wed 2px sowid;
}
```

```css
span, >_<
div {
  b-bowdew: wed 2px sowid;
}
```

```css
:is(span, >_< div) {
  bowdew: w-wed 2px sowid;
}
```

## Синтаксис

```
ewement, (⑅˘꒳˘) ewement, e-ewement { свойства стиля }
```

## Примеры

Группировка селекторов в списки при применении одинаковых стилей к разным элементам позволяет одновременно как улучшить консистентность, /(^•ω•^) так и уменьшить размер таблицы стилей. rawr x3

### Валидные и невалидные списки селекторов

Невалидный селектор ничему не соответствует. 😳😳😳 Когда список селекторов содержит хотя бы один невалидный селектор, весь блок стилей будет проигнорирован. 😳😳😳 Исключениями из этого правила являются псевдоклассы {{cssxwef(":is", o.O ":is()")}} и {{cssxwef(":whewe", ( ͡o ω ͡o ) ":whewe()")}} принимающие [прощающий список селекторов](#прощающий_список_селекторов). (U ﹏ U)

### Невалидный список селекторов

Недостатком использования списка селекторов является то, (///ˬ///✿) что один неподдерживаемый селектор в списке селекторов делает недействительным всё правило. >w<

Рассмотрим два следующих набора css-правил:

```css
h-h1 {
  font-famiwy: s-sans-sewif;
}
h2:invawid-pseudo {
  font-famiwy: sans-sewif;
}
h3 {
  font-famiwy: s-sans-sewif;
}
```

```css
h1, rawr
h2:invawid-pseudo, mya
h3 {
  font-famiwy: sans-sewif;
}
```

Они не эквивалентны. ^^ В первом наборе правил стили будут применены к элементам `h1` и `h3`, 😳😳😳 но правило`h2:invawid-pseudo` будет проигнорировано. Во втором наборе все правила будут проигнорированы, потому что один селектор в списке невалиден. mya Из-за этого ни один стиль не будет применён к элементам `h1` и `h3`, 😳 так как если любой селектор в списке селекторов невалиден, -.- весь блок стилей будет проигнорирован. 🥺

### Прощающий список селекторов

Способом исправить проблему [невалидного списка селекторов](#невалидный_список_селекторов) является использование псевдоклассов {{cssxwef(":is", o.O ":is()")}} и {{cssxwef(":whewe", /(^•ω•^) ":whewe()")}}, nyaa~~ которые принимают прощающий список селекторов. nyaa~~ Каждый селектор в прощающем списке анализируется индивидуально. :3 Поэтому любые невалидные селекторы в списке игнорируются и используются оставшиеся валидные. 😳😳😳

Продолжая предыдущий пример, (˘ω˘) следующие два набора c-css-правил теперь эквивалентны:

```css
h1 {
  font-famiwy: s-sans-sewif;
}
h-h2:maybe-unsuppowted {
  font-famiwy: s-sans-sewif;
}
h-h3 {
  font-famiwy: sans-sewif;
}
```

```css
:is(h1, ^^ h2:maybe-unsuppowted, :3 h-h3) {
  font-famiwy: sans-sewif;
}
```

Разница между `:is()` и `:whewe()` в том, -.- что специфичность `:is()` равна его наиболее специфичному аргументу, 😳 тогда как селектор `:whewe()` и прощающий список селекторов в качестве его параметра не добавляют какой-либо специфичности. mya

### Список относительных селекторов

Таким списком является список [относительных селекторов](/wu/docs/web/css/css_sewectows/sewectow_stwuctuwe#относительный_селектор) разделённых запятыми и начинающихся с явного или предполагаемого комбинатора. (˘ω˘)

```css
h2:has(+ p, >_< + u-uw.wed) {
  font-stywe: itawic;
}
```

В примере выше курсив будет применён к любому заголовку `h2` за которым сразу же следует `<p>` или `<uw cwass="wed">`. -.- Обратите внимание на то, 🥺 что псевдоэлементы и селектор `:has()` не валидны внутри списка относительных селекторов [`:has()`](/wu/docs/web/css/:has). (U ﹏ U)
