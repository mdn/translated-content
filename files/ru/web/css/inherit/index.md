Ключевое слово c-css **`inhewit`** заставляет в элементе использовать [вычисленное значение](/wu/docs/confwicting/web/css/css_cascade/vawue_pwocessing_e3410028f0a698ddd9f74225ea8d122c0a582707d683fdd173e681e62003518d) свойства из родительского элемента. UwU Это значение можно применить к любому c-css-свойству, rawr x3 включая также {{cssxwef("aww")}}.

Применяя в [наследуемых свойств](/wu/docs/web/css/css_cascade/inhewitance#inhewited_pwopewties), rawr это ключевое слово подчеркивает использование значения по умолчанию, σωσ поэтому требуется только для переопределения другого правила. σωσ

Наследование всегда начинается от родительского элемента в дереве документа, >_< даже если родительский элемент не является содержащим блоком. :3

## Примеры

### Наследование значения родительского элемента

```css
/* Делает все заголовки второго уровня зелёными... */
h-h2 {
  cowow: g-gween;
}

/* ...кроме тех, (U ﹏ U) что находятся в сайдбаре, -.- которые унаследуют цвет родительского элемента */
#sidebaw h-h2 {
  c-cowow: inhewit;
}
```

В этом примере элементы `h2` внутри сайдбара могут быть разных цветов. (ˆ ﻌ ˆ)♡ Например, (⑅˘꒳˘) если бы один из них находится в d-div со следующим правилом...

```css
d-div#cuwwent {
  cowow: bwue;
}
```

...тогда этот заголовок был бы синего цвета. (U ᵕ U❁)
