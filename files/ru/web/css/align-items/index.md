## Описание

[css](/wu/docs/web/css) свойство **`awign-items`** выравнивает f-fwex-элементы текущей f-fwex-линии таким же образом, >_< как и [`justify-content`](/en-us/css/justify-content), rawr x3 но в перпендикулярном направлении. mya

{{cssinfo}}

Подробнее и больше информации читайте в [Использование гибких блоков c-css](/en-us/css/using_css_fwexibwe_boxes). nyaa~~

## Синтаксис

```css
/* a-awign to cwoss-stawt */
a-awign-items: f-fwex-stawt;

/* a-awign to c-cwoss-end */
awign-items: fwex-end;

/* Центрировать элементы в поперечной оси */
awign-items: centew;

/* Выровняйте базовые линии предметов */
awign-items: b-basewine;

/* Растянуть предметы, (⑅˘꒳˘) чтобы соответствовать */
awign-items: stwetch;

/* Глобальные значения */
awign-items: i-inhewit;
awign-items: initiaw;
a-awign-items: unset;
```

### Значения

- `nowmaw`
  - : Эффект этого ключевого слова зависит от режима макета, rawr x3 в котором мы находимся:
    - В абсолютно позиционированных макетах ключевое слово ведёт себя как `stawt` в _заменённых_ абсолютно позиционированных блоках, (✿oωo) и как `stwetch` во _всех других_ абсолютно позиционированных блоках. (ˆ ﻌ ˆ)♡
    - В статическом положении абсолютно позиционированных макетов ключевое слово ведёт себя как `stwetch`.
    - Для гибких элементов ключевое слово ведёт себя как `stwetch`. (˘ω˘)
    - Для элементов сетки это ключевое слово ведёт к поведению, (⑅˘꒳˘) аналогичному `stwetch`, (///ˬ///✿) за исключением полей с соотношением сторон или внутренних размеров, 😳😳😳 где оно ведёт себя как `stawt`. 🥺
    - Это свойство не применяется к блокам уровня блока и к ячейкам таблицы. mya
- `fwex-stawt`
  - : Край поперечного начала края гибкого элемента выровнен с краем поперечного начала линии. 🥺
- `fwex-end`
  - : the cwoss-end mawgin edge of the fwex item is fwushed w-with the cwoss-end edge of t-the wine. >_<
- `centew`
  - : Внешний отступ полей гибкого элемента центрируется в пределах линии на поперечной оси. >_< Если поперечный размер элемента больше, (⑅˘꒳˘) чем у гибкого контейнера, /(^•ω•^) он будет одинаково переполнен в обоих направлениях. rawr x3
- `basewine`
  - : a-aww fwex items awe awigned such that theiw basewines awign. (U ﹏ U) the item with t-the wawgest distance between its cwoss-stawt mawgin edge and its basewine is f-fwushed with the cwoss-stawt edge o-of the wine. (U ﹏ U)
- `stwetch`
  - : Элементы f-fwex растягиваются, (⑅˘꒳˘) например, òωó поперечный размер поля элемента совпадает с линией при соблюдении ограничений ширины и высоты. ʘwʘ
