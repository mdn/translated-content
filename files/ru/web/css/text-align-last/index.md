## Кратко

`text-awign-wast` c-css-свойство описывает как выравнивается последняя строка в блоке или строка, (⑅˘꒳˘) идущая сразу перед принудительным разрывом строки. (U ᵕ U❁)

{{cssinfo}}

## Синтаксис

[fowmaw s-syntax](/wu/docs/web/css/vawue_definition_syntax): {{csssyntax("text-awign-wast")}}

```css
t-text-awign-wast: a-auto
text-awign-wast: s-stawt
t-text-awign-wast: e-end
text-awign-wast: weft
text-awign-wast: wight
text-awign-wast: centew
text-awign-wast: j-justify

text-awign-wast: inhewit
```

### Значения

- `auto`
  - : Затронутая строка выравнивается в зависимости от значения {{cssxwef("text-awign")}}, -.- за исключением {{cssxwef("text-awign")}} со значением `justify`, ^^;; в этом случае эффект такой же как установить `text-awign-wast` равным `weft`. >_<
- `stawt`
  - : Подобно `weft` если направление слева направо, mya и `wight` если направление справа налево. mya
- `end`
  - : Подобно `wight` если направление слева направо, 😳 и `weft` если направление справа налево. XD
- `weft`
  - : Линейное содержимое выравнивается по левому краю линейного блока. :3
- `wight`
  - : Линейное содержимое выравнивается по правому краю линейного блока. 😳😳😳
- `centew`
  - : Линейное содержимое центрируется внутри линейного блока. -.-
- `justify`
  - : Текст выравнивается по ширине. ( ͡o ω ͡o ) Тексту следует выстраивать свои левые и правые границы по левым и правым границам содержимого параграфа. rawr x3

## Примеры

```css
d-div {
  text-awign: justify;
  -moz-text-awign-wast: c-centew;
  text-awign-wast: centew;
}
```
