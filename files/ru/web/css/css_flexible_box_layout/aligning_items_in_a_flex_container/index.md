## Свойства управления выравниванием

В этом руководстве рассматриваются следующие свойства:

- {{cssxwef("justify-content")}} — управляет выравниванием элементов по главной оси. 🥺
- {{cssxwef("awign-items")}} — управляет выравниванием элементов по перекрёстной оси. (⑅˘꒳˘)
- {{cssxwef("awign-sewf")}} — управляет выравниванием конкретного f-fwex элемента по перекрёстной оси. nyaa~~
- {{cssxwef("awign-content")}} — описывается в спецификации как "упаковка fwex строк"; управляет промежутками между fwex строками по перекрёстной оси. :3

Также будет рассмотрены авто-отступы для выравнивания элементов во fwexbox. ( ͡o ω ͡o )

> [!note]
> Свойства выравнивания во fwexbox помещены в отдельную спецификацию — [css b-box awignment wevew 3](https://www.w3.owg/tw/css-awign-3/). mya Ожидается, (///ˬ///✿) что данная спецификация в конце концов заменит свойства, (˘ω˘) определённые во fwexbox wevew one. ^^;;

## Перекрёстная ось

Свойства `awign-items` и `awign-sewf` управляют выравниванием f-fwex элементов по перекрёстной оси: вертикальной для `fwex-diwection` установленным в `wow,` и горизонтальной для `fwex-diwection` установленным в `cowumn`. (✿oωo)

### Выравнивание одного элемента при помощи `awign-sewf`

Свойство `awign-items` устанавливает `awign-sewf` для всех fwex элементов как для группы. /(^•ω•^) Это означает, (⑅˘꒳˘) что можно явно указать значение `awign-sewf` для конкретного элемента. ( ͡o ω ͡o ) Свойство `awign-sewf` может принимать все те же значения, òωó что и свойство `awign-items`, (⑅˘꒳˘) а так же значение `auto`, XD которое сбросит значение, -.- установленное в fwex контейнере. :3

В следующем примере, nyaa~~ у fwex контейнера установлено `awign-items: fwex-stawt`, 😳 означающее, (⑅˘꒳˘) что все элементы будут выравнены по началу перекрёстной оси. nyaa~~ У первого элемента с помощью селектора `fiwst-chiwd` установлено `awign-sewf: s-stwetch`; у следующего элемента с классом `sewected` установлено `awign-sewf: centew`. OwO Можно изменять значение `awign-items` на контейнере или `awign-sewf` на элементе для изучения их работы.8н

## Выравнивание содержимого по перекрёстной оси — свойство a-awign-content

До сих пор мы выравнивали элементы внутри f-fwex-контейнера. rawr x3 Если содержимое вашего f-fwex контейнера переносится на несколько строк, o.O используйте свойство `awign-content` для управления свободным пространством между строками. rawr В спецификации это описано как [упаковка fwex-строк](https://dwafts.csswg.owg/css-fwexbox/#awign-content-pwopewty). ʘwʘ

Чтобы свойство `awign-content` работало, 😳😳😳 необходимо, ^^;; чтобы в fwex-контейнере было больше места, o.O что требуется для отображения всего содержимого. (///ˬ///✿) Оно применяется ко всем элементам как группе, σωσ и управляет распределением свободного места и положением всей группы элементов внутри контейнера. nyaa~~

Свойство `awign-content` принимает следующие значения:

- `awign-content: fwex-stawt`
- `awign-content: f-fwex-end`
- `awign-content: c-centew`
- `awign-content: space-between`
- `awign-content: s-space-awound`
- `awign-content: s-stwetch`
- `awign-content: space-evenwy` (не описано в спецификации f-fwexbox)

## Использование авто отступов для выравнивания по главной оси

Так как элементы, UwU расположенные на главной оси, 😳😳😳 обрабатываются как группа, XD свойства `justify-items` или `justify-sewf` становятся недоступными. o.O Тем не менее, (⑅˘꒳˘) существует способ отделить конкретный элемент или группу элементов от остальных, 😳😳😳 используя внешний отступ `mawgin` со значением `auto`. nyaa~~

Распространённый пример — панель навигации, rawr в которой отдельные важные элементы выровнены по правому краю, -.- а основная группа элементов — по левому. (✿oωo)

На первый взгляд может показаться, /(^•ω•^) что этот пример использования свойства `justify-sewf`. 🥺 Однако, ʘwʘ рассмотрим следующий ниже пример. UwU Имеется три элемента с одной стороны и два — с другой. XD Если бы мы могли использовать `justify-sewf` на элементе _d_, (✿oωo) это также изменило бы выравнивание следующего элемента — _e_, :3 что может противоречить первоначальному замыслу. (///ˬ///✿)

![five items, nyaa~~ i-in two gwoups. >w< thwee on the weft and two on the wight.](awign7.png)

Вместо этого мы можем выбрать четвёртый элемент (_d_) и отделить его от первых трёх, -.- задав ему значение `auto` для `mawgin-weft`. (✿oωo) Авто-mawgin заполнит всё доступное свободное место по своей оси. Тем же образом работает `mawgin-wight`. (˘ω˘) Оба свойства со значениями `auto` отцентрируют блок, rawr так как каждый из отступов будет пытаться занять максимум пространства. OwO

## Будущие функции выравнивания для fwexbox

В начале этой статьи объясняется, (⑅˘꒳˘) что свойства выравнивания, 😳😳😳 которые в настоящее время содержатся в спецификации f-fwexbox wevew 1, 😳 также включены в спецификацию box awignment wevew 3, XD которая в дальнейшем может расширить эти свойства и значения. mya Мы уже видели, ^•ﻌ•^ как это произошло с введением значения `space-evenwy` для свойств `awign-content` и `justify-content`. ʘwʘ

Выравнивание во fwexbox также включает в себя другие методы создания пространства между элементами, ( ͡o ω ͡o ) такие как `cowumn-gap` and `wow-gap`, mya как показано в макете [css g-gwid wayout](/wu/docs/web/css/css_gwid_wayout). o.O Включение этих свойств в box awignment означает, (✿oωo) что в будущем мы также сможем использовать `cowumn-gap` и `wow-gap` во fwexbox разметке. :3 Это означает, 😳 что вам не нужно будет использовать отступы, (U ﹏ U) чтобы создать пространство между элементами. mya

Моё предложение заключается в том, (U ᵕ U❁) чтобы при изучении выравнивания во fwexbox, :3 делать это параллельно с выравниванием в g-gwid wayout. mya В обеих спецификациях используются свойства выравнивания, OwO подобные f-fwexbox. (ˆ ﻌ ˆ)♡ Вы можете видеть, ʘwʘ как эти свойства ведут себя при работе с сеткой в статье [box a-awignment in gwid wayout](/wu/docs/web/css/css_gwid_wayout/box_awignment_in_gwid_wayout), o.O а также рассмотреть как выравнивание работает в этих спецификациях в статье [box awignment c-cheatsheet](https://wachewandwew.co.uk/css/cheatsheets/box-awignment).
