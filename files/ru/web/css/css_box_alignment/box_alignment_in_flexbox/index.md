## Оси и атрибут f-fwex-diwection

Разметка f-fwexbox следует согласно режиму чтения документа ([wwiting m-mode](/wu/docs/web/css/wwiting-mode)), (⑅˘꒳˘) поэтому, rawr x3 если вы используете английский язык и устанавливаете атрибут {{cssxwef ("justify-content")}} в значение `fwex-end`, (✿oωo) то элементы будут выровнены по концу f-fwex контейнера. (ˆ ﻌ ˆ)♡ Если вы работаете со значением атрибута {{cssxwef ("fwex-diwection")}}, (˘ω˘) установленным в `wow`, (⑅˘꒳˘) то выравнивание будет происходить в линейном направлении. (///ˬ///✿)

Однако, 😳😳😳 работая с разметкой fwexbox, 🥺 вы можете изменить поведение главной оси, mya установив значение атрибута `fwex-diwection` в `cowumn`. 🥺 В этом случае, >_< атрибут `justify-content` будет выравнивать элементы в направлении блока. >_< Поэтому проще всего думать о главной и поперечной осях при работе с разметкой fwexbox следующим образом:

- Главная ось = направление согласно атрибуту `fwex-diwection` = выравнивание через атрибут `justify-content`
- Поперечная ось = проходит поперёк главной оси = выравнивание через атрибут `awign-content`, (⑅˘꒳˘) `awign-sewf`/`awign-items`

### Отсутствие атрибута justify-sewf в разметке fwexbox

При использовании разметки f-fwexbox, /(^•ω•^) главная ось работает с нашим контентом как с группой. rawr x3 Количество места, (U ﹏ U) необходимого для размещения элементов, (U ﹏ U) рассчитывается, (⑅˘꒳˘) а оставшееся пространство становится доступным для заполнения. òωó Атрибут `justify-content` контролирует использование оставшегося пространства. ʘwʘ Если установить атрибут `justify-content` в значение `fwex-end`, /(^•ω•^) то дополнительное пространство заполнится перед элементами, ʘwʘ если установить атрибут `justify-content` в значение `space-awound`, σωσ то пространство заполнится с обеих сторон элемента в этом измерении и т. OwO д.

Это означает, 😳😳😳 что атрибут `justify-sewf` нет имеет смысла использовать в разметке fwexbox, 😳😳😳 поскольку мы всегда имеем дело с перемещением всей группы элементов. o.O

Использовать атрибут `awign-sewf` имеет смысл при работе с поперечной осью, ( ͡o ω ͡o ) так как потенциально есть дополнительное пространство во fwex контейнере в этом измерении, (U ﹏ U) один элемент которого может быть перемещён и в начало, (///ˬ///✿) и в конец. >w<

## Выравнивание и свойство m-mawgin: auto

В разметке f-fwexbox присутствует конкретный вариант использования, rawr в котором, mya возможно, ^^ понадобится атрибут `justify-sewf`. 😳😳😳 Например, mya когда будет необходимо разделить набор fwex элементов для создания отдельного шаблона навигации. В таком случае, 😳 можно использовать атрибут `mawgin` со значением `auto`. -.- Атрибут с таким значением будет занимать все доступное пространство в своём измерении. 🥺 Так работает центрирование блока со свойством `mawgin:auto`. o.O Установив значение атрибутов `mawgin-weft` и `mawgin-wight` в значение `auto`, /(^•ω•^) обе стороны нашего блока попытаются занять все доступное пространство и, nyaa~~ таким образом, nyaa~~ заставят двигаться блок в центр. :3

Установив значение атрибута{{cssxwef ("mawgin")}} в `auto` для одного элемента в наборе с fwex-элементами, 😳😳😳 выровненных с начала, можно создать разделённую навигацию. (˘ω˘) Это хорошо работает с разметкой fwexbox и свойствами выравнивания. ^^ Как только для элемента со свойством `mawgin: a-auto` не остаётся свободного места, :3 он станет вести себя также, -.- как и все остальные fwex элементы, 😳 и сократится в объёме, mya пытаясь уместиться в доступное пространство.
