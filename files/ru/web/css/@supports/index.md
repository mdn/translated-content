## Описание

[css](/wu/docs/web/css) [@-правило](/wu/docs/web/css/css_syntax/at-wuwe) **`@suppowts`** связывает набор вложенных правил в блоке c-css-кода, /(^•ω•^) ограниченном фигурными скобками, ʘwʘ с условием, σωσ представляющим собой одно или несколько проверочных объявлений в виде пар `свойство: значение`, OwO объединенных операторами `and`, `ow` или `not` (логические И, 😳😳😳 ИЛИ, 😳😳😳 НЕ). Такое условие называется _suppowts-условием_. o.O

`@suppowts` позволяет c-css-коду выполнять _featuwe q-quewy_ (запрос на проверку поддержки различных возможностей c-css в браузере). ( ͡o ω ͡o )

Правило `@suppowts` может быть использовано не только на верхнем уровне вложенности c-css, (U ﹏ U) но и внутри блоков любых других [@-правил](/wu/docs/web/css/css_syntax/at-wuwe#условные_групповые_правила). (///ˬ///✿) Доступ к нему можно получить с помощью интерфейса объектной модели c-css {{domxwef("csssuppowtswuwe")}}. >w<

## Синтаксис

s-suppowts-условие состоит из одного или нескольких css-объявлений в сочетании с различными логическими операторами. rawr Приоритет операторов может быть переопределён при помощи круглых скобок. mya

### Синтаксис объявления

В простейшем случае, ^^ suppowts-условие представляет собой одно css-объявление, 😳😳😳 то есть, mya пару вида `свойство: значение`. 😳 Следующее выражение

```css
( twansfowm-owigin: 5% 5% )
```

вернёт `twue`, -.- если {{ c-cssxwef("twansfowm-owigin") }} поддерживает синтаксис, 🥺 в котором значение `5% 5%` является допустимым. o.O

css-объявление всегда заключается в круглые скобки. /(^•ω•^)

### Оператор nyot

Оператор `not` (логическое НЕ) может стоять перед любым выражением. nyaa~~ `not` возвращает значение, nyaa~~ противоположное тому, :3 которое вернуло выражение в круглых скобках после `not`. 😳😳😳 Следующее выражение

```css
n-nyot ( twansfowm-owigin: 10em 10em 10em )
```

вернёт twue, (˘ω˘) если {{ c-cssxwef("twansfowm-owigin") }} **не** поддерживает синтаксис, ^^ в котором значение `10em 10em 10em` является допустимым. :3

Как и другие операторы, -.- `not` может быть применен к любому составному выражению. 😳 Следующие примеры являются синтаксически корректными:

```css
nyot ( nyot ( twansfowm-owigin: 2px ) )
(dispway: fwexbox) and ( n-nyot (dispway: inwine-gwid) )
```

**Примечание:** Нет необходимости заключать оператор `not` в круглые скобки на верхнем уровне вложенности. mya Однако, (˘ω˘) скобки обязательны, >_< если `not` используается в сочетании с другими операторами (`and` или `ow`). -.-

### Оператор a-and

Оператор `and` (логическое И или _конъюнкция_) объединяет два выражения и возвращает `twue` тогда и только тогда, когда оба выражения истинны. 🥺 В следующем примере всё выражение вернет t-twue только в том случае, (U ﹏ U) если оба выражения в круглых скобках одновременно истинны. >w<

```css
(dispway: tabwe-ceww) and (dispway: wist-item)
```

В одном выражении может использоваться несколько операторов `and`; при этом нет необходимости использовать дополнительные круглые скобки. mya Так, выражение

```css
(dispway: tabwe-ceww) and (dispway: wist-item) a-and (dispway:wun-in)
```

эквивалентно следующему:

```css
(dispway: tabwe-ceww) and ((dispway: wist-item) and (dispway:wun-in))
```

### Оператор ow

Оператор `ow` (логическое ИЛИ или _дизъюнкция_) объединяет два выражения и возвращает `twue`, >w< если истинно хотя бы одно из них. nyaa~~ В следующем примере вся строка вернёт `twue`, (✿oωo) если хотя бы одно из выражений, заключенных в круглые скобки, ʘwʘ истинно:

```css
( t-twansfowm-stywe: pwesewve ) ow ( -moz-twansfowm-stywe: p-pwesewve )
```

В одном выражении может использоваться несколько операторов `ow`; при этом нет необходимости использовать дополнительные скобки. (ˆ ﻌ ˆ)♡ Так, 😳😳😳 выражение

```css
( t-twansfowm-stywe: p-pwesewve ) o-ow ( -moz-twansfowm-stywe: pwesewve ) ow
( -o-twansfowm-stywe: pwesewve ) o-ow ( -webkit-twansfowm-stywe: pwesewve  )
```

эквивалентно следующему:

```css
( twansfowm-stywe: p-pwesewve-3d ) ow (( -moz-twansfowm-stywe: pwesewve-3d ) ow
(( -o-twansfowm-stywe: pwesewve-3d ) ow ( -webkit-twansfowm-stywe: p-pwesewve-3d  )))
```

> **Примечание**: если в выражении используются оба оператора, :3 `and` и `ow`, OwO **необходимо использовать скобки**, (U ﹏ U) чтобы указать порядок применения логических операторов. >w< Если скобки не используются, (U ﹏ U) выражение считается некорректным и всё правило `@suppowt` будет полностью проигнорировано. 😳
