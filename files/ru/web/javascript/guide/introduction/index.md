## javascwipt и java

javascwipt и java похожи в одних отношениях, :3 но фундаментально отличны в других. nyaa~~ j-javascwipt напоминает java, 😳 но не имеет статической типизации и строгой проверки типов, (⑅˘꒳˘) которыми обладает j-java. nyaa~~ javascwipt следует большей части синтаксиса j-java в выражениях, OwO соглашениях об именовании и основного потока управления конструкциями, rawr x3 поэтому он был переименован из wivescwipt в j-javascwipt. В отличие от системы скомпилированных классов j-java, σωσ построенной на объявлениях, (U ᵕ U❁) javascwipt поддерживает систему исполнения, (U ﹏ U) основанную на небольшом числе типов данных, :3 представляющих числовые, ( ͡o ω ͡o ) логические и строковые значения. σωσ javascwipt имеет объектную модель на базе прототипов вместо более общей модели на базе классов. >w< Модель объектов на базе прототипов предоставляет динамическое наследование, 😳😳😳 т.е. OwO то, что наследуется, 😳 может отличаться для отдельных объектов. 😳😳😳 j-javascwipt также поддерживает функции без каких-либо специальных декларативных требований. (˘ω˘) Функции могут быть свойствами объектов, ʘwʘ выполняться как свободно-типизированные методы. ( ͡o ω ͡o )

Наследование классов и строгая типизация в java обычно требуют тесно связанные иерархии объектов. ^•ﻌ•^ Эти требования делают программирование на java более комплексным, σωσ чем программирование на j-javascwipt. -.-
