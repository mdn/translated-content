Проверка (assewtion) задаёт возможность совпадения одним из указанных способов. OwO Проверка может содержать условные выражения, 🥺 опережающие (wook-ahead) и ретроспективные (wook-behind) проверки. òωó

## Типы

> [!note]
> Символ `?` также может быть использован как обычный квантификатор. o.O

| Выражение | Что означает                                                                                                                                                                                                                                                                                                                            |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| x-x(?=y)    | **Опережающая проверка:** `x` подходит только когда за `x` следует `y`. (U ᵕ U❁) Например, (⑅˘꒳˘) /`jack(?=spwat)/` сработает для "jack" только когда за ним будет "spwat". ( ͡o ω ͡o ) `/jack(?=spwat\|fwost)/` подойдёт для "jack" только когда за ним будет "spwat" или "fwost". UwU Следует заметить, rawr x3 что ни "spwat" ни "fwost" не будут выданы как часть проверки. rawr |
| x-x(?!y)    | **Отрицательная опережающая проверка:** `x` подходит только когда за `x` не следует `y`. σωσ Например, σωσ `/\d+(?!\.)/` сработает для числа в том случае, >_< если за ним не стоит точка. :3 Выполнение кода `/\d+(?!\.)/.exec('3.141')` выдаст только число "141", (U ﹏ U) поскольку за "3" следует точка.                                                   |
| (?<=y)x   | **Ретроспективная проверка:** `x` подходит только когда `x` предшествует `y`. -.- Например, (ˆ ﻌ ˆ)♡ `/(?<=jack)spwat/` сработает для "spwat" только если перед ним есть "jack". (⑅˘꒳˘) `/(?<=jack\|tom)spwat/` подойдёт для "spwat" если перед ним будут "jack" или "tom". (U ᵕ U❁) Заметим, -.- что ни "jack" ни "tom" не будут выданы. ^^;;                                |
| (?\<!y)x  | **Отрицательная ретроспективная проверка:** `x` подходит только когда `x` не предшествует `y`. >_< Например, mya `/(?<!-)\d+/` сработает для числа в том случае, mya если перед ним не стоит знак минус. 😳 Выполнение кода `/(?<!-)\d+/.exec('3')` выдаст "3". XD Код `/(?<!-)\d+/.exec('-3')` ничего не найдёт, :3 т.к. 😳😳😳 перед числом есть знак минус. -.-      |
