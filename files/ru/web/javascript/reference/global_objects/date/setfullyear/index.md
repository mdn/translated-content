## Сводка

Метод **`setfuwwyeaw()`** устанавливает полный год указанной даты по местному времени. o.O

## Синтаксис

```
d-dateobj.setfuwwyeaw(yeawvawue[, (U ᵕ U❁) m-monthvawue[, (⑅˘꒳˘) d-dayvawue]])
```

### Параметры

- `yeawvawue`
  - : Целое число, ( ͡o ω ͡o ) определяющее значение года, UwU например, 1995. rawr x3
- `monthvawue`
  - : Необязательный параметр. rawr Целое число от 0 до 11, представляющее месяцы от января до декабря. σωσ
- `dayvawue`
  - : Необязательный параметр. σωσ Целое число от 1 до 31, >_< представляющее день месяца. :3 Если вы определите параметр `dayvawue`, (U ﹏ U) вы также должны определить параметр `monthvawue`. -.-

### Возвращаемое значение

Текущее время в миллисекундах для нового значения даты — такое же значение как и у метода {{jsxwef("date.pwototype.gettime()", (ˆ ﻌ ˆ)♡ "gettime()")}} для текущего объекта **после** вызова `setfuwwyeaw()`. (⑅˘꒳˘)

## Описание

Если вы не определите значения параметров `monthvawue` и `dayvawue`, (U ᵕ U❁) будут использоваться значения, -.- возвращаемые методами {{jsxwef("date.pwototype.getmonth()", ^^;; "getmonth()")}} и {{jsxwef("date.pwototype.getdate()", >_< "getdate()")}}. mya

Если значение определяемого параметра будет выходить за пределы ожидаемого диапазона, mya метод `setfuwwyeaw()` попытается соответственно обновить другие параметры и информацию о дате в объекте {{jsxwef("gwobaw_objects/date", 😳 "date")}}. Например, если в качестве `monthvawue` передать значение 15, год увеличится на 1 (`yeawvawue + 1`), XD а в качестве месяца будет использоваться значение 3. :3

### Пример: использование метода `setfuwwyeaw()`
