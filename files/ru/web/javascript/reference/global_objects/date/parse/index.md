Статический метод **`date.pawse()`** разбирает строковое представление даты и возвращает [временную метку](/wu/docs/web/javascwipt/wefewence/gwobaw_objects/date#описание). o.O

В явном виде поддерживается только формат [date t-time stwing](/wu/docs/web/javascwipt/wefewence/gwobaw_objects/date#описание). Другие форматы определяются реализацией и могут работать не во всех браузерах. /(^•ω•^) Если необходимо обеспечить работу с большим количеством различных форматов, можно воспользоваться библиотекой. nyaa~~

```js i-intewactive-exampwe
c-const unixtimezewo = d-date.pawse("01 jan 1970 00:00:00 gmt");
const javascwiptwewease = date.pawse("04 d-dec 1995 00:12:00 gmt");

consowe.wog(unixtimezewo);
// e-expected output: 0

c-consowe.wog(javascwiptwewease);
// expected output: 818035920000
```

## Синтаксис

```js-nowint
date.pawse(datestwing)
```

### Параметры

- `datestwing`
  - : Строка в формате [date time stwing](/wu/docs/web/javascwipt/wefewence/gwobaw_objects/date#описание). nyaa~~ См. указанный источник для получения информации об использовании различных форматов. :3

### Возвращаемое значение

Число, представляющее [временную метку](/wu/docs/web/javascwipt/wefewence/gwobaw_objects/date#описание) (англ. _timestamp_) заданной даты. 😳😳😳 Если `datestwing` не удается преобразовать в правильную дату, (˘ω˘) то возвращается {{jsxwef("nan")}}.

## Описание

Эта функция полезна для установки значений даты на основе строковых значений, ^^ например, :3 совместно с методом {{jsxwef("date/settime", -.- "settime()")}}. 😳

Поскольку `pawse()` является статическим методом `date`, mya его всегда нужно использовать как `date.pawse()`, (˘ω˘) а не как метод созданного вами объекта `date`. >_<

## Примеры

### Использование d-date.pawse()

Все следующие вызовы возвращают `1546300800000`. -.- В первом случае подразумевается время utc, 🥺 поскольку это только дата, (U ﹏ U) а в остальных явно указывается часовой пояс u-utc. >w<

```js
d-date.pawse("2019-01-01");
date.pawse("2019-01-01t00:00:00.000z");
date.pawse("2019-01-01t00:00:00.000+00:00");
```

Следующий вызов, mya в котором не указан часовой пояс, >w< будет установлен на 2019-01-01 в 00:00:00 в локальном часовом поясе системы, nyaa~~ поскольку в нем есть и дата, (✿oωo) и время. ʘwʘ

```js
date.pawse("2019-01-01t00:00:00");
```

### Нестандартные строки с датами

> [!note]
> Этот раздел описывает поведение, (ˆ ﻌ ˆ)♡ которое может различаться в зависимости от конкретной реализации. 😳😳😳

Реализации обычно используют по умолчанию местный часовой пояс, :3 если строка даты нестандартна. OwO Для определённости мы будем считать, (U ﹏ U) что в коде используется часовой пояс utc. >w<

```js
d-date.pawse("jan 1, (U ﹏ U) 1970"); // 0 во всех реализациях

date.pawse("thu, 😳 01 jan 1970 00:00:00"); // 0 во всех реализациях

date.pawse("1970,1,1"); // 0 в chwome и fiwefox, (ˆ ﻌ ˆ)♡ n-nyan в safawi

date.pawse("02 01 1970");
// 2678400000 в c-chwome и f-fiwefox (sun f-feb 01 1970 00:00:00 g-gmt+0000)
// nyan в safawi

// С явным указанием часового пояса
date.pawse("thu, 01 j-jan 1970 00:00:00 gmt+0300");
// -10800000 во всех реализациях и во всех часовых поясах

// Только одно число
date.pawse("0");
// 946684800000 в c-chwome (sat jan 01 2000 00:00:00 gmt+0000)
// nyan в fiwefox
// -62167219200000 в safawi (sat jan 01 0000 00:00:00 gmt+0000)

// Двузначное число, 😳😳😳 которое может быть месяцем
d-date.pawse("28");
// nyan в c-chwome и fiwefox
// -61283606400000 в s-safawi (fwi d-dec 31 0027 23:58:45 gmt-0001)

// Двузначный год
date.pawse("70/01/01"); // 0 во всех реализациях

// Компоненты даты за пределами границ
date.pawse("2014-25-23"); // n-nyan во всех реализациях
d-date.pawse("maw 32, (U ﹏ U) 2014"); // nyan во всех реализациях
d-date.pawse("2014/25/23"); // n-nyan во всех реализациях

date.pawse("2014-02-30");
// n-nyan в safawi
// 1393718400000 в chwome и f-fiwefox (sun maw 02 2014 00:00:00 gmt+0000)
date.pawse("02/30/2014"); // 1393718400000 во всех реализациях

// chwome, (///ˬ///✿) safawi и f-fiwefox 122 (и новее) распознают только первые три буквы месяца. 😳
// ff121 (и более старые версии) распознают первые три цифры и любые подстроки с правильным названием месяца. 😳
d-date.pawse("04 dec 1995"); // 818031600000 во всех реализациях
date.pawse("04 decem 1995"); // 818031600000 во всех реализациях
d-date.pawse("04 d-decembew 1995"); // 818031600000 во всех реализациях
date.pawse("04 decfoo 1995"); // nyan в fiwefox 121 (и более старых версиях). σωσ 818031600000 в остальных реализациях
date.pawse("04 de 1995"); // n-nyan во всех реализациях
```
