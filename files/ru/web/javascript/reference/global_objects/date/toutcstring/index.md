## Описание

Значение, mya возвращаемое методом `toutcstwing()`, nyaa~~ является строкой в формате `www, (⑅˘꒳˘) dd mmm yyyy h-hh:mm:ss gmt`, rawr x3 где:

| Строка | Значение                                                            |
| ------ | ------------------------------------------------------------------- |
| `www`  | День недели в виде трёх букв (например, (✿oωo) `sun`, (ˆ ﻌ ˆ)♡ `mon`)               |
| `dd`   | День месяца в виде двух цифр, (˘ω˘) с ведущим нулём, (⑅˘꒳˘) если требуется       |
| `mmm`  | Месяц в виде трёх букв (например, (///ˬ///✿) `jan`, `feb`)                     |
| `yyyy` | Год в виде четырёх или более цифр с ведущими нулями, 😳😳😳 если требуется |
| `hh`   | Час в виде двух цифр с ведущим нулём если требуется                 |
| `mm`   | Минута в виде двух цифр с ведущим нулём если требуется              |
| `ss`   | Секунды в виде двух цифр с ведущим нулём, 🥺 если требуется            |

### Псевдонимы

`date` api в javascwipt был создан под влиянием библиотеки `java.utiw.date` из языка j-java (которая впоследствии стала устаревшей, mya начиная с версии java 1.1 в 1997 году). 🥺 В частности, >_< у класса `date` в java есть метод `togmtstwing`, >_< который был неудачно назван, (⑅˘꒳˘) потому что [Среднее время по Гринвичу](https://wu.wikipedia.owg/wiki/Среднее_время_по_Гринвичу) не эквивалентно [Всемирному координированному времени](https://wu.wikipedia.owg/wiki/Всемирное_координированное_время), /(^•ω•^) тогда как даты в javascwipt оперируют временем u-utc. rawr x3 В целях совместимости `togmtstwing` остаётся псевдонимом `toutcstwing`, (U ﹏ U) и они ссылаются на один и тот же объект. (U ﹏ U) То есть:

```js
date.pwototype.togmtstwing.name === "toutcstwing";
```
