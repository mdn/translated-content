> [!NOTE]
> В некоторых версиях некоторых браузеров этот метод был реализован как метод `Array.prototype.groupToMap()`. Из-за проблем с веб-совместимостью теперь он реализован как статический метод. Подробности смотрите в [таблице совместимости с браузерами](#совместимость_с_браузерами).

Статический метод **`Map.groupBy()`** группирует элементы переданного итерируемого объекта, используя значения, возвращаемые переданной функцией обратного вызова. Итоговый {{jsxref("Map")}} использует уникальные значения, полученные от этой функции, в качестве ключей, по которым можно получить массив элементов каждой группы.

Метод полезен прежде всего при группировке элементов, связанных с некоторым объектом, — особенно если этот объект может меняться со временем. Если объект неизменяем, его можно представить строкой и группировать элементы с помощью {{jsxref("Object.groupBy()")}}.

{{InteractiveExample("JavaScript Demo: Map.groupBy()", "taller")}}

```js interactive-example
const inventory = [
  { name: "asparagus", type: "vegetables", quantity: 9 },
  { name: "bananas", type: "fruit", quantity: 5 },
  { name: "goat", type: "meat", quantity: 23 },
  { name: "cherries", type: "fruit", quantity: 12 },
  { name: "fish", type: "meat", quantity: 22 },
];

const restock = { restock: true };
const sufficient = { restock: false };
const result = Map.groupBy(inventory, ({ quantity }) =>
  quantity < 6 ? restock : sufficient,
);
console.log(result.get(restock));
// [{ name: "bananas", type: "fruit", quantity: 5 }]
```

## Синтаксис

```js-nolint
Map.groupBy(items, callbackFn)
```

### Параметры

- `items`
  - : [Итерируемый объект](/ru/docs/Web/JavaScript/Reference/Iteration_protocols#протокол_итерируемый) (например, {{jsxref("Array")}}), элементы которого будут сгруппированы.
- `callbackFn`
  - : Функция, которая будет вызвана для каждого элемента итерируемого объекта. Она должна возвращать значение ({{Glossary("object", "объект")}} или {{Glossary("primitive", "примитив")}}), указывающее группу текущего элемента. Функция вызывается со следующими аргументами:
    - `element`
      - : Текущий обрабатываемый элемент.
    - `index`
      - : Индекс текущего обрабатываемого элемента.

### Возвращаемое значение

Объект {{jsxref("Map")}} с ключами для каждой группы, каждому из которых присвоен массив с элементами соответствующей группы.

## Описание

`Map.groupBy()` вызывает переданную функцию `callbackFn` один раз для каждого элемента итерируемого объекта. Функция обратного вызова должна возвращать значение, указывающее группу соответствующего элемента. Значения, возвращаемые `callbackFn`, используются как ключи {{jsxref("Map")}}, возвращаемого `Map.groupBy()`. Каждому ключу соответствует массив, содержащий все элементы, для которых функция обратного вызова вернула одно и то же значение.

Элементы в возвращаемом {{jsxref("Map")}} и в исходном итерируемом объекте — одни и те же (не {{Glossary("deep copy", "глубокие копии")}}). Изменение внутренней структуры элементов отразится и на исходном итерируемом объекте, и на возвращаемом {{jsxref("Map")}}.

Этот метод полезен, когда нужно сгруппировать информацию, связанную с конкретным объектом, который со временем может измениться. Даже если объект будет изменён, он продолжит работать как ключ возвращаемого `Map`. Если же вместо этого создать строковое представление объекта и использовать его как ключ группировки в {{jsxref("Object.groupBy()")}}, то при изменении объекта придётся поддерживать соответствие между исходным объектом и его строковым представлением.

> [!NOTE]
> Чтобы получить доступ к группам в возвращаемом `Map`, нужно использовать тот же самый объект, который изначально был использован в качестве ключа `Map` (хотя его свойства можно изменять). Использовать другой объект, у которого просто совпадают имя и свойства, нельзя.

`Map.groupBy` не читает значение `this`. Его можно вызвать на любом объекте, и будет возвращён новый экземпляр {{jsxref("Map")}}.

## Примеры

### Использование Map.groupBy()

Сначала определим массив объектов, представляющих запасы различных продуктов. У каждого продукта есть тип (`type`) и количество (`quantity`).

```js
const inventory = [
  { name: "asparagus", type: "vegetables", quantity: 9 },
  { name: "bananas", type: "fruit", quantity: 5 },
  { name: "goat", type: "meat", quantity: 23 },
  { name: "cherries", type: "fruit", quantity: 12 },
  { name: "fish", type: "meat", quantity: 22 },
];
```

Код ниже использует `Map.groupBy()` со стрелочной функцией, которая возвращает ключи-объекты с именами `restock` или `sufficient` в зависимости от того, выполняется ли для элемента условие `quantity < 6`. Возвращаемый объект `result` — это `Map`, поэтому для получения массива нужно вызвать `get()` с соответствующим ключом.

```js
const restock = { restock: true };
const sufficient = { restock: false };
const result = Map.groupBy(inventory, ({ quantity }) =>
  quantity < 6 ? restock : sufficient,
);
console.log(result.get(restock));
// [{ name: "bananas", type: "fruit", quantity: 5 }]
```

Обратите внимание, что аргумент функции `{ quantity }` — это простейший пример [синтаксиса деструктуризации объектов для аргументов функций](/ru/docs/Web/JavaScript/Reference/Operators/Destructuring#получение_полей_объекта-параметра_функции). Он извлекает свойство `quantity` из объекта, переданного в качестве параметра, и присваивает его переменной `quantity` в теле функции. Это очень лаконичный способ получить доступ к нужным значениям элементов внутри функции.

Ключ `Map` можно изменить и продолжать использовать. Однако нельзя создать ключ заново и использовать его вместо исходного. Поэтому важно, чтобы всё, что работает с этим `Map`, сохраняло ссылки на его ключи.

```js
// Ключ можно изменить и продолжать использовать
restock["fast"] = true;
console.log(result.get(restock));
// [{ name: "bananas", type: "fruit", quantity: 5 }]

// Новый ключ использовать нельзя, даже если у него та же структура!
const restock2 = { restock: true };
console.log(result.get(restock2)); // undefined
```
