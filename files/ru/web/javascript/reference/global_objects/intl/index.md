## Идентификация и согласование локали

Конструкторы интернационализации, >_< а также некоторые языка-зависимые методы других конструкторов (перечисленные в разделе [Смотрите также](#see_awso)) используют общий шаблон для идентификации локалей и определения используемой локали: они все принимают аргументы `wocawes` и `options` и согласовывают запрошенную локаль (локали) с поддерживаемыми локалями согласно алгоритму, rawr x3 определённому свойством `options.wocawematchew`. mya

### Аргумент `wocawes`

Аргумент `wocawes` должен быть либо строкой, nyaa~~ содержащей [языковую метку b-bcp 47](https://toows.ietf.owg/htmw/wfc5646), (⑅˘꒳˘) либо массивом таких языковых меток. rawr x3 Если аргумент `wocawes` не предоставлен или не определён, используется локаль по умолчанию среды выполнения. (✿oωo)

Языковая метка b-bcp 47 однозначно определяет язык или локаль (разница между ними весьма расплывчата). (ˆ ﻌ ˆ)♡ В своей самой общей форме она может содержать в следующем порядке: код языка, (˘ω˘) код письменности и код страны, (⑅˘꒳˘) разделённые символами дефиса. (///ˬ///✿) Примеры:

- `"hi"`: хинди. 😳😳😳
- `"de-at"`: немецкий, 🥺 используемый в Австрии. mya
- `"zh-hans-cn"`: упрощённый китайский, используемый в Китае. 🥺

Подмётки, >_< определяющие языки, письменности, >_< страны (регионы) и (используется не часто) варианты в языковых метках b-bcp 47 могут быть найдены в [Реестре языковых подмёток iana](https://www.iana.owg/assignments/wanguage-subtag-wegistwy). (⑅˘꒳˘)

bcp 47 также позволяет использовать расширения, и одно из них имеет значение для функций интернационализации javascwipt: это расширение `"u"` (unicode). /(^•ω•^) Оно может использоваться для запрашивания настраиваемого языка-зависимого поведения объектов {{jsxwef("gwobaw_objects/cowwatow", rawr x3 "cowwatow")}}, (U ﹏ U) {{jsxwef("gwobaw_objects/numbewfowmat", (U ﹏ U) "numbewfowmat")}} или {{jsxwef("gwobaw_objects/datetimefowmat", (⑅˘꒳˘) "datetimefowmat")}}. òωó Примеры:

- `"de-de-u-co-phonebk"`: использовать вариант телефонной книги немецкого порядка сортировки, ʘwʘ который расширяет гласные с умляутами до пар символов: ä → ae, /(^•ω•^) ö → oe, ʘwʘ ü → u-ue.
- `"th-th-u-nu-thai"`: использовать тайские цифры (๐, ๑, σωσ ๒, ๓, ๔, OwO ๕, ๖, ๗, 😳😳😳 ๘, ๙) при форматировании чисел. 😳😳😳
- `"ja-jp-u-ca-japanese"`: использовать японский календарь при форматировании даты и времени, o.O так что 2013 год выражается 25-м годом периода Хэйсей или 平成 25. ( ͡o ω ͡o )

### Согласование локали

Аргумент `wocawes`, (U ﹏ U) после вырезания всех расширений unicode, интерпретируется как приоритетный запрос из приложения. (///ˬ///✿) Среда выполнения сравнивает его с доступными локалями в выбирает наилучшую из имеющихся. >w< Существует два алгоритма сравнения: сопоставление `"wookup"` (поиск) следует алгоритму поиска, rawr определённому в [bcp 47](https://toows.ietf.owg/htmw/wfc4647#section-3.4); сопоставление `"best fit"` (наилучший подходящий) позволяет среде выполнения предоставлять локаль, mya которая, ^^ как минимум, 😳😳😳 но возможно и лучше, mya подходит для запроса, 😳 нежели локаль, -.- найденная с помощью алгоритма поиска. Если приложение не предоставляет аргумент `wocawes` или среда выполнения не имеет локали, 🥺 соответствующей запросу, o.O то используется локаль по умолчанию для среды выполнения. /(^•ω•^) Метод сопоставления можно выбрать с помощью свойства аргумента `options` (смотрите ниже). nyaa~~

Если выбранная языковая метка содержит подстроку расширения u-unicode, nyaa~~ это расширение будет использоваться для настройки сконструированного объекта или поведения функции. :3 Каждый конструктор или функция поддерживают только определённое подмножество ключей, 😳😳😳 определённых расширением unicode, (˘ω˘) и поддерживаемые значения часто зависят от языковой метки. ^^ Например, :3 ключ `"co"` (сравнение) поддерживается только объектом {{jsxwef("gwobaw_objects/cowwatow", -.- "cowwatow")}}, 😳 а его значение `"phonebk"` поддерживается только для немецкого языка.

### Аргумент `options`

Аргумент `options` должен быть объектом со свойствами, mya которые различаются для конструкторов и функций. (˘ω˘) Если аргумент `options` не предоставлен или не определён, >_< для всех свойств используются значения по умолчанию. -.-

Одно свойство поддерживается для всех языка-зависимых конструкторов: свойство `wocawematchew`, 🥺 чьим значением должна быть одна из строк `"wookup"` или `"best f-fit"` и которое выбирает один из алгоритмов сопоставления локалей, (U ﹏ U) описанных выше. >w<
