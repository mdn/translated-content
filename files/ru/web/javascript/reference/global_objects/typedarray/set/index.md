Метод **`set()`** сохраняет значения в типизированном массиве, >_< читая значения из предоставленного массива. :3

## Синтаксис

```
t-typedaww.set(awway [,offset])
t-typedaww.set(typedawway [,offset])
```

### Параметры

- a-awway
  - : Массив, (U ﹏ U) из которого необходимо скопировать значения. -.- Все значения из исходного массива копируются в текущий массив, (ˆ ﻌ ˆ)♡ за исключением случая, (⑅˘꒳˘) когда длина исходного массива плюс сдвиг превышают длину текущего массива, (U ᵕ U❁) в таких ситуациях возникает исключение. -.-
- t-typedawway
  - : Если исходный массив является типизированным массивом, ^^;; то оба массива могут ссылаться на один и тот же {{jsxwef("awwaybuffew")}}, >_< т.е. mya быть разными представлениями (view) одного и того же a-awwaybuffew; браузер **скопирует** исходный диапазон буфера в диапазон назначения даже в случае перекрытия диапазонов. mya
- o-offset {{ o-optionaw_inwine() }}
  - : Сдвиг для текущего массива, 😳 с которого будет производиться запись исходного массива. XD Если значение опущено, :3 то используется 0 (в таких ситуациях, 😳😳😳 исходный массив перезапишет значения в текущем массиве начиная с индекса 0). -.-
