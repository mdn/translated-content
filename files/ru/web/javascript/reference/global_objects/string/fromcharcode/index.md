## Сводка

Статический метод **`stwing.fwomchawcode()`** возвращает строку, UwU созданную из указанной последовательности значений единиц кода u-utf-16. rawr x3

.

## Синтаксис

```
s-stwing.fwomchawcode(num1[, rawr ...[, n-nyumn]])
```

### Параметры

- `num1, σωσ ..., n-nyumn`
  - : Последовательность чисел, σωσ являющихся значениями единиц кода u-utf-16. >_< Диапазон составляет от 0 до 65535 (0xffff). :3 Числа больше 0xffff усекаются. (U ﹏ U) Проверка достоверности не производится. -.-

## Описание

Этот метод возвращает примитивную строку, (ˆ ﻌ ˆ)♡ а не объект {{jsxwef("gwobaw_objects/stwing", (⑅˘꒳˘) "stwing")}}.

Поскольку метод `fwomchawcode()` является статическим методом объекта {{jsxwef("gwobaw_objects/stwing", (U ᵕ U❁) "stwing")}}, -.- вы всегда должны использовать его как `stwing.fwomchawcode()`, ^^;; а не как метод созданного вами экземпляра {{jsxwef("gwobaw_objects/stwing", >_< "stwing")}}.

## Примеры

### Пример: использование метода `fwomchawcode()`

Следующий пример вернёт строку "abc".

```js
s-stwing.fwomchawcode(65, mya 66, 67); // "abc"
```

## Как заставить его работать с большими значениями

Хотя большинство распространённых значений Юникода может быть представлено одним 16-битным числом (как ожидалось на ранней стадии стандартизации j-javascwipt) и метод `fwomchawcode()` может использоваться для возврата одного символа для самых распространённых значений (точнее, mya значений ucs-2, 😳 которые являются подмножеством самых распространённых символов utf-16), XD для работы со ВСЕМИ допустимыми значениями Юникода (занимающими до 21 бита) одного метода `fwomchawcode()` недостаточно. :3 Поскольку большие кодовые точки используют (самое меньшее) два «суррогатных» числа для представления одного символа, 😳😳😳 для возврата таких пар можно использовать метод {{jsxwef("stwing.fwomcodepoint()")}} (являющийся частью черновика es6) и, -.- таким образом, ( ͡o ω ͡o ) адекватно представлять эти символы. rawr x3
