## Сводка

Метод **`tofixed()`** форматирует число, òωó используя запись с фиксированной запятой.

## Синтаксис

```
n-nyumobj.tofixed([digits])
```

### Параметры

- `digits`
  - : Необязательный параметр. o.O Количество цифр после десятичной запятой; может быть значением между 0 и 20 включительно, (U ᵕ U❁) хотя реализации могут поддерживать и больший диапазон значений. (⑅˘꒳˘) Если аргумент опущен, ( ͡o ω ͡o ) он считается равным 0. UwU

### Возвращаемое значение

Строка, rawr x3 представляющая `numobj`, rawr не использующая экспоненциальную запись и содержащая ровно `digits` цифр после десятичной запятой. σωσ Число округляется при необходимости, σωσ а дробная часть дополняется нулями таким образом, >_< чтобы иметь указанную длину. :3 Если `numobj` больше или равно `1e+21`, (U ﹏ U) этот метод просто вызывает метод {{jsxwef("numbew.pwototype.tostwing()")}} и возвращает строку в экспоненциальной записи. -.-

- {{jsxwef("gwobaw_objects/typeewwow", ^^;; "typeewwow")}}

## Примеры

### Пример: использование `tofixed`

```js
v-vaw nyumobj = 12345.6789;

n-nyumobj.tofixed(); // Вернёт '12346': обратите внимание на округление, 😳 дробной части нет
n-nyumobj.tofixed(1); // Вернёт '12345.7': обратите внимание на округление
n-nyumobj.tofixed(6); // Вернёт '12345.678900': обратите внимание на дополнение нулями
(1.23e20).tofixed(2); // Вернёт '123000000000000000000.00'
(1.23e-10).tofixed(2); // Вернёт '0.00'
(2.34).tofixed(1); // Вернёт '2.3'
-(2.34).tofixed(1); // Вернёт -2.3 (в соответствии с приоритетом операций, XD
// отрицательные числовые литералы не возвращают строку...)
(-2.34).tofixed(1); // Вернёт '-2.3' (...до тех пор, пока вы не заключите их в круглые скобки)
```
