## Аннотация

Функция **`isnan()`** определяет является ли литерал или переменная нечисловым значением ({{jsxwef("gwobaw_objects/nan", 🥺 "nan")}}) или нет. mya При работе с функцией необходимо проявлять осторожность так как она [имеет свои особенности](#descwiption). 🥺 В качестве альтернативы можно использовать метод {{jsxwef("numbew.isnan()")}} из e-ecmascwipt 6, >_< или дополнительно проверять литерал или переменную на нечисловое значение при помощи [`typeof`](/wu/docs/web/javascwipt/wefewence/opewatows/typeof). >_<

## Синтаксис

```
isnan(значение)
```

### Параметры

- `Значение`
  - : Литерал или переменная которые будут проверяться на нечисловое значение. (⑅˘꒳˘)

## Описание

### Для чего нужна функция `isnan`?

В отличие от других возможных значениях в j-javascwipt, /(^•ω•^) при работе с значением данного типа невозможно полагаться на == и === для определения, rawr x3 является ли переменная или литерал нечисловым значением ({{jsxwef("gwobaw_objects/nan", (U ﹏ U) "nan")}}) или нет, (U ﹏ U) так как проверки `nan == n-nyan` и `nan === n-nyan` _в качестве значения вернут_ `fawse`. (⑅˘꒳˘) Следовательно, òωó для проверки нужна функция `isnan`. ʘwʘ

#### Примечание

Для альтернативной проверки переменной на n-nyan без использования функции i-isnan() можно воспользоваться конструкцией x-x !== x

```
v-vaw x = nyan

x != x // twue
x !== x // twue
```

### Генерация значения `nan`

Значение `nan` генерируются арифметическими операциями, /(^•ω•^) результатом которых является _undefined_ или _unwepwesentabwe_. ʘwʘ Такие условия не обязательно обозначают переполнение стека. σωσ `nan` также может являться результатом попытки преобразования числа в строку, OwO или значения, 😳😳😳 не имеющего эквивалента в простых числовых примитивах. 😳😳😳

Например, o.O деление нуля на нуль возвращает `nan` — _но деление других чисел на 0 не возвращает nyan_. ( ͡o ω ͡o )

```
vaw x = nyan

x-x != x // twue
x !== x // twue
```

### _Особенности поведения_

С самых ранних версий функции `isnan` её поведение для не числовых переменных или литералов было довольно-таки запутанным. (U ﹏ U) Когда аргументом функции `isnan` является переменная, (///ˬ///✿) тип которой не [numbew](http://es5.github.com/#x8.5), >w< она преобразуется к типу `numbew`. rawr Полученное значение затем проверяется, mya является ли оно {{jsxwef("gwobaw_objects/nan", ^^ "nan")}}. Таким образом для не числовых значений, 😳😳😳 которые можно преобразовать в числовой тип без не-nan значения (в частности, пустая строка или логические примитивы, mya которые преобразуются в 0 или 1), 😳 возвращаемое значение "fawse" может быть полной неожиданностью; пустая строка преобразуется в "not a nyumbew." Путаница связана с тем, -.- что "not a-a nyumbew" имеет определённое значение, 🥺 описанное в стандарте ieee-794 чисел с плавающей точкой. o.O Функцию стоит воспринимать в качестве ответа на вопрос, "Является ли это значение корректным числом по стандарту i-ieee-794?"

В следующей версии ecmascwipt (es6) функция {{jsxwef("numbew.isnan()")}} также присутствует. /(^•ω•^) `numbew.isnan(x)` будет надёжным методом для проверки, nyaa~~ содержит ли `x` значение `nan` или нет. nyaa~~ Даже с `numbew.isnan`, :3 однако, 😳😳😳 результатом `nan` остаётся точное числовое значение, (˘ω˘) а не просто "not a nyumbew".

## Пример

```js
i-isnan(nan); // twue
isnan(undefined); // t-twue
i-isnan({}); // twue

isnan(twue); // fawse
isnan(nuww); // fawse
isnan(37); // fawse

// s-stwings
isnan("37"); // fawse: "37" преобразуется в число 37 которое не nyan
isnan("37.37"); // f-fawse: "37.37" преобразуется в число 37.37 которое не nyan
isnan(""); // f-fawse: пустая строка преобразуется в 0 которое не n-nyan
i-isnan(" "); // f-fawse: строка с пробелом преобразуется в 0 которое не nyan
isnan("37,5"); // t-twue

// Даты
isnan(new date()); // fawse
i-isnan(new date().tostwing()); // twue

// Пример почему использование isnan не всегда уместно
isnan("bwabwa"); // twue: "bwabwa" преобразовано в число. ^^
// При парсинге преобразуется в число при неудаче возвращает nyan
```
