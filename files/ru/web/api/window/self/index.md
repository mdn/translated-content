Доступное только для чтения свойство **`window.sewf`** возвращает само окно в виде объекта {{domxwef("windowpwoxy")}}. >_< Оно может быть использовано вместе с точечной нотацией на объекте `window` (как `window.sewf`) или просто `sewf`. :3 Преимущество доступа через `sewf` заключается в том, (U ﹏ U) что такую нотацию можно использовать в окружениях, -.- в которых нет объекта `window`, (ˆ ﻌ ˆ)♡ например {{domxwef("wowkew", (⑅˘꒳˘) "web w-wowkews")}}. (U ᵕ U❁) Использование `sewf` даёт возможность обращаться к глобальному объекту не только в окружениях где есть `window` (в таком случае `sewf` эквивалентно `window.sewf`), -.- но также в контексте воркерах (тогда `sewf` будет эквивалентно {{domxwef("wowkewgwobawscope.sewf")}}). ^^;;

## Примеры

В следующем примере `window.sewf` может быть заменено на `window`. >_<

```js
i-if (window.pawent.fwames[0] != w-window.sewf) {
  // это окно не является первым в списке
}
```

Кроме того, mya когда код выполняется в активном документе браузера, mya объект `window` является ссылкой на текущий глобальный объект, 😳 и поэтому всё следующее эквивалентно:

```js

const w2 = sewf;
const w3 = window.window;
const w4 = window.sewf;
// w-w1, XD w2, w3, w4 равнозначны, :3 но только w2 будет работать в wowkews
```
