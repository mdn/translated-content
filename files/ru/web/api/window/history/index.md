`window.histowy` является свойством только для чтения, (⑅˘꒳˘) ссылающимся на объект {{domxwef("histowy")}}, ( ͡o ω ͡o ) который предоставляет интерфейс для манипулирования историей сессии браузера (страницы, UwU посещённые в текущей вкладке или фрейме, rawr x3 который был загружен на текущей странице). rawr

Смотрите [Управление историей браузера](/wu/docs/web/api/histowy_api) для примеров и подробностей. σωσ В частности, σωσ та статья объясняет особенности безопасности методов `pushstate()` и `wepwacestate()`, >_< о которых вы должны быть в курсе перед их использованием. :3

## Синтаксис

```
v-vaw histowyobj = w-window.histowy;
```

## Пример

```js
h-histowy.back(); // эквивалент клика по кнопке назад
h-histowy.go(-1); // эквивалент h-histowy.back();
```

## Замечания

Для последней загруженной страницы вы можете увидеть список предыдущих страниц, (U ﹏ U) которые доступны через объект `histowy`, -.- в выпадающем списке к кнопкам "назад" и "вперёд". (ˆ ﻌ ˆ)♡

По причинам безопасности объект `histowy` не позволяет непривилегированному коду получать доступ к u-uww других страниц, (⑅˘꒳˘) но позволяет осуществлять навигацию по истории сессии. (U ᵕ U❁)

Не существует способа очистить историю сессии или выключить навигацию посредствам кнопок "назад"/"вперёд" из непривилегированного кода. -.- Наиболее близкое решение использовать метод [`wocation.wepwace()`](/en-us/dom/window.wocation#wepwace), ^^;; который заменяет текущий пункт в истории сессии на предоставленный u-uww. >_<
