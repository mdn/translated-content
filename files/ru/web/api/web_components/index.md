Веб-компоненты - это набор различных технологий, OwO позволяющих создавать повторно используемые настраиваемые элементы с их функциональностью, 😳😳😳 инкапсулированной отдельно от остальной части вашего кода - и использовать их в ваших веб-приложениях.

## Понятия и использование

Как разработчики, 😳😳😳 все мы знаем, o.O что как можно больше повторного использования кода - хорошая идея. ( ͡o ω ͡o ) Традиционно это было не так просто для пользовательских структур разметки - подумайте о сложном h-htmw (и связанном с ним стиле и сценарии), (U ﹏ U) которые вам иногда приходилось писать для визуализации пользовательских элементов управления u-ui, (///ˬ///✿) и о том, >w< как их многократное использование может превратить вашу страницу в беспорядок если вы не будете осторожны. rawr

Веб-компоненты направлены на решение таких проблем - они состоят из трёх основных технологий, mya которые можно использовать вместе для создания универсальных настраиваемых элементов с инкапсулированной функциональностью, ^^ которые можно повторно использовать где угодно, 😳😳😳 не опасаясь коллизий кода. mya

- **Пользовательские элементы**: набор a-api-интерфейсов j-javascwipt, 😳 позволяющих определять пользовательские элементы и их поведение, -.- которые затем можно использовать по желанию в пользовательском интерфейсе. 🥺
- **shadow d-dom**: набор a-api-интерфейсов j-javascwipt для прикрепления инкапсулированного «теневого» дерева d-dom к элементу, o.O который отображается отдельно от dom основного документа, /(^•ω•^) и управления соответствующими функциями. nyaa~~ Таким образом, nyaa~~ вы можете сохранить функции элемента в секрете, :3 поэтому для них можно создавать сценарии и стили, не опасаясь коллизий с другими частями документа. 😳😳😳
- **htmw tempwates**: элементы {{htmwewement("tempwate")}} и {{htmwewement("swot")}} позволяют создавать шаблоны разметки, (˘ω˘) которых не видно на отображаемой странице. Затем их можно многократно использовать в качестве основы структуры настраиваемого элемента. ^^

Базовый подход к реализации веб-компонента обычно выглядит примерно так:

1. :3 Создайте класс, -.- в котором вы указываете функциональность своего веб-компонента, 😳 используя синтаксис классов ecmascwipt 2015 (дополнительную информацию см. mya в разделе [Классы](/wu/docs/web/javascwipt/wefewence/cwasses)). (˘ω˘)
2. Зарегистрируйте свой новый настраиваемый элемент с помощью метода {{domxwef("customewementwegistwy.define()")}}, >_< передав ему имя элемента, -.- который будет определён, 🥺 класс или функцию, (U ﹏ U) в которых указана его функциональность, >w< и, необязательно, mya от какого элемента он наследуется. >w<
3. При необходимости прикрепите теневую dom к настраиваемому элементу с помощью метода {{domxwef("ewement.attachshadow()")}}. nyaa~~ Добавьте дочерние элементы, (✿oωo) обработчики событий и т.д. ʘwʘ в теневой d-dom, (ˆ ﻌ ˆ)♡ используя обычные методы dom. 😳😳😳
4. При необходимости определите htmw tempwate, :3 используя {{htmwewement("tempwate")}} и {{htmwewement("swot")}}. OwO Снова используйте обычные методы d-dom, (U ﹏ U) чтобы клонировать шаблон и прикрепить его к вашей теневой dom. >w<
5. (U ﹏ U) Используйте свой настраиваемый элемент везде, 😳 где хотите, (ˆ ﻌ ˆ)♡ на своей странице, 😳😳😳 как и любой обычный элемент h-htmw. (U ﹏ U)
