**Сериализуемый (Serializable) объект** - это объект, который может быть сериализован, а затем десериализован в любой среде JavaScript ("realm").
Это позволяет, например, хранить объекты на диске, а затем восстанавливать или клонировать их с помощью {{DOMxRef("Window.structuredClone", "structuredClone()")}}, или передавать их между коллегами благодаря {{domxref("DedicatedWorkerGlobalScope.postMessage()")}}.

Процесс сериализации может учитывать не все свойства и лишь некоторые аспекты исходного объекта.
Например, сериализация {{domxref("DOMException")}} должна включать свойства `name` и `message`, но будет ли она включать остальные свойства - зависит от реализации.
то есть, десериализованный объект может быть не идентичен клону/копии исходного объекта.
Однако новый десериализованный объект будет представлять собой {{glossary("deep copy")}}, поэтому любые свойства, которые были сериализованы из исходного объекта и затем десериализованы в новый объект, будут иметь отличные ссылки от исходного объекта.

В некоторых случаях при сериализации и десириализации объекта предпочтительней будет передать некоторые ресурсы в, а не создавать их копию.
Объекты, которые могут быть переданы, называются [Передаваемыми объектами](/en-US/docs/Web/API/Web_Workers_API/Transferable_objects).

## Поддерживаемые объекты

Все примитивные типы являются сериализуемыми.
Не все объекты могут быть сериализуемыми.
Объекты, которые могут быть сериализованы, перечислены в: [Алгоритм структурированного клонирования > Поддерживаемые типы](/ru/docs/Web/API/Web_Workers_API/Structured_clone_algorithm#поддерживаемые_типы).

> [!NOTE]
> Сериализуемые объекты помечены в [Web IDL files](https://github.com/w3c/webref/tree/main/ed/idl) атрибутом `[Serializable]`.
