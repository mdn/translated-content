title: Доступное имя

**Доступное имя** — это имя элемента пользовательского интерфейса; текст, связанный с HTML-элементом, который предоставляет пользователям вспомогательных технологий метку для этого элемента.

Доступные имена передают назначение или предназначение элемента. Это помогает пользователям понять, для чего нужен элемент и как они могут с ним взаимодействовать. Как правило, доступные имена элементов должны быть уникальными на странице. Это помогает пользователям отличать один элемент от других и идентифицировать элемент, с которым они хотят взаимодействовать.

В зависимости от элемента и разметки HTML значение доступного имени может быть получено из видимого (например, текст внутри {{HTMLElement("figcaption")}}) или невидимого (например, атрибут `aria-label`, установленный на элементе) содержимого, либо из их комбинации. Способ определения доступного имени элемента основан на [вычислении доступного имени](https://www.w3.org/WAI/ARIA/apg/practices/names-and-descriptions/#name_calculation), которое отличается для разных элементов.

Лучше всего использовать видимый текст в качестве доступного имени. Многие элементы, включая {{HTMLElement("a")}}, {{HTMLElement("td")}} и {{HTMLElement("button")}}, могут получать своё доступное имя из своего содержимого. Например, для `<a href="foo.html">Бар</a>` доступным именем этой гиперссылки будет «Бар».

Другие элементы получают доступное имя из содержимого связанных элементов. Например, когда элемент {{HTMLElement("fieldset")}} или {{HTMLElement("table")}} содержит внутри себя элемент {{HTMLElement("legend")}} или {{HTMLElement("caption")}} соответственно, связь вложенного элемента, предоставляющего доступное имя для родителя, устанавливается автоматически. Для элементов формы, таких как {{HTMLElement("textarea")}} и {{HTMLElement("input")}}, доступное имя берётся из связанного элемента {{HTMLElement("label")}}. Связь должна быть явно задана с помощью атрибута `for` в элементе `<label>`, который должен совпадать с `id` элемента формы. Альтернативно, неявная связь создаётся, когда элемент управления формой напрямую вложен в элемент `<label>`.

Для некоторых элементов доступное имя берётся из атрибутов элемента; например, атрибут `alt` в случае {{HTMLElement("img")}}. Для `<img src="grape.jpg" alt="банан"/>` доступным именем изображения будет «банан».

Для создания связи между видимым содержимым и элементом или несколькими текстовыми узлами и элементом можно использовать атрибут [`aria-labelledby`](/ru/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-labelledby). Если нет видимого текста, который можно связать с элементом интерфейса, нуждающимся в доступном имени, можно использовать атрибут [`aria-label`](/ru/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-label). Не следует добавлять имена к элементам, размечающим строчный текст, таким как {{HTMLElement("code")}}, {{HTMLElement("del")}} и {{HTMLElement("mark")}}.

Многие элементы, например, разделы текстового содержимого, не нуждаются в доступном имени. Все элементы управления должны иметь доступное имя. Все изображения, которые передают информацию и не являются чисто декоративными, также должны его иметь.

Вспомогательные технологии предоставляют пользователям свойство доступного имени, которое включает доступное имя вместе с ролью элемента. Хотя многие элементы не нуждаются в доступном имени, необходимо предоставлять доступное имя для переопределения или дополнения содержимого элементов с определёнными [ролями](/ru/docs/Web/Accessibility/ARIA/Reference/Roles). Например, [`tabpanel`](/ru/docs/Web/Accessibility/ARIA/Reference/Roles/tabpanel_role) — это раздел содержимого, который появляется после того, как пользователь активирует связанный элемент с ролью [`tab`](/ru/docs/Web/Accessibility/ARIA/Reference/Roles/tab_role). Эта роль может быть установлена на элементе, которому не нужно имя, например, на элементе {{HTMLElement("div")}}. Элемент `tab` является элементом управления и должен иметь доступное имя. Элемент `tabpanel` является дочерним (разделом содержимого) по отношению к `tab`. Добавление `aria-labelledby` к `tabpanel` — это рекомендуемая практика.
