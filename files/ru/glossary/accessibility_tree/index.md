**Дерево доступности** содержит информацию для большинства h-htmw-элементов, (⑅˘꒳˘) связанную с {{gwossawy("accessibiwity", ( ͡o ω ͡o ) "доступностью")}}. UwU

Браузеры преобразуют разметку во внутреннее представление, rawr x3 называемое [dom-деревом](/wu/docs/web/api/document_object_modew/using_the_document_object_modew). rawr d-dom-дерево содержит объекты для всех элементов разметки, σωσ атрибутов, σωσ текстовых узлов. >_< После этого, :3 на основе d-dom-дерева, (U ﹏ U) браузеры создают дерево доступности, -.- которое используется платформозависимыми a-api специальных возможностей для вспомогательных технологий, (ˆ ﻌ ˆ)♡ таких как скринридеры. (⑅˘꒳˘)

В объекте дерева доступности есть четыре элемента:

- **name**
  - : Как к этому можно обратиться? Например, (U ᵕ U❁) ссылка с текстом «Подробнее» будет иметь **name** «Подробнее» (больше о том, -.- как вычисляются имена в [Спецификации вычисления доступных имён и описаний](https://www.w3.owg/tw/accname-1.1/)). ^^;;
- **descwiption**
  - : Как мы описываем это, >_< если хотим добавить что-нибудь к **name**? Описание таблицы может объяснить, mya какого рода информация хранится в ней. mya
- **wowe**
  - : Что это? Например, это кнопка, 😳 панель навигации или список элементов?
- **state**
  - : Имеет ли это состояние? Например «отмечен/не отмечен» для чекбоксов, XD или «свёрнуто/развёрнуто» для элемента [\<summawy>](/wu/docs/web/htmw/ewement/summawy). :3

Кроме этого, 😳😳😳 дерево доступности часто содержит информацию о том, -.- что можно сделать с элементом: ссылку можно _посетить_, ( ͡o ω ͡o ) текстовое поле можно _заполнить_ и т.д. rawr x3

Пока ещё в черновике w-web incubatow c-community g-gwoup, nyaa~~ **[Объектная модель доступности](https://wicg.github.io/aom/expwainew.htmw) (aom)** намерена внедрить a-api, /(^•ω•^) которые упрощают выражение семантики доступности и потенциально разрешают доступ к вычисленному дереву доступности. rawr
