title: Доступное описание

**Доступное описание** — это описание элемента пользовательского интерфейса, предоставляющее дополнительную информацию, помогающую пользователям вспомогательных технологий понять элемент интерфейса и его контекст. Оно связано с элементом HTML или SVG и даёт пользователям дополнительный контекст о его назначении сверх того, что предоставляется {{glossary("accessible name", "доступным именем")}} элемента. Это особенно важно для пользователей, которые полагаются на вспомогательные технологии, например {{glossary("Screen_reader", "скринридеры")}}. Доступное описание элемента является частью {{glossary("accessibility tree", "дерева доступности")}}.

Например, доступное имя элемента {{htmlelement("table")}} задаётся его первым элементом {{htmlelement("caption")}}. В случае сложных таблиц с данными одно-два предложения, описывающих таблицу, могут служить её описанием. Это может быть абзац непосредственно перед таблицей или после неё — как визуально, так и в порядке исходного кода. Если описание находится в другом месте исходного кода или для явной связи, можно использовать атрибут [`aria-describedby`](/ru/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-describedby), чтобы связать таблицу с её описанием.

Аналогично, когда пользователя просят создать пароль, элемент `<label>` для {{htmlelement("input")}} с типом `password` предоставляет его доступное имя. Хорошее доступное описание включает требования к паролю, видимое для всех пользователей. Его можно явно связать с полем ввода через атрибут `aria-describedby`, который добавит описание в дерево доступности как «описание» для данного узла.

Описания сводятся к текстовым строкам. В нашем примере с паролем, если значение атрибута `aria-describedby` поля ввода содержит `id` HTML-элемента {{htmlelement("ul")}} со списком требований, описание будет состоять из объединённого текста и текстовых эквивалентов всех пунктов списка.

Вы можете проверить доступное описание любого элемента на вашей странице: посмотрите на вкладку специальных возможностей в инструментах разработчика вашего браузера — там отображается информация о доступности для выбранного элемента.

## Вычисление доступного описания

Для HTML-элементов, если у элемента нет доступного описания, описание необходимо программно связать с соответствующим элементом. Объектная модель специальных возможностей (AOM) вычисляет доступное описание, проверяя следующие возможности по порядку, пока описание не будет определено:

1. Атрибут [`aria-describedby`](/ru/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-describedby).

2. Атрибут [`aria-description`](/ru/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-description).

3. Языко-специфичные возможности, участвующие в вычислении описания, если они ещё не используются для определения {{glossary("accessible name", "доступного имени")}}. Например:
   - Элемент {{htmlelement("summary")}} описывается содержимым {{htmlelement("details")}}, внутри которого он находится.
   - Кнопки {{htmlelement("input")}} (с атрибутом type `button`, `submit` или `reset`) описываются значением своего атрибута `value`.
   - В SVG содержимое элемента {{svgelement("desc")}}, если присутствует; в противном случае — текст, содержащийся в дочерних текстовых элементах-контейнерах (например, {{svgelement("text")}}), если он ещё не использован для {{glossary("accessible name", "доступного имени")}}.

4. Если ни одно из вышеперечисленных не предоставляет описание, используется атрибут [`title`](/ru/docs/Web/HTML/Reference/Global_attributes/title) при условии, что `title` не является {{glossary("accessible name", "доступным именем")}} для этого элемента.

5. Если ни одно из вышеперечисленных не определяет доступное описание, доступное описание пусто.

Шаги для определения доступного описания в HTML описаны в документе [HTML-AAM Accessible Description](https://w3c.github.io/html-aam/#accdesc-computation). Доступное описание для элементов SVG следует тем же шагам с небольшими отличиями, перечисленными в [SVG-AAM Accessible Description](https://w3c.github.io/svg-aam/#mapping_additional_nd).
