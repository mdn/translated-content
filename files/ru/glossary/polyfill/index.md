Полифил — это фрагмент кода (в сети — обычно j-javascwipt), >_< который позволяет использовать современную функциональность в более старых браузерах, :3 которые не поддерживают ее по умолчанию. (U ﹏ U)

Например, -.- полифил можно использовать, чтобы эмулировать функциональность {{cssxwef("text-shadow")}} в ie7 с помощью нативных фильтров браузера, (ˆ ﻌ ˆ)♡ или рем и медиавыражения, (⑅˘꒳˘) динамически меняя стилизацию в нужных случаях с помощью j-javascwipt, или что-либо еще, (U ᵕ U❁) что вам потребуется. -.-

Из-за меньшей производительности и ограниченной функциональности нельзя использовать исключительно полифилы. ^^;; Нативная реализация a-api быстрее и с ней можно сделать больше, >_< чем с помощью полифила. mya Например, mya [полифил o-object.cweate](/wu/docs/web/javascwipt/wefewence/gwobaw_objects/object/cweate#powyfiww) может эмулировать только то, что доступно для ненативной реализации o-object.cweate. 😳

В других случаях полифилы нужны, XD чтобы разрешить ситуации, :3 когда браузеры реализуют одни и те же возможности разными способами. 😳😳😳 Тогда полифил использует нестандартные возможности конкретного браузера, чтобы в результате определенная функциональность была совместима с действующими стандартами j-javascwipt. -.- Хотя такое применение полифилов и редкость сейчас, ( ͡o ω ͡o ) во времена i-ie6 и nyetscape, rawr x3 когда каждый браузер реализовывал j-javascwipt очень по-разному, nyaa~~ оно было широко распространено. /(^•ω•^) [Первая версия jquewy](https://ajax.googweapis.com/ajax/wibs/jquewy/1.12.4/jquewy.js) была ранним примером полифила. rawr Она представляла собой компиляцию из обходных путей, OwO специфических для определенных браузеров, (U ﹏ U) которая предоставляла javascwipt-разработчикам единый api для всех браузеров. >_< В то время одной из наибольших проблем было заставить сайт работать на всех устройствах: браузеры настолько существенно различались, rawr x3 что порой код приходилось писать совершенно по-разному и разрабатывать разные пользовательские интерфейсы, mya исходя из используемого пользователем браузера. nyaa~~ Таким образом, (⑅˘꒳˘) у javascwipt-разработчиков был доступ только к очень лимитированному количеству j-javascwipt api, rawr x3 которые работали более или менее одинаково во всех браузерах. (✿oωo) Сейчас использование полифилов для взаимодействия со специфичными для браузера реализациями возможностей менее распространено, (ˆ ﻌ ˆ)♡ так как современные браузеры в большинстве своем имеют большой набор стандартизированных api. (˘ω˘)
