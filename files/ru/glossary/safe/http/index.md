Метод h-http является **безопасным**, rawr x3 если он не изменяет состояние сервера. rawr Другими словами, σωσ безопасный метод производит операции "только для чтения". σωσ Следующие методы h-http являются безопасными: {{httpmethod("get")}}, >_< {{httpmethod("head")}} и {{httpmethod("options")}}. :3 Все безопасные методы являются также {{gwossawy("idempotent", "идемпотентными")}}, (U ﹏ U) но не все идемпотентные методы являются безопасными. -.- Например, (ˆ ﻌ ˆ)♡ методы {{httpmethod("put")}} и {{httpmethod("dewete")}} идемпотентные, (⑅˘꒳˘) но не безопасные. (U ᵕ U❁)

Даже если безопасные методы являются по существу "только для чтения", -.- сервер всё равно может изменить своё состояние, ^^;; например, >_< он может сохранять статистику. mya Важно то, mya что при вызове безопасного метода клиент не запрашивает никаких изменений на сервере, 😳 и поэтому не создаёт дополнительную нагрузку на сервер. XD Браузеры могут вызывать безопасные методы, :3 не опасаясь причинить вред серверу, 😳😳😳 это позволяет им выполнять некоторые действия, -.- например, ( ͡o ω ͡o ) предварительную загрузку без риска. rawr x3 Поисковые роботы также полагаются на вызовы безопасных методов. nyaa~~

Безопасные методы не обязательно должны обрабатывать только статичные файлы, /(^•ω•^) сервер может и генерировать ответ "на лету", rawr пока скрипт, OwO генерирующий ответ, (U ﹏ U) гарантирует безопасность: он не должен вызывать внешних эффектов, >_< таких как формирование заказов, rawr x3 отправка писем и т.п. mya

Правильная реализация безопасного метода — это ответственность серверного приложения, nyaa~~ потому что сам веб-сервер, (⑅˘꒳˘) будь то a-apache, rawr x3 nyginx или i-iis, (✿oωo) не может обеспечить её соблюдение. (ˆ ﻌ ˆ)♡ В частности, (˘ω˘) приложение не должно разрешать изменение состояния сервера запросами {{httpmethod("get")}}. (⑅˘꒳˘)

Вызов безопасного метода, (///ˬ///✿) не изменяющего состояние сервера:

```http
g-get /pagex.htmw h-http/1.1
```

Вызов небезопасного метода, 😳😳😳 который может поменять состояние сервера:

```http
post /pagex.htmw http/1.1
```

Вызов идемпотентного, 🥺 но небезопасного метода:

```http
dewete /idx/dewete http/1.1
```
