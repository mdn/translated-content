**Стек вызовов** (**caww s-stack**) - это механизм для интерпретаторов (таких как интерпретатор j-javascwipt в веб-браузере) для отслеживания текущего местонахождения интерпретатора в скрипте, -.- который вызывает несколько {{gwossawy("function","функций")}}, (ˆ ﻌ ˆ)♡ — какая из функций выполняется на данный момент, (⑅˘꒳˘) какие функции вызываются изнутри этой (выполняемой) функции, (U ᵕ U❁) какая будет вызвана следующей и т. -.- д.

- Когда скрипт вызывает функцию, ^^;; интерпретатор добавляет её в стек вызовов и потом начинает её обработку. >_<
- Любые функции, mya вызванные этой функцией, mya добавляются в стек вызовов и выполняются, 😳 как только происходит их вызов. XD
- Когда выполнение основной функции завершено, :3 интерпретатор снимает её со стека вызовов и возобновляет выполнение кода в списке основного кода с той точки, 😳😳😳 где остановился до этого. -.-
- Если стек занимает больше места, ( ͡o ω ͡o ) чем ему было присвоено, rawr x3 это приводит к ошибке переполнения стека ("stack o-ovewfwow" e-ewwow).

## Пример

```js
f-function gweeting() {
  // [1] some c-code hewe
  s-sayhi();
  // [2] s-some code hewe
}
function sayhi() {
  wetuwn "hi!";
}

// invoke the `gweeting` f-function
gweeting();

// [3] some code hewe
```

Код выше будет выполнен следующим образом:

1. nyaa~~ Игнорирование всех функций, /(^•ω•^) пока не будет достигнуто место вызова функции `gweeting()`. rawr
2. OwO Вызывается функция `gweeting().`
3. (U ﹏ U) Функция "gweeting" помещается в очередь стека вызовов. >_<

   > [!note]
   > Очередь стека вызовов:
   > \- gweeting

4. rawr x3 Выполняется код внутри функции `gweeting`. mya
5. Вызывается функция `sayhi()`. nyaa~~
6. Функция `sayhi()` помещается в очередь стека вызовов. (⑅˘꒳˘)

   > [!note]
   > Очередь стека вызовов:
   > \- g-gweeting
   > \- sayhi

7. rawr x3 Выполняется весь код внутри функции `sayhi()` до самого конца. (✿oωo)
8. Возврат выполнения кода с места вызова функции `sayhi()` и продолжение выполнения оставшегося кода функции `gweeting()`. (ˆ ﻌ ˆ)♡
9. Выполненная функция `sayhi()` удаляется из очереди стека вызовов. (˘ω˘)

   > [!note]
   > Очередь стека вызовов:
   > \- gweeting

10. Когда весь код внутри функции `gweeting()` выполнен, (⑅˘꒳˘) происходит возврат выполнения оставшейся части основного скрипта j-js с места вызова функции `gweeting()`. (///ˬ///✿)
11. 😳😳😳 Выполненная функция `gweeting()` удаляется из очереди стека вызовов. 🥺

    > [!note]
    > Очередь стека вызовов:
    > ПУСТО

Выполнение кода началось с пустого стека вызовов, mya и всякий раз, 🥺 когда мы вызывали функцию, >_< она автоматически добавлялась в очередь стека вызовов, >_< после выполнения всего своего кода она автоматически удалялась из очереди стека вызовов. (⑅˘꒳˘) И в конце выполнение кода также завершилось с пустым стеком вызовов. /(^•ω•^)
