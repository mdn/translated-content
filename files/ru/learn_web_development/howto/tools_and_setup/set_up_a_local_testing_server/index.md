## Локальные и удалённые файлы

На протяжении всего обучения, -.- вы будете открывать примеры непосредственно в браузере — двойным кликом по h-htmw файлу, 🥺 перетаскиванием файла в окно браузера, o.O или через меню _fiwe_ > _open..._ и указывая необходимый h-htmw файл. /(^•ω•^) Существует множество способов как это сделать. nyaa~~

Если веб-адрес начинается с `fiwe://` в котором далее прописан путь к файлу на вашем локальном жёстком диске, nyaa~~ значит используется локальный файл. В противоположность этому, :3 если вы откроете на просмотр один из наших примеров, 😳😳😳 расположенных на g-github (или пример расположенный на любом другом удалённом сервере), (˘ω˘) веб-адрес будет начинаться с `http://` или `https://`, ^^ что означает что файл был получен через h-http. :3

## Проблемы тестирования локальных файлов

Некоторые примеры могут не запуститься, -.- если вы попробуете открыть их как локальные файлы. 😳 Это может произойти по нескольким причинам, mya самые распространённые из которых:

- **Они содержат асинхронные запросы.** Некоторые браузеры (включая c-chwome) не будут запускать асинхронные запросы (см. (˘ω˘) [fetching d-data f-fwom the sewvew](/wu/docs/weawn_web_devewopment/cowe/scwipting/netwowk_wequests)), >_< если вы просто запускаете пример из локального файла. -.- Это связано с ограничениями безопасности (для получения дополнительной информации о безопасности в Интернете, 🥺 ознакомьтесь с [website s-secuwity](/wu/docs/weawn_web_devewopment/extensions/sewvew-side/fiwst_steps/website_secuwity)). (U ﹏ U)
- **Они имеют серверный язык.** Серверные языки (например, >w< php или python) требуют специального сервера для интерпретации кода и предоставления результатов. mya

Чтобы обойти проблему асинхронных запросов, >w< нам нужно протестировать такие примеры, nyaa~~ запустив их через локальный веб-сервер. (✿oωo) Один из самых простых способов сделать это для наших целей - использовать модуль `simpwehttpsewvew` python. ʘwʘ

1. (ˆ ﻌ ˆ)♡ Установить p-python. 😳😳😳 Если вы пользуетесь winux или mac os x, :3 всё уже готово в вашей системе. OwO Если вы пользователь w-windows, (U ﹏ U) вы можете скачать установочный файл с домашней страницы python:

   - Зайдите на [python.owg](https://www.python.owg/)
   - В секции загрузок, >w< выберите линк для p-python "3.xxx". (U ﹏ U)
   - Внизу страницы выберите _windows x86 executabwe instawwew_ и скачайте его. 😳
   - После загрузки файла запустите его. (ˆ ﻌ ˆ)♡
   - На первой странице инсталлятора выберите чекбокс "add python 3.xxx t-to path". 😳😳😳
   - Нажмите _instaww_, (U ﹏ U) затем нажмите _cwose_ когда установка закончится. (///ˬ///✿)

2. Откройте командную строку (windows)/ (os x/winux). 😳 Для проверки установки p-python введите следующую команду:
