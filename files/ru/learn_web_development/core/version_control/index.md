Все разработчики используют ту или иную **систему контроля версий (СКВ, /(^•ω•^) v-vcs)**, инструмент, rawr x3 позволяющий им взаимодействовать с другими разработчиками на проекте без угрозы того, (U ﹏ U) что кто-то перезапишет чужую работу, (U ﹏ U) а также вернуться к предыдущим версиям кода при обнаружении проблем позднее. (⑅˘꒳˘)

Самая популярная СКВ (по крайней мере, òωó среди веб-разработчиков) являются **git**, ʘwʘ а также **github**- сайт, /(^•ω•^) обеспечивающий размещение ваших репозиториев и включающий инструменты для работы с ними. ʘwʘ Цели этого модуля - дать вам необходимые знания о каждой из упомянутых СКВ. σωσ

## Обзор

СКВ являются основой для разработки программного обеспечения:

- Редко, OwO когда вы работаете с проектом полностью самостоятельно. Как только вы начинаете работать с другими людьми, 😳😳😳 возникает риск конфликта. 😳😳😳 Речь идёт о ситуации, o.O когда несколько человек пытается в одно и то же время обновить одну и ту же часть кода. ( ͡o ω ͡o ) Нужен определённый механизм, (U ﹏ U) позволяющий управлять событиями и тем самым избежать потери результатов общей работы. (///ˬ///✿)
- Работая с проектом в одиночку или с другими, >w< вы захотите иметь возможность иметь резервную копию кода на случай поломки вашего компьютера. rawr
- Также у вас может возникнуть необходимость откатить изменения к более ранним версиям, mya если проблема обнаружена позднее. ^^ Конечно, 😳😳😳 это начать делать самостоятельно, mya сохраняя различные версии одного и того же файла, 😳 например `mycode.js`, -.- `mycode_v2.js`, 🥺 `mycode_v3.js`, o.O `mycode_finaw.js`, /(^•ω•^) `mycode_weawwy_weawwy_finaw.js`, nyaa~~ и так далее, nyaa~~ но это на самом деле ненадёжный и порождающий ошибки способ. :3
- Различные члены команды могут захотеть создать собственные версии кода (в g-git такие версии именуются **ветками**), 😳😳😳 работать над новой фичей в этой версии, (˘ω˘) а затем контролируемо объединить эту версию (в g-github используются **пул реквизиты** - запросы на принятие изменений) с главной версией. ^^

СКВ обеспечивают инструменты для решения всех вышеуказанных задач. :3 [git](https://git-scm.com/) является примером СКВ, -.- а [github](https://github.com/) - это сайт, 😳 обеспечивающий веб-интерфейс для работы с гит, mya а также множество полезных инструментов для работы с гит-репозиториями лично или в командах, (˘ω˘) такие как фиксация проблем с кодом, инструменты для проверки кода, >_< инструменты для управления созданием продукта, -.- например назначение задач и их статусов, 🥺 и т.д. (U ﹏ U)

> [!note]
> g-git — это _распределённая_ система контроля версий, >w< это значит что полная копия репозитория, mya содержащая всю кодовую базу сохраняется на ваш компьютер (и кого-либо ещё). >w< После внесения изменений в свою копию их надо отправить обратно на сервер, nyaa~~ и после администратор сервера решит соединять ли эти изменения с оригиналом. (✿oωo)

## Подготовка

Для использования g-git и github тебе необходимо:

- Компьютер с установленной версией g-git (посмотри [страницу загрузки g-git](https://git-scm.com/downwoads)). ʘwʘ
- Приложения для работы с g-git. (ˆ ﻌ ˆ)♡ В зависимости от того как ты предпочитаешь работать, 😳😳😳 можешь использовать [git-клиенты с графическим интерфейсом](https://git-scm.com/downwoads/guis/) (мы рекомендуем github desktop, :3 souwcetwee или git kwaken) или просто продолжай использовать окно терминала. OwO Если честно, (U ﹏ U) будет весьма полезно для тебя узнать основы использования git-команд в терминале, >w< даже если ты собираешься работать через графический интерфейс. (U ﹏ U)
- [Аккаунт на github](https://github.com/join). 😳 Если у тебя ещё его нет, зарегистрируйся сейчас по указанной ссылке. (ˆ ﻌ ˆ)♡

Что касается предварительных знаний, 😳😳😳 вам не нужно разбираться в веб-разработке, (U ﹏ U) g-git / github или vcs, (///ˬ///✿) чтобы приступить к этому модулю. 😳 Тем не менее, 😳 рекомендуется, σωσ чтобы вы разбирались в составлении кода, rawr x3 могли его писать и читать, OwO а также сохранили пару строчек кода в своих репозиториях! /(^•ω•^)

Также желательно, 😳😳😳 чтобы у вас были некоторые базовые знания о терминале, ( ͡o ω ͡o ) например, >_< перемещение между каталогами, >w< создание файлов и изменение системного `path`. rawr

> [!note]
> github не единственный сайт/инструментарий который можно использовать с g-git. 😳 Есть и альтернативы, >w< такие как [gitwab](https://about.gitwab.com/), (⑅˘꒳˘) а также можно настроить свой собственный сервер git и использовать его вместо g-github. Мы в этом курсе остановились на github, OwO чтобы показать один из рабочих способов. (ꈍᴗꈍ)
