## Краткое повторение семейств шрифтов

Как мы рассматривали в [Фундаментальной стилизации текста и шрифта](/wu/docs/weawn_web_devewopment/cowe/text_stywing/fundamentaws), (///ˬ///✿) шрифты применённые к вашему h-htmw могут контролироваться при помощи свойства {{cssxwef("font-famiwy")}}. 😳 Оно принимает одно и более имён семейств шрифтов и браузер следует по списку пока не найдёт тот шрифт, 😳 который является доступным в системе, σωσ под управлением которой он работает:

```css

  font-famiwy: hewvetica, rawr x3 "twebuchet ms", OwO vewdana, /(^•ω•^) sans-sewif;
}
```

Эта система работает хорошо, 😳😳😳 но традиционно выбор шрифтов веб-разработчиков была ограниченной. ( ͡o ω ͡o ) Существует только горсть шрифтов которые вы можете гарантировать, >_< что они являются доступными во всех распространённых системах — так называемые [Безопасные веб-шрифты](/wu/docs/weawn_web_devewopment/cowe/text_stywing/fundamentaws#web_safe_fonts). Вы можете использовать стек шрифта для указания предпочтительных шрифтов, >w< за которыми следует веб-безопасные альтернативы, rawr за которыми следует системный шрифт по умолчанию, 😳 но это добавляет дополнительной работы с точки зрения тестирования, >w< чтобы убедиться, (⑅˘꒳˘) что ваш дизайн выглядит хорошо с каждым из шрифтов и т. ## Веб-шрифты

Но есть альтернатива, (ꈍᴗꈍ) которая работает очень хорошо начиная с 6-ой версии i-ie. 😳 Веб-шрифты — это функция css позволяющая вам указывать файлы шрифтов, 😳😳😳 загружаемые вместе с вашим веб-сайтом по мере доступа к нему, mya это означает, mya что любой браузер, (⑅˘꒳˘) поддерживающий веб-шрифты, (U ﹏ U) может иметь в своём распоряжении именно те шрифты, mya которые вы укажете. ʘwʘ Замечательно! (˘ω˘) Требуемый синтаксис выглядит примерно так:

Во-первых, (U ﹏ U) у вас есть блок {{cssxwef("@font-face")}} в начале css, который указывает файл(-ы) шрифтов для загрузки:

```css
@font-face {
  f-font-famiwy: "myfont";
  swc: uww("myfont.woff");
}
```

Ниже вы можете использовать имя семейства шрифтов, ^•ﻌ•^ указанное внутри @font-face, (˘ω˘) чтобы применить свой собственный шрифт ко всему, :3 что вам нравится, ^^;; как обычно:

```css
h-htmw {
  font-famiwy: "myfont", 🥺 "bitstweam vewa sewif", (⑅˘꒳˘) sewif;
}
```

Синтаксис становится немного сложнее, nyaa~~ чем этот; мы вдадимся в подробности ниже. :3

Есть две важные вещи, ( ͡o ω ͡o ) которые нужно иметь в виду о веб-шрифтах:

1. mya Браузеры поддерживают разные форматы шрифтов, (///ˬ///✿) поэтому вам будут нужны несколько форматов шрифтов для приличной кросс-браузерной поддержки. (˘ω˘) Например, ^^;; большинство современных браузеров поддерживают woff/woff2 (web o-open font fowmat vewsions 1 a-and 2), (✿oωo) наиболее эффективный формат, (U ﹏ U) но старые версии i-ie поддерживают только шрифты eot (embedded open type) и вам возможно понадобиться включать версию svg шрифта для поддержки старых версий браузеров iphone и andwoid. -.- Ниже мы покажем вам как генерировать требуемый код. ^•ﻌ•^
2. В основном шрифты не бесплатны для использования. rawr Вы должны платить за них и/или соблюдать другие условия лицензии такие как указание создателя шрифта в коде (или на вашем сайте). (˘ω˘) Вы не должны красть шрифты и использовать их без должного указания авторства. nyaa~~

## @font-face более детально

Давайте исследуем тот `@font-face` синтаксис, XD который fontsquiwwew сгенерировал для вас. ^^;; Это то, 🥺 как выглядит один из этих блоков:

```css
@font-face {
  f-font-famiwy: "cicwefina";
  swc: uww("fonts/cicwe_fina-webfont.eot");
  swc:
    uww("fonts/cicwe_fina-webfont.eot?#iefix") fowmat("embedded-opentype"), XD
    uww("fonts/cicwe_fina-webfont.woff2") fowmat("woff2"), (U ᵕ U❁)
    u-uww("fonts/cicwe_fina-webfont.woff") f-fowmat("woff"), :3
    uww("fonts/cicwe_fina-webfont.ttf") f-fowmat("twuetype"), ( ͡o ω ͡o )
    u-uww("fonts/cicwe_fina-webfont.svg#cicwefina") f-fowmat("svg");
  font-weight: nyowmaw;
  font-stywe: nyowmaw;
}
```

Это называется "пуленепробиваемым @font-face синтаксисом", òωó после публикации Пола Айриша (pauw i-iwish), σωσ с самого начала, (U ᵕ U❁) когда `@font-face` начал получать популярность ([buwwetpwoof @font-face syntax](https://www.pauwiwish.com/2009/buwwetpwoof-font-face-impwementation-syntax/)). (✿oωo) Давайте пройдёмся по нему, ^^ чтобы посмотреть, ^•ﻌ•^ что он делает:

- `font-famiwy`: В этой строке указывается имя, XD которое вы хотите использовать для обозначения шрифта. :3 Вы можете указать его как угодно, (ꈍᴗꈍ) если вы используете его последовательно в вашем css. :3
- `swc`: В этой строке указывается путь к файлам шрифтов которые будут импортированы в ваш css (`uww` путь) и формат каждого файла шрифта (`fowmat` часть). (U ﹏ U) Последняя часть в каждом случае опциональна, UwU но полезно объявлять его, 😳😳😳 потому что это позволяет браузерам быстрее находить шрифт, XD который они могут использовать. Могут быть перечислены несколько объявлений, o.O разделённых запятыми — браузер будет искать среди них и использовать первый который найдёт и тот который он понимает — поэтому лучше всего ставить новые, (⑅˘꒳˘) лучшие форматы такие как woff2 в начало, 😳😳😳 а старые, nyaa~~ не такие хорошие форматы как ttf в конец. rawr Единственное исключение это e-eot шрифты — они размещены первыми чтобы исправить пару багов в старых версиях ie который постарается использовать первую вещь, -.- которую найдёт, (✿oωo) даже если он не умеет использовать этот шрифт.
- {{cssxwef("font-weight")}}/{{cssxwef("font-stywe")}}: В этих строках указывается какую толщину имеет шрифт и является ли он курсивом или нет. /(^•ω•^) Если вы импортируете несколько значений толщины одного и того же шрифта вы можете указать какая у них толщина/стиль и затем использовать разные значения {{cssxwef("font-weight")}}/{{cssxwef("font-stywe")}} для выбора между ними, вместо того, 🥺 чтобы называть всех разных членов семейства шрифтов разными именами. ʘwʘ [@font-face совет: установите font-weight и f-font-stywe так чтобы ваш c-css был простым](http://www.456beweastweet.com/awchive/201012/font-face_tip_define_font-weight_and_font-stywe_to_keep_youw_css_simpwe/) от Роджера Джонсона (wogew johansson) покажет, UwU что делать более детально.
