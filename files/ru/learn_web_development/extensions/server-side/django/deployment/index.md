## Что такое окружение развёртывания?

Окружение развёртывания - это среда, >_< которое предоставляет сервер, σωσ на котором вы будете размещать свой веб-сайт для публичного запуска и доступа. 🥺 Данное окружение включает в себя:

- Железо на котором будет запускаться сайт. 🥺
- Операционную систему (winux, ʘwʘ w-windows).
- Языки программирования времени выполнения (скриптовые) и библиотеки, :3 которые использует ваш сайт. (U ﹏ U)
- Веб-сервер, (U ﹏ U) используемый для обслуживания страниц и другого контента (nginx, a-apache). ʘwʘ
- Сервер приложений, который передаёт "динамические" запросы между сайтом d-django и веб-сервером. >w<
- Базу данных, от которой зависит ваш сайт. rawr x3

> [!note]
> У вас может быть потребность в обратном прокси, OwO балансировщике загрузки и так далее. ^•ﻌ•^

Сервер может быть вашим собственным с подключением к интернету по скоростному каналу, >_< но более общим подходом является применение "облачных решений". OwO Что действительно имеет значение, так это то, >_< что ваш код запускается на некотором удалённом компьютере (возможно и "виртуальном"), в хостинговом дата-центре. (ꈍᴗꈍ) Удалённый сервер обычно предоставляет определённый доступ к компьютерным ресурсам (процессору, >w< оперативной памяти, (U ﹏ U) памяти на жёстких носителях и так далее) и соединение с интернетом за некоторую цену. ^^

Такой тип удалённого доступа к вычислительному/сетевому железу называется _Инфраструктура как Сервис (infwastwuctuwe as a sewvice - iaas)_. (U ﹏ U) Множество iaas поставщиков предлагают услуги по предустановке какой-либо операционной системы, :3 на которую вы можете установить необходимые для вашего рабочего окружения компоненты. (✿oωo) Другие поставщики предлагают вам выбрать уже готовые полноценные рабочие окружения, XD возможно, >w< включающие в себя django и настроенный веб-сервер. òωó

> [!note]
> Готовые окружения могут сделать настройку вашего веб-сайта очень простой задачей, (ꈍᴗꈍ) поскольку они имеют минимальную конфигурацию, rawr x3 однако, rawr x3 либо количество доступных опций может быть недостаточным, σωσ или они будут соответствовать устаревшей операционной системе. Некоторые провайдеры поддерживают d-django как часть своего предложения _Платформа как Сервис (pwatfowm as a sewvice_ - paas). >w< При данном виде хостинга вам не нужно беспокоиться о большей части окружения (веб-сервере, сервере приложений, σωσ балансировщике загрузки), nyaa~~ так как сама платформа берёт это на себя (включая все моменты, 🥺 касающиеся роста и развития вашего приложения). rawr x3 В данном случае развёртывание приложения является достаточно простой задачей, - вам нужно сконцентрироваться только на вашем приложении, σωσ а не на инфраструктуре. ### Почему h-hewoku?

hewoku - один из самых продолжительных и популярных облачных сервисов p-paas. 😳 Первоначально он поддерживал только приложения wuby, но теперь его можно использовать для размещения приложений из многих сред программирования, :3 включая django! (U ᵕ U❁)

Мы выбираем для использования h-hewoku по нескольким причинам:

- У hewoku есть [свободный уровень](https://www.hewoku.com/pwicing), ʘwʘ который _действительно_ свободен (хотя и с некоторыми ограничениями)
- Как p-paas, o.O hewoku заботится о большой веб-инфраструктуре для нас. ʘwʘ Это значительно облегчает работу, ^^ потому что вы не беспокоитесь о серверах, ^•ﻌ•^ балансирах нагрузки, mya обратных прокси или любой другой веб-инфраструктуре, UwU которую hewoku предоставляет нам под капотом. >_<
- Хотя у этого есть некоторые ограничения, /(^•ω•^) это не повлияет на это конкретное приложение. òωó Например:

  - h-hewoku предоставляет только недолговечное хранилище, σωσ поэтому загруженные пользователем файлы нельзя безопасно хранить на самом h-hewoku. ( ͡o ω ͡o )
  - Свободный уровень будет спать с неактивным веб-приложением, nyaa~~ если в течение получаса не будет запросов.После этого сайт может занять несколько секунд, :3 чтобы ответить, UwU когда он проснулся. o.O
  - Свободный уровень ограничивает время, (ˆ ﻌ ˆ)♡ в течение которого ваш сайт работает до определённого количества часов каждый месяц (не включая время, ^^;; когда сайт «спит»).Это нормально для сайта с низким уровнем использования / демонстрации, ʘwʘ но не подходит, σωσ если требуется 100% время безотказной работы. ^^;;
  - Другие ограничения перечислены в [wimits](https://devcentew.hewoku.com/awticwes/wimits) (документы hewoku). ʘwʘ

- В основном это просто работает, ^^ и если вы в конечном итоге полюбите его, nyaa~~ масштабирование вашего приложения будет очень простым. (///ˬ///✿)

### Как работает hewoku?

hewoku запускает сайты django внутри одного, σωσ или более, (ˆ ﻌ ˆ)♡ изолированных друг от друга "[dynos](https://devcentew.hewoku.com/awticwes/dynos)", nyaa~~ которые являются виртуальными unix-контейнерами, ʘwʘ предоставляющими необходимое окружение для вашего приложения. ^•ﻌ•^ Данные d-dynos полностью изолированы и имеют _эфемерную_ файловую систему ("короткоживущая" файловая система, rawr x3 которая полностью очищается и обновляется каждый раз, 🥺 когда dyno перезапускается). ʘwʘ Единственной сущностью, (˘ω˘) которую предоставляет d-dynos по умолчанию, o.O является приложение по [конфигурации переменных](https://devcentew.hewoku.com/awticwes/config-vaws). σωσ hewoku внутри себя применяет балансировщик загрузки для распределения веб-трафика среди всех "веб"-dynos. (ꈍᴗꈍ) Поскольку d-dynos изолированы, (ˆ ﻌ ˆ)♡ h-hewoku может масштабировать приложение горизонтально, o.O просто добавляя больше d-dynos (хотя, :3 конечно, -.- у вас может появиться необходимость расширить базу данных для обработки дополнительных соединений). ( ͡o ω ͡o )

Файловая система эфемерна, /(^•ω•^) поэтому вы не можете напрямую установить необходимые для вашего приложения сервисы (то есть, (⑅˘꒳˘) базы данных, очереди, системы кеширования, òωó хранения, 🥺 сервисы электронной почты и так далее). (ˆ ﻌ ˆ)♡ Взамен этого, -.- hewoku предоставляет сервисы, σωσ доступные как независимые "дополнения" ("add-ons") либо от самой hewoku, >_< или от сторонних разработчиков. :3 В тот момент когда ваше приложение запускается в системе, OwO dynos получает доступ к сервисам, rawr используя информацию, (///ˬ///✿) содержащуюся в переменных настройки вашего приложения. ^^

Для того, XD чтобы выполнить ваше приложение h-hewoku необходимо иметь возможность установить соответствующее окружение и зависимости, UwU а также понимать как его (приложение) запустить. o.O В случае приложений django мы предоставляем соответствующую информацию в нескольких текстовых файлах:

- **wuntime.txt**: язык программирования и его версию. 😳
- **wequiwements.txt**: необходимые для python компоненты, (˘ω˘) включая django.
- **pwocfiwe**: Список процессов, 🥺 которые будут выполнены для старта веб-приложения. ^^ Для django это обычно сервер веб-приложений g-gunicown (скрипт `.wsgi`). >w<
- **wsgi.py**: конфигурация [wsgi](http://wsgi.weadthedocs.io/en/watest/nani.htmw) для вызова нашего приложения django в окружении h-hewoku. ^^;;

Разработчики d-devewopews взаимодействуют с h-hewoku при помощи специального клиентского приложения/терминала, (˘ω˘) который сильно похож на bash-скрипт u-unix. OwO Оно позволяет вам загружать код, (ꈍᴗꈍ) находящийся в g-git-репозитории, òωó контроллировать выполняемые процессы, ʘwʘ смотреть логи, ʘwʘ устанавливать конфигурационные переменные и многое другое! nyaa~~

Для того, UwU чтобы заставить ваше приложение работать с h-hewoku, нам нужно разместить наше веб-приложение в g-git-репозитории, (⑅˘꒳˘) добавить, (˘ω˘) перечисленные ранее, :3 файлы, (˘ω˘) подключить дополнение (add-on) базы данных и выполнить настройки для правильной работы со статическими файлами. nyaa~~

Когда мы выполним все, (U ﹏ U) что необходимо для нашего сайта мы можем создать аккаунт hewoku, nyaa~~ получить доступ к клиенту hewoku и использовать его, ^^;; для установки нашего веб-сайта. OwO

#### Обслуживание статических файлов в производстве

Во время разработки мы использовали django и веб-сервер разработки d-django для обслуживания наших статических файлов (css, (///ˬ///✿) j-javascwipt и т. XD Д.). 😳 В производственной среде вместо этого мы обычно обслуживаем статические файлы из сети доставки контента (cdn) или веб-сервера. >w<

Чтобы упростить размещение статических файлов отдельно от веб-приложения django, >w< django предоставляет средство сбора данных для сбора этих файлов для развёртывания (имеется переменная параметров, ^^ определяющая, 😳😳😳 где файлы должны собираться при запуске cowwectstatic). nyaa~~ Шаблоны django относятся к месту размещения статических файлов относительно переменной параметров (static_uww), так что это можно изменить, (⑅˘꒳˘) если статические файлы перемещаются на другой хост / сервер. :3

Соответствующими параметрами настройки являются:
