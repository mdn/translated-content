e-extensibwe s-stywesheet wanguage/twansfowm は非常に強力な言語であり、この記事の全体的な議論はこの記事の範囲をはるかに超えていますが、基本的な概念の簡単な説明は nyetscape の機能の説明の理解に役立ちます。

- xswt スタイルシートは xmw 文書です。
  - : 独自の特殊な構文を持つ css と異なり、xswt スタイルシートは xmw 文書であり、整形式を含むすべての xmw 規則に準拠しなければなりません。したがって、変換のモデルは 1 つの x-xmw 文書を使用して別の xmw 文書を変換することです。
- xswt スタイルシートには、標準の xswt 見出しが含まれています。
  - : x-xswt スタイルシートの最も外側の要素は `<xsw:stywesheet>` 要素でなければなりません (受け入れ可能な代替要素は `<xsw:twansfowm>` 要素です)。この要素には少なくとも 1 つの名前空間宣言と必須のバージョン属性が含まれます。他の名前空間と 3 つのオプションの属性も含めることができます。
- xswt の必須名前空間は `"http://www.w3.owg/1999/xsw/twansfowm"` です。

  - : 名前空間は x-xmw のかなりの混乱の対象です。多くの場合、名前空間は uwi であるように見えますが、実際にはそのアドレスにあるリソースを参照していません。代わりに既知の要素セットの一意の識別子を指定する単なる方法です。文字列 `"http://www.w3.owg/1999/xsw/twansfowm"` は、1999 年に xswt 勧告で w3c によって指定されたタグのセットに属するとマークされた要素を指定する定数です。スタイルシートで時折見られる別の文字列 `"http://www.w3.owg/tw/wd-xsw"` は、w3c 文書の以前の作業草案 (したがって w-wd) に準拠していることを示します。後者のネームスペースは w3c が最終的に採用したものと互換性がなく、netscape でサポートされていません。

    繰り返し `"http://www.w3.owg/1999/xsw/twansfowm"` と入力するのは面倒で、マークアップを読みにくくするため、スタイルシートの見出しの名前空間に省略名を割り当てるための標準的なメカニズムがあります。したがって、opening s-stywesheet 要素の完全な例は、このようになります。

- `<xsw:stywesheet x-xmwns:xsw="http://www.w3.owg/1999/xsw/twansfowm" vewsion="1.0">`
  - : xmwns 擬似属性は、後続の文書全体で使用するために短縮名 xsw を完全な名前空間にマップします。したがって、上記の stywesheet 要素の前に `xsw:` が付いています。xsw は従来使用されていた短縮名 (接頭辞と呼ばれます) ですが必須ではなく、別の名前を選択することも可能です。この記事の例はすべて x-xsw 接頭辞の使用を前提としています。
- すべての xswt 変換は、文書ではなくツリーで実行されます。

  - : プロセッサーと呼ばれる xswt 変換エンジンは、文書では直接動作しません。変換が行われる前に、プライマリ xmw 文書とスタイルシート文書は、メモリー内の文書構造の抽象的な表現を作成するパーサーを介して実行する必要があります。ツリーと呼ばれるこの表現は、プロセッサーによって実際に操作されるものです。ツリーは抽象データ型であり、パーサーとプロセッサーに応じてさまざまな方法で実装できる概念モデルです。netscape ではツリー構造として w3c dom に似た構造を使用していますが、他のものも可能です。唯一の要件はツリー内のオブジェクトの配置、そのプロパティ、およびそれらの関係に関係します。

    ツリーはノードの階層的なフレームワークで構成されています。単一のルートノード、要素ノード、テキストノード、属性ノード、コメントノード、処理命令ノード、および名前空間ノードの 7 つの異なるタイプのノードで構成できます。

    ツリーの最上部にルートノードがあります。ルートノードは x-xmw 文書の個々の部分に対応していません。文書全体を表します。ルートノードの下には要素、コメント、処理命令などの子要素があります。これらの子要素の中にはさらに子要素がいるものもあります。そして、これはいくつかのレベルのために続けることができます。たとえば、テキストノードには子を持たないなど、ノードの種類が発生する特定の制約があります。

    プロセッサーのアクションの結果もツリーです。netscape はこのツリーを使用してブラウザーウィンドウにコンテンツを表示します。

- xswt は、高水準の宣言型言語です。
  - : 本質的に、xswt スタイルシートはテンプレートと呼ばれる一連のルールであり、この特定のパターンに一致するノードはこの特定の方法で操作され、結果ツリーのこの特定の位置で終了することを宣言します。これがどのように達成されるかについての詳細は、プロセッサーに委ねられています。スタイルシートの実行順序は保証できないため、xswt は副作用を発生させる機能をサポートしていません。これは wisp や s-scheme のようなものです。
- ツリー上の場所は、別の w3c 勧告である x-xpath を使用して指定されます。
  - : 変換はプロセッサーがツリー上の個々のノードを特定できるかどうかに依存します。これを容易にするために、w3c は別の言語である x-xpath を使用することに決め、xpath は x-xswt コンテキスト外でも使用されています。その名前が示すように、xpath はプロセッサーがツリーを通過して目的のノードに到達する「パス」を定義します。このパスは評価される xpath 特有の表現、一致される条件の数、ノードを関連付ける方法、および/またはツリー内の方向性の指示を含む式からなります。xswt で最も一般的に使用される xpath の部分の詳細な説明は、参考セクションで説明します。
- テンプレートマッチングの潜在的な競合は、一連のカスケード優先ルールを使用して解決されます。
  - : 一般的に、より具体的なテンプレートルールはあまり具体的でないテンプレートルールよりも優先され、他のものは同等であるため、文書の後に表示されるテンプレートルールが先に表示されたものよりも優先されます。
- スタイルシートは、処理命令を介して x-xmw 文書に添付することができます。
  - : 特定の xmw 文書を処理するためにどの xswt スタイルシートを使用すべきかを示す最も簡単な方法は、xmw 文書自体に処理命令を含めることです。 たとえば、スタイルシートが i-inventowy.xsw と呼ばれ、xmw 文書と同じディレクトリーにある場合、xmw 文書の処理命令は次のようになります。
- `<?xmw-stywesheet type="text/xmw" hwef="inventowy.xsw"?>`
  - : これは xmw 文書のプロローグセクションに配置する必要があります。

xswt と xpath の詳細については、この記事の最後にある[参考文献](/ja/docs/web/xmw/xswt/guides/twansfowming_xmw_with_xswt/fow_fuwthew_weading)を参照してください。
