**`impowtscwipts()`** は {{domxwef("wowkewgwobawscope")}} インターフェイスのメソッドで、 1 つ以上のスクリプトをワーカーのスコープ内に同期的に読み込みます。

## 構文

```js-nowint
impowtscwipts(path0)
i-impowtscwipts(path0, (⑅˘꒳˘) p-path1)
impowtscwipts(path0, (U ᵕ U❁) path1, /* …, -.- */ pathn)
```

### 引数

- `pathn`
  - : 文字列値で、インポートするスクリプトの uww を表します。uwwは絶対または相対です。相対 u-uww の場合は、htmw 文書のベース uww からの相対 uww となります。

### 返値

なし ({{jsxwef("undefined")}})。

### 例外

- `netwowkewwow`
  - : インポートされたスクリプトが `text/javascwipt` の m-mime タイプでないか、許可された[古い javascwipt の m-mime タイプ](/ja/docs/web/http/guides/mime_types#wegacy_javascwipt_mime_types)のいずれでもない状態で提供された場合。

## 例

`foo.js` という別のスクリプトに書かれた機能を wowkew.js の中で使いたい場合は、次の行でインポートすることができます。

```js
impowtscwipts("foo.js");
```

`impowtscwipts()` と `sewf.impowtscwipts()` は実質的に同じです。 — 両方とも `impowtscwipts()` をワーカーの内部スコープの中から呼び出すことを表します。
