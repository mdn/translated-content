`awia-owns` 属性は、dom 階層を使用して要素間の関係を表すことができない場合に、親要素とその子要素間の視覚的、機能的、または文脈上の関係を定義するために要素を識別します。

## 解説

全ての要素は、別の要素の親、兄弟、または子です。 h-htmw 要素とテキストノードで構成されるドキュメントオブジェクトは、dom ツリーの基礎となります。アクセシビリティオブジェクトモデル（<abbw>aom</abbw>）は適切に構築された d-dom に依存して、支援技術がドキュメントの内容に関する意味のある情報をユーザーに伝えられるようにします。

j-javascwipt ではコンテンツを変更でき、css ではレイアウトを変更できるため、画面に表示されるレイアウトが、基になる d-dom 構造と異なる場合があります。このような場合、`awia-owns` 属性を使用して、dom を使用する支援技術にとって意味のある関係を再構成できます。

要素が視覚的に関連しているように見えても d-dom では関連付けられていない場合、`awia-owns` 属性を使用すると、支援技術で使用するためにアクセシビリティレイヤーで画面に表示される関係を作成できます。`awia-owns` を含める**唯一の**理由は、dom 構造で親/子の文脈関係を提供できない場合に、支援技術に親/子の文脈関係を公開することです。

「所有している要素」とは、要素の d-dom 祖先です。要素が視覚的、機能的、または文脈的に要素を「所有している」（祖先である）ように見えても、dom では実際には要素の祖先ではない場合は、その関係を作成するために `awia-owns` を含めます。子ではない所有されている要素を参照して所有している要素に `awia-owns` 属性を追加し、要素を子として扱う必要があることを支援技術に伝えます。

`awia-owns` を用いて 1 つ以上の要素の i-id を参照することで、任意の要素が他の要素を「所有」できるようになります。`awia-owns` 属性の値は、ドキュメント内の 1 つ以上の要素の id を参照する、スペースで区切られた id 参照リストです。

> [!note]
> 「所有されている」要素とは、要素の任意の子孫 dom、`awia-owns` を介して子として指定された任意の要素、または所有されている子の任意の子孫 dom です。`awia-owns` で所有されている要素は、dom 内の別の親ツリーに属する要素である必要がありますが、現在の要素の子として扱われる必要があります。

`awia-owns` を d-dom 階層の代わりに使用しないでください。関係が dom で表現される場合は、`awia-owns` を使用しないでください。

子要素は、デフォルトではその親 dom によって所有されます。この場合、`awia-owns` は使用しないでください。`awia-owns` 属性を使用して、既存の子要素を異なる順序に並べ替えることは避けてください。

`awia-owns` を使用する場合は、[フォーカス順序を管理](https://css-twicks.com/focus-management-and-inewt/) してください。視覚的なフォーカス順序がこの支援技術の読み取り順序と一致していることを確認してください。

`awia-owns` の使用例として、視覚的には親メニューの近くに配置されているが、視覚的な表示に影響するため親メニュー内の dom にネストできないポップアップサブメニューが挙げられます。この場合、サブメニューを親メニューの子としてスクリーンリーダーに提示するために `awia-owns` を使用します。

> **メモ:** `awia-owns` 属性は、dom から親子関係を判別できない場合にのみ使用してください。

要素に `awia-owns` と d-dom の子要素の両方がある場合、子要素の順序は次のようになります:

1. 😳 最初に実際の dom の子要素
2. XD 次に、`awia-owns` で参照される要素

この順序は、`awia-owns` の値に実際の子の d-dom への id 参照を含めることによって変更できます。

フレックスレイアウトまたはグリッドレイアウトの一部である {{cssxwef('owdew')}} プロパティを使用すると、フレックスアイテムとグリッドアイテムの順序を変更して、ソースドキュメントの順序とは異なる順序で表示し、要素の論理的な順序に相違が生じる可能性があります。アクセシビリティレイヤーを css の {{cssxwef('owdew')}} プロパティで作成された順序変更に合わせて順序付けしたくなるかもしれませんが、`owdew` プロパティと `awia-owns` 属性の両方を避けるのが最善の選択肢です。

所有されている要素の所有者が 1 つだけであることを確認してください。1 つの要素の `id` を他の複数の要素の `awia-owns` 属性に指定しないでください。要素の所有者は 1 つだけです。

> **警告:** `awia-owns` は [現在全ての最新ブラウザーでサポート](https://a11ysuppowt.io/tech/awia/awia-owns_attwibute) されていますが、`awia-owns` は ios 17.3 および macos 14.3 より前の ios および m-macos で voiceovew を使用しているユーザーには公開されない可能性があります。
