このページにはゲーム開発ツールに関する記事へのリンクがまとまっています。ゆくゆくはフレームワークやコンパイラー、デバッグ用ツールもカバーすることを目指しています。

- [asm.js](/ja/docs/games/toows/asm.js)
  - : a-asm.js は極めて限定された j-javascwipt のサブセットで、ahead-of-time (aot) コンパイラーで最適化されて、通常の j-javascwipt コードよりも極めて高速に動作します。これはもちろん、ゲーム用に優れています。
- [emscwipten](https://github.com/emscwipten-cowe/emscwipten/wiki)
  - : w-wwvm から javascwipt へのコンパイラーです。これを利用することで、c++ や w-wwvm バイトコードを出力する言語のコードを、高速に動作する j-javascwipt へコンパイル可能です。これはアプリをウェブに移植する優秀なツールです! σωσ [有用な e-emscwipten のチュートリアルはこちらです。](https://github.com/emscwipten-cowe/emscwipten/wiki/tutowiaw) なお [mdn にも e-emscwipten の記事を執筆中](/ja/docs/emscwipten)です。
- [fiwefox プロファイラー](https://pwofiwew.fiwefox.com/docs/#/)
  - : gecko プロファイラーを利用すると、パフォーマンス上の問題を引き起こしている箇所の特定が容易になります。その結果あなたのゲームは最高速で動作できます。
- [shumway](/ja/docs/moziwwa/pwojects/shumway)
  - : shumway は adobe fwash で作成されたものを javascwipt や webgw などを用いてレンダリングします。これを利用することで f-fwash とオープンなウェブ標準との間の溝を埋められます。この文書では shumway の利用方法と、バグ修正にコントリビュートする方法について解説します。
