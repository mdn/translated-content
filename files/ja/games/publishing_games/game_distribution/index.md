[チュートリアル](/ja/docs/Games/Tutorials/2D_Breakout_game_pure_JavaScript)を一つまたは[二つ](/ja/docs/Games/Tutorials/2D_breakout_game_Phaser)実行し、HTML ゲームを作成しました。素晴らしいですね！この記事では、新しく作成したゲームを世界に配信する方法について知っておく必要があるすべてのことを説明します。これには、オンラインで自らホストすること、オープンなマーケットプレイスに提出すること、Google Play や iOS App Store などの閉じられたマーケットプレイスに提出することが含まれます。

## ネイティブに対する HTML の利点

HTML でゲームを構築すると、次のような追加の利点があります。

### マルチプラットフォームの至福

この技術自体がマルチプラットフォームなので、コードを一度書けば複数のデバイスをターゲットにできます。これは、ローエンドのスマートフォンやタブレットから、ノートパソコンやデスクトップコンピューター、スマート TV、時計、さらには最新のブラウザーを処理できる冷蔵庫まで、幅広く対応できます。

単一のコードベースだけを心配すればよいので、異なるプラットフォームをターゲットにする同じタイトルに取り組む別々のチームを持つ必要はありません。[宣伝](/ja/docs/Games/Publishing_games/Game_promotion)と[収益化](/ja/docs/Games/Publishing_games/Game_monetization)により多くの時間とお金を費やすことができます。

### 即座の更新

ゲームのコードを更新するのに何日も待つ必要はありません。ユーザーがバグを見つけた場合、すぐに修正し、システムを更新し、サーバー上のゲームを更新すれば、ほぼ即座に更新されたコードをプレイヤーに提供できます。

### 直接リンク配信と即座のプレイ

HTML ゲームでは、アプリストアでゲームを検索するように人々に伝える必要はありません。ゲームにアクセスするための直接 URL を送信するだけで、クリックしてサードパーティのプラグインを使用したり、大きなパッケージをダウンロードしてインストールしたりすることなく、すぐにゲームをプレイできます。ゲームのサイズとネットワーク速度によっては、ゲームのダウンロードに少し時間がかかる場合があることに注意してください。いずれにせよ、ゲームを宣伝することがはるかに簡単になります。トラフィックを直接希望する場所に誘導でき、プレイするために多くの障害を乗り越える必要がないからです。

## デスクトップ対モバイル

私たちが興味を持っているトラフィックの大部分、つまり HTML ゲームをプレイする人々は、モバイルデバイスから来ています。本当に成功したいのであれば、これに焦点を当てる必要があります。モバイルデバイスは、HTML 技術が真に輝き、その利点を示すことができる場所です。Flash がない、HTML は完全にマルチプラットフォームです。

デスクトップゲームと直接競合しようとするのは非常に困難です。HTML ゲームを同じ土俵に置くこともできます（後述の[ネイティブデスクトップ](#ネイティブデスクトップ)を参照）。サポートするプラットフォームを多様化することは良いことですが、デスクトップゲームを作成する開発者には長年の経験、優れたツール、安定した配信チャネルがあることを覚えておく必要があります。多くの HTML ゲームは、ネイティブデスクトップゲームとは異なる市場セグメントをターゲットにします。例えば、移動中にプレイするシンプルな暇つぶしゲームであり、巨大な没入型体験ではありません。このようなゲームは、多くの場合、2 本の指、あるいは 1 本の指でプレイできるように設計されているため、デバイスを持ち、ゲームをプレイし、現在必要なものに 2 本目の手を使用できます。

とはいえ、デスクトッププラットフォームは、ゲームのネイティブビルドを準備するのに役立つラッパーが利用できるため、配信に非常に簡単に使用できます。[ゲームのパッケージ化](#ゲームのパッケージ化)を参照してください。また、主にモバイルをターゲットにしている場合でも、ゲームにデスクトップコントロールを提供することは良いことです。プレイヤーは利用可能なあらゆるプラットフォームでゲームを楽しんでおり、デスクトップもその 1 つです。さらに、通常、デスクトップでゲームを最初に構築してテストし、次にモバイルのデバッグに移行する方が簡単です。

## ゲームの公開

ゲームを公開する場合、主に 3 つのオプションがあります。

- セルフホスティング
- パブリッシャー
- ストア

ゲームの名前は、後で簡単に[宣伝](/ja/docs/Games/Publishing_games/Game_promotion)できるように十分にユニークでありながら、人々が忘れないようにキャッチーであることを忘れないでください。

### セルフホスティング

フロントエンド開発者であれば、何をすべきかすでに知っているかもしれません。HTML ゲームは別のウェブサイトにすぎません。リモートサーバーにアップロードし、キャッチーなドメイン名を取得して、自分でホストできます。

ゲーム開発でお金を稼ぎたい場合は、何らかの方法でソースコードを保護し、簡単に取得して自分のものとして販売できる人々から守る必要があります。コードを連結して縮小してサイズを小さくし、難読化してリバースエンジニアリングをはるかに困難にすることができます。iTunes や Steam などの閉じられたストアでパッケージ化して販売する予定がある場合は、オンラインデモを提供することも良い対策です。

サイドプロジェクトとして楽しみのために作業している場合は、ソースをオープンにしておくことで、作成したものから学びたい人々に利益をもたらします。ホスティングプロバイダーを探すことを心配する必要さえありません。[GitHub Pages でゲームをホストする](https://end3r.com/blog/host-your-html5-games-on-github-pages)ことが可能だからです。無料のホスティング、バージョン管理、プロジェクトが十分に興味深い場合は貢献者の可能性を得られます。

### パブリッシャーとポータル

名前が示すように、パブリッシャーはゲームの公開を代行してくれます。この道を進むべきかどうかは、ゲームを配信する計画によって異なります。可能な限りどこにでも送信しますか、それとも[独占ライセンス](/ja/docs/Games/Publishing_games/Game_monetization)を購入した人々に存在を制限しますか？それはあなた次第です。さまざまなオプションを検討し、実験し、結論を出してください。パブリッシャーについては、[収益化](/ja/docs/Games/Publishing_games/Game_monetization)の記事で詳しく説明します。

また、[HTML5Games.com](https://html5games.com/)、[GameArter.com](https://www.gamearter.com/)、[MarketJS.com](https://www.marketjs.com/)、[GameFlare](https://distribution.gameflare.com/)、[GameDistribution.com](https://gamedistribution.com/)、[GameSaturn.com](https://gamesaturn.com/)、[Playmox.com](https://www.playmox.com/)、[Poki](https://developers.poki.com/)、[CrazyGames](https://developer.crazygames.com/) などの興味深いゲームを収集する独立したポータルもあります。これらのサイトにゲームを送信すると、これらのサイトが引き付ける大きなトラフィックのおかげで、自然な宣伝を受けることができます。これらの一部はファイルを取得してサーバーにホストしますが、他の一部はウェブサイトにリンクするか、サイトにゲームを埋め込むだけです。このような露出は、ゲームの[宣伝](/ja/docs/Games/Publishing_games/Game_promotion)を提供するだけかもしれませんし、ゲームの横に広告を表示する場合（または他の収益化オプション）、収益化も提供する可能性があります。

### ウェブおよびネイティブストア

ゲームをさまざまな種類のストアやマーケットプレイスに直接アップロードして公開することもできます。そのためには、ターゲットにする各アプリエコシステムに固有のビルド形式にゲームを準備してパッケージ化する必要があります。利用可能なマーケットプレイスの種類の詳細については、[マーケットプレイス — 配信プラットフォーム](#マーケットプレイス_—_配信プラットフォーム)を参照してください。

## マーケットプレイス — 配信プラットフォーム

さまざまなプラットフォームやオペレーティングシステムで利用可能なマーケットプレイス/ストアに関する利用可能なオプションを見てみましょう。

> [!NOTE]
> これらは最も人気のある配信プラットフォームですが、これらが唯一のオプションであるとは言えません。例えば iOS ストアに何千ものゲームの中にゲームを追加しようとする代わりに、ニッチを見つけて、ゲームに興味を持つ聴衆に直接宣伝することもできます。ここではあなたの創造性が重要です。

### ウェブストア

HTML ゲームに最適なプラットフォームは、ウェブベースのストアです。マニフェストファイルやリソースなどのデータを zip パッケージに追加して、ウェブストア用にゲームを準備できます。ゲーム自体の変更はあまり必要ありません。

- [Chrome Web Store](https://chromewebstore.google.com/) も魅力的なオプションです。マニフェストファイルを準備し、ゲームを zip 圧縮し、オンライン送信フォームに記入するだけで済みます。

### ネイティブモバイルストア

モバイル市場に関しては、iOS 用の Apple App Store、Android 用の Google Play、その他すべての競合があります。ネイティブストアはすでに優れたゲームを販売する確立された開発者でいっぱいなので、注目されるには才能と運が必要です。

- iOS App Store は、厳格な要件があるため、参入が非常に困難です。ゲームが満たさなければならない厳格な要件があり、承認されるまで 1 週間か 2 週間待つ必要があります。さらに、何十万ものアプリがある最も著名なモバイルストアなので、群衆から目立つことは非常に困難です。
- Google Play の要件はそれほど厳格ではないため、ストアは低品質のゲームで汚染されています。それでも、毎日送信されるアプリの数が膨大であるため、そこで注目されることは非常に困難です。ここでお金を稼ぐのも難しいです。iOS の有料ゲームのほとんどは、Android では無料ゲームとして公開されており、アプリ内購入（IAP）または広告から収益化されています。
- Windows Phone や Blackberry などのネイティブモバイルプラットフォーム用の他のストアは、ケーキの一部を獲得しようと懸命に取り組んでおり、競合他社に大きく遅れをとっています。そこにゲームを提出することは良いことかもしれません。注目される方がはるかに簡単だからです。

さまざまな種類のアプリストアに関する詳細情報を探している場合は、Wikipedia の[モバイルソフトウェア配信プラットフォームのリスト](https://en.wikipedia.org/wiki/List_of_mobile_software_distribution_platforms)の記事を確認できます。

### ネイティブデスクトップ

聴衆を広げるために、HTML ゲームでデスクトップエコシステムにも進出できます。ただし、市場シェアのほとんどを占める人気のある AAA ゲームをすべて覚えておき、これが戦略に適しているかどうかを慎重に検討してください。デスクトップを適切に行うには、3 つのオペレーティングシステム（Windows、macOS、Linux）すべてをサポートする必要があります。ゲーム用の最大のデスクトップストアは間違いなく [Steam](https://steamcommunity.com/) です。インディー開発者は [Steam Direct](https://partner.steamgames.com/steamdirect) プログラムを通じて Steam に参加できます。異なるプラットフォーム用の個別のバージョンをアップロードすることで、クロスプラットフォームの問題に自分で対処する必要があることを忘れないでください。

Steam をカバーした後、[Humble Bundle](https://www.humblebundle.com/) などのイニシアチブに関する多くの話題があります。最も人気のあるインディーゲームがより広い聴衆に提示されます。ただし、これは多くのお金を稼ぐ方法というよりも、優れた宣伝の機会です。バンドル内のゲームに支払われる価格は通常非常に低いからです。

## ゲームのパッケージ化

ウェブは HTML ゲームの最初で最良の選択肢ですが、より広い聴衆にリーチし、閉じられたエコシステムでゲームを配信したい場合は、パッケージ化することで実現できます。良い点は、さまざまなプラットフォーム用の同じゲームに取り組む複数の別々のチームが必要ないことです。一度構築し、ツールを使用してネイティブストア用にゲームをパッケージ化できます。結果のパッケージは通常非常に信頼性が高いですが、それでもテストし、修正すべき小さな問題やバグに注意する必要があります。

### 利用可能なツール

スキル、好みのフレームワーク、またはターゲットプラットフォームに応じて、さまざまなツールから選択できます。特定のタスクに最適なツールを選択することがすべてです。

- [Ejecta](https://impactjs.com/ejecta) — [ImpactJS](https://impactjs.com/) フレームワークで作成されたゲームを iOS 用にパッケージ化するための特別なツールで、ImpactJS の作者によって構築されました。ImpactJS とのシームレスな統合を提供しますが、1 つのフレームワークとアプリストアのみをサポートします。
- [NW.js](https://nwjs.io/) — 以前は Node-WebKit として知られていましたが、これは Windows、Mac、Linux で動作するデスクトップゲームを構築する場合の最初の選択肢です。配布は WebKit エンジンでパッケージ化され、あらゆるプラットフォームでレンダリングを提供します。

その他の代替ツールは次のとおりです。

- [Intel XDK](https://www.intel.com/content/www/us/en/developer/tools/overview.html) — CocoonIO に似たエキサイティングな代替手段です。
- [Electron](https://www.electronjs.org/) — Atom Shell として知られており、GitHub のオープンソースでクロスプラットフォームのツールです。
- [Manifold.js](https://www.manifoldjs.com/) — Microsoft チームのこのツールは、iOS、Android、Windows から HTML ゲームのネイティブ配布を作成できます。

## まとめ

配信は、世界があなたのゲームにアクセスできるようにする方法です。利用可能なオプションは多数あり、どれが最良かという単一の正解はありません。ゲームを公開したら、[宣伝](/ja/docs/Games/Publishing_games/Game_promotion)に焦点を当てる時が来ました。人々にゲームが存在することを知らせることです。宣伝がなければ、人々はそれについて学び、プレイすることさえできません。
