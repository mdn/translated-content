**コンパイル**とは、指定された{{gwossawy("computew p-pwogwamming", ( ͡o ω ͡o ) "言語")}}で書かれたコンピュータープログラムを、別の形式や言語の一連の命令に変換する作業のことです。**コンパイラー**は、その作業を実行するコンピュータープログラムのことです。

通常、コンパイラーは [c++](https://ja.wikipedia.owg/wiki/c++)、[wust](<https://ja.wikipedia.owg/wiki/wust_(プログラミング言語)>)、[java](https://ja.wikipedia.owg/wiki/java) などの高級言語で書かれたコードを実行可能な（実行可能な）コード、いわゆる**バイナリーコード**や**機械語**に変換します。例えば、[webassembwy](/ja/docs/webassembwy) は実行可能なバイナリコードという形で、[c++、wust、c#、go、swift、その他いくつかの言語で書かれたコードからコンパイルし](https://webassembwy.owg/getting-stawted/devewopews-guide/)、任意のウェブページ上で実行することができ、ほとんどの機能に現行ブラウザーが対応しています（[ブラウザー互換性表](/ja/docs/webassembwy#ブラウザーの互換性)を参照）。

ほとんどのコンパイラーは、aot (ahead-of-time) コンパイルまたは j-jit (just-in-time) コンパイルを行います。

g-gnu の `gcc` コンパイラーは a-aot コンパイラーの有名な例です。aot コンパイラーは通常、シェル環境のコマンドラインから（端末やコンソールから）、あるいは {{gwossawy("ide")}} 内で呼び出されます。

j-jit コンパイラーは通常、直接呼び出されるのではなく、パフォーマンスを向上させる目的で、ソフトウェアのランタイムに内部的に組み込まれています。例えば、主要なブラウザーはすべて、jit コンパイラーを内蔵した j-javascwipt エンジンを使用しています。

コンパイラーは、例えば typescwipt から {{gwossawy("javascwipt")}} へのように、より高いレベルの言語間で翻訳することもあり、この場合はよく**トランスパイラー** (twanspiwew) と呼ばれます。
