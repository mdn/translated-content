技術系の就職面接は、とても厳しいものであり、かなり特定の要件が求められる場合もあります。最初の段階で、応募すべき適切な仕事をを見つけること自体が難しい場合もあります。この記事では、有望な仕事を見つけ、応募し、面接で成功するためのヒントやリソースをご紹介します。下記に概要を示すプロセスは、すべての人にそのまま適用できるとは限りません。少し異なる手法や順序の方が効果的である場合もあります。しかし、ほとんどの方は、このプロセスが役立つと感じるでしょう。

## 希望する仕事の種類を決める

これを書き出すのはあまりにも当たり前すぎて意味がないように思えるかもしれませんが、就職活動を開始する前に、自分がどのような仕事をしていきたいかをよく考えておく必要があります。そうしないと、あまり効果のない散弾銃のような手法で、さまざまな仕事に応募し、どの仕事にも理想的な候補者とは見られないまま、結局終わってしまうでしょう。

このサイトを読んでいる方は、おそらく何らかのウェブ開発の仕事に興味をお持ちでしょう。それは素晴らしいことですが、もっと具体的に考えて、次のカテゴリーについて、自分が望むことをいくつか書き出してみてください。

- 的確な職種: あなたはどの職種をより希望しますか？
  - ウェブデザイナー: HTML、CSS、グラフィックパッケージを使用して、ユーザーインターフェイスを設計します。
  - JS 開発者: JavaScript およびフレームワークを使用して機能を実装します。
  - 開発者連携担当またはセールスエンジニア: お客様と話し合い、技術について期待を持ってもらい、問題解決やソリューションの開発を手伝います。
  - テクニカルライター: 技術がどのように動作するのか、その使用方法を説明する文書を作成します。
  - よりオールラウンドな開発者。
  - その他
- 雇用者の種類: どのような仕事を望みますか？
  - 自営の請負業者として、さまざまなプロジェクトに携わること。
  - デザイン/開発代理店、ここでもさまざまなプロジェクトに携わります。
  - 企業または法人の社内開発者として。
  - 非営利団体、または慈善団体。
- 分野：希望する仕事
  - 技術。
  - 健康。
  - 金融
  - 慈善活動。
  - 音楽産業。
  - その他。
- 勤務地：どこで働きたいか
  - 自宅からフルタイムで。
  - オフィスで。
  - ある種のハイブリッドな設定で。
  - 常に同じ場所でか、さまざまな地域や国を移動しながらか。

これらの質問に対する答えがわからない場合でも、問題はありません。同僚、友人、教師、指導者に、彼らが興味を持っていることを尋ねてインスピレーションを得たり、業界を調べて何が利用できるかを見ていくといいでしょう。現実的には、多くの皆さんは、興味深い求人広告や企業を見て、それが上記のカテゴリーにどのように当てはまるか、その企業に「自分を売り込む」方法を考え始めるでしょう。

## 自分を売り込む

就職するには、**自分を売り込む**方法を学ばなければなりません。つまり、自分が希望する仕事に適した候補者であることをアピールする必要があります。これにはいくつかのステップがありますが、その手順については以下の節で説明します。

### 雇用主が求める資質を身につける

一般的なポイントとして、採用担当者が候補者に求める資質を把握し、それに応じて準備しておく必要があります。当然のことながら、その職務を直接遂行するための技術的なスキルやその他の能力は必要です。しかし、企業は、職務の遂行を間接的に支援する「ソフトスキル」も求めています。また、従業員が職場でより働きやすく、より幸せに過ごせるようなスキルも重視しています。これらの点については、[コラボレーションとチームワーク](/ja/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork)の記事で既に触れています。

要約すると、採用担当者は以下の点を重視します。

- 一緒にやっていける人。
- 前向き、敬意、共感、建設的。
- 多様な視点を持つ多様なチームで、心を開いて作業すること。
- 自分たちの方法で物事が進まない場合でも、より大きな全体のために調整できる。
- コミュニケーション能力が高く、人間関係を構築する能力に長けていること。
- 粘り強く、集中力があり、問題解決能力に優れていること。

これらの記述が自分に当てはまるかどうか考えてみてください。これは、自分の弱点を特定し、改善するためのステップを踏む上で役立つかもしれません。例えば、コミュニケーションや問題解決が苦手なら、そのスキルを向上させるためのガイドや講座を探してみてください。友人たちに自分の仕事について話したり、プロジェクトについて説明したりといった、カジュアルなこともコミュニケーション能力の向上に役立ちます。

### 履歴書の作成

雇用主に自分の価値を伝える履歴書を作成しましょう。履歴書の書き方については、ウェブ上に多くのガイドがありますが、簡単に言えば、一般的に以下の内容を含める必要があります。

- 連絡先情報：お名前、ご住所、電話番号、メールアドレス、およびあなたにとって重要で、業務上の連絡に適した連絡先をもう1つ追加しましょう。 [LinkedIn のプロフィール]も登録することをお勧めします。
- 自分を一文で表現した要約/自己紹介文。これは書くのがとても難しいかもしれません。自信に満ちていながら傲慢にならないように、前向きでありながら現実的な内容にする必要があります。良い戦略としては、できるだけ短くし、事実、あなたが信じていること、そして現在のレベルに焦点を当てることです。例えば、若手ウェブ開発者の良い個人プロフィール文は、「ボブ・スミスは、2年の経験を持つウェブ開発者で、セマンティクス、アクセシビリティ、インクルーシブデザインに情熱を注いでいます」のようなものになるかもしれません。
- 職歴の一覧、それぞれについて以下を記載しましょう。
  - 在職期間を具体的に記入しましょう。例えば、「2023 年 6 月～ 2024 年 10 月」や、現在も在職中の場合は「2023 年 6 月～現在」と記入します。
  - 会社名および役職名。
  - その仕事やインターンシップなどで何をしたかを、短い文章または箇条書きで説明しましょう。すべてを説明する必要はありません。自分をアピールするために役立つと思う経験だけを選んでください。
- 学歴の概要。取得した学位、その学位を取得した学校、取得した成績を記載します。学位やその他の高等教育、資格、および自分の分野に直接関連のある実績（ウェブ開発認定資格やサマースクールなど）を含め、雇用主がしばしば重視する通常の学歴（英語、その他の言語、数学など）も記載しましょう。
- 次のような補足情報を含むセクション。
  - [ポートフォリオ](#ポートフォリオの作成)へのリンク。
  - 上記のセクションには該当しない、その他の関連経験について記述しましょう。
  - 個人的な趣味 — 多くの雇用主は、従業員が仕事以外の生活も豊かで、単なる会社のロボットではないことを好みます。音楽を演奏したり、映画を観たり、動物保護施設でボランティアをしたりなど、その内容は問いません。また、面接時の話題にも役立ちます。

履歴書の書き方に関する一般的なヒントをいくつかご紹介します。

- 簡潔にまとめましょう。A4用紙の2ページ以内に収まる程度が理想です。採用担当者は、あなたのスキルセットを長々と説明した文章を読む気はありません。
- 専門的な言葉を使用しましょう。会話調の言葉やスラングは使用しないでください。応募書類では、応募に真剣に臨んでいることを示す必要があります。
- 文章のスペルや文法ミスがないか慎重に確認しましょう。 [Grammarly](https://www.grammarly.com/) などのサービスを利用して、ミスを見逃さないよう確認し、さらに改善するためのヒントを得てください。採用担当者は、タイプミスを含む履歴書やレジュメは却下することがよくあります。タイプミスは、プロ意識の欠如や細部への注意力の欠如を示すからです。
- 履歴書や職務経歴書にすべての職歴を記載する必要はありません。職歴が豊富な場合は、最も重要または最近の職歴を記載しましょう。簡潔にまとめながら、採用担当者が疑問を抱くような不自然な職歴の空白が生じないようにしましょう。職歴の完全なリストは LinkedIn のプロフィールに記載し、履歴書や職務経歴書には要約のみ記載しましょう。
- ソフトスキルを含めましょう。「[雇用主が求める資質](#雇用主が求める資質を身につける)」の節で述べたように、ソフトスキルは重要であり、履歴書や職務経歴書に記載する価値があります。どのスキルが関連しているかをよく考えてください。雇用主は、あなたがどれほど素晴らしい人物であるかを何ページも記載した説明には興味を示しません。しかし、あなたがチームワークが得意で、プレッシャーに強く、問題解決能力があることは、彼らにとって興味深い情報となります。
- 推奨状（必要に応じて提出）。雇用主は、あなたやあなたのスキルについて第三者の意見を聞くために、必ず推薦状を求めます。理想的には、一緒に働いたことのある、良好な関係にある人、例えば以前の上司や教師などが望ましいです。現在、推薦者が誰にすべきか分からない場合は、履歴書/レジュメの最後に「推薦状はご要望に応じて提出いたします」と記載しましょう。
- 履歴書/職務経歴書を最新の状態に保ちましょう。毎年確認し、追加すべき内容や削除すべき内容がないか確認しましょう。

> [!NOTE]
> 履歴書/レジュメは一般的な書類として考え、応募する各職種に最も適した職務経験やスキルに合わせてカスタマイズしましょう。

### いくつか経験を得る

就職活動を始めると、悪循環に陥ってしまいます。「就職するには経験が必要ですが、経験を積むには就職しなければなりません」。しかし、絶望しないでください。経験を積む方法は他にもあります。以下にいくつかのアイデアをご紹介します。

- 地元の企業、慈善団体、教会、学校などに連絡して、ウェブサイトを作成したいかどうか、または既存のウェブサイトの更新を手伝ってほしいかどうかを確認しましょう。まだ経験が浅い場合は、経験を積むために無料で手伝うことを申し出てみてください。
- 前の項目に続き、自分の仕事に自信がつき、報酬を請求できるようになったら、小さなフリーランスの仕事を始めてみましょう。地元の企業に問い合わせたり、 [UpWork](https://www.upwork.com/) などの専門サービスを利用して仕事を探してみてください。
- 自分の趣味のプロジェクトを作成しましょう。例えば、自分の作品を紹介するウェブサイト、楽しいデモやゲーム、または働きたい会社の関連ウェブアプリなどです。
  - 会社のウェブサイト上の既存の問題を特定し、それを修正して、修正後のバージョンへのリンクを送信することもできます。
- フォーラムに参加して、他の人のコーディングの問題を助けましょう。 （「[他人を助け、質問する](/ja/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork#参加する、人々を支援する、質問をする)」を参照してください。）
- [オープンソースプロジェクトに協力する](/ja/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork#participate_in_open_source)。

### ポートフォリオの作成

話せるような関連経験がいくつか積めたら、次のステップは、履歴書やレジュメに添付して、雇用主に提示できるポートフォリオを作成することです。ここでのポイントは、「自分が何ができるかを説明するのではなく、実際に何ができるかを示す」ことです。

これには以下のようなものがあります。

- すべてを要約した見やすいランディングページ。
- 携わったウェブサイト、アプリ、その他の関連製品のリスト。それぞれについて、タイトル、サムネイル画像、プロジェクトの説明、およびあなたがそのプロジェクトで担当した業務内容を記載しましょう。プロジェクトについて詳しく説明したい場合は、プロジェクトごとに個別のページを作成しましょう。プロジェクトの開発に使用したテクノロジーのリスト、および可能であればソースコードへのリンクも記載しましょう（すべてのソースがオープンであるとは限りません）。
- 自分の個人的なプロジェクトや実験のソースコードやライブデモへのリンク。各プロジェクトには、その機能の説明と、その構築に使用した技術を含めてください。
- ブログ記事、執筆サンプル、会議での講演、その他、自分のスキルを示すのに役立つと思われる資産へのリンク。
- オープンソースプロジェクトへの協力の一覧。協力の内容、プロジェクトにどのような改善をもたらしたか、使用した技術などを記載してください。実際の変更内容（ソースコード、公開ドキュメントなど）へのリンク、および作業を行ったことを証明する変更履歴（GitHub のプルリクエストなど）へのリンクも記載してください。

ポートフォリオは、自分の最高の作品のハイライトとしてまとめましょう。アクセスしやすく、見やすいものにしてください。各セクションには、何百もの作品ではなく、いくつかの重要な作品を含めるようにしてください。そうしないと、採用担当者は飽きてしまい、次の作品に移ってしまうでしょう。すっきりとしたデザインは、見やすさを高めますが、複雑なデザインは読みづらくなる可能性があります。

ポートフォリオを実際に公開する方法については、以下の通りです。

- 作品は、ソースコードの場合は GitHub 組織、ブログ記事の場合は Medium などのブログプラットフォーム（または自分のウェブサイト）など、一般に公開されている場所に[公開しましょう](/ja/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork#publish_your_work)。
- ウェブサイト全体ではなく、デモやスニペットなどのソースコードの場合は、ライブバージョンのコードを公開して、他の人がライブの結果を簡単に確認できるようにしてください。その方法については、「[ウェブサイトの公開](/ja/docs/Learn_web_development/Getting_started/Your_first_website/Publishing_your_website)」を参照してください。
- ポートフォリオのランディングページについては、GitHub リポジトリーを作成し、`README.md` ファイルにリンク一覧を記載する方法もありますが、これでは見た目があまり良くありません。独自の[ドメイン名](/ja/docs/Learn_web_development/Howto/Web_mechanics/What_is_a_domain_name)を使用した、ポートフォリオ専用のウェブサイトを作成する方がはるかに良いでしょう。デザイン作業が得意でない場合は、同僚や友人に手伝ってもらうのも良いでしょう。その代わり、彼らのためにコーディング作業を手伝うなど、何か見返りを提案すると良いでしょう。

### LinkedIn プロフィールの作成

[LinkedIn](https://www.linkedin.com/) は、求職者が仕事を探すため、また雇用者が従業員を探すために世界で最も人気のあるサイトの 1 つです。自分の職歴やスキルに関するオンラインアカウントを掲載するのに最適な場所であり、履歴書や職務経歴書と似ていますが、異なる点もあります。

LinkedIn には、履歴書やレジュメと組み合わせてプロフィールを効果的に活用できる、多くの便利な機能があります。以下のことができます。

- プロフィールが長くなりすぎないように、自分の経験とスキルをすべて記載してください。
- 人々に推薦を投稿してもらいます。
- 過去の勤務先、教育機関などのプロフィールへのリンクを記載してください。
- 投稿を書いて、ブログとして使いましょう。
- 積極的に求人情報を検索して応募し、自分のスキルや興味に基づいて新しい求人情報を送信してもらうように設定してください。
- 人脈を作り、スキルや情報を共有できるネットワークを構築しましょう。これは最終的に就職活動に役立ちます。
- 研修コースを受講してください。

このような理由から、プロフィールを作成することをお勧めします。見栄えがよく、プロフェッショナルな印象を与えるように作成してください。[効果的なLinkedInプロフィールを作成する](https://www.linkedin.com/help/linkedin/answer/a554351/how-do-i-create-a-good-linkedin-profile-?lang=ja-JP)には、役立つヒントがあります。

## 求人への応募

それでは、就職の応募プロセスをみていきましょう。

### 仕事を探す

就職活動にはさまざまなアプローチがありますが、複数の方法を組み合わせることが最も効果的です。

- LinkedIn など、求人情報が掲載されているサイトに登録し（お住まいの地域に特化したサイトもあります）、定期的に求人情報を受け取るように設定します。
- [コミュニティネットワーク](/ja/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork#join_a_community_network)にアクセスし、求人広告を探して、知り合いに興味のある仕事がないか尋ねます。
- 働きたい企業を調査し、その企業の求人ページを確認して、募集中の求人を確認してください。
- ネットワーキングイベントや就職フェアに参加して、潜在的な雇用主と出会ってください。

以前に「[希望する仕事の種類を決める](#希望する仕事の種類を決める)」で作成した基準は、働きたい企業や応募したい求人のリストを絞り込む際に役立ちます。

### 応募前に

就職活動を開始する前に、準備として役立つ手順がいくつかあります。

- 求人情報を見て、その仕事が自分に適しているかどうかを確認してください。たとえば、英国に住んでいて、カリフォルニアのオフィスでフルタイムで働きたいと思っていないのに、その条件のある求人に応募しても時間の無駄になります。また、 JavaScript を 3 ヶ月間しか学んでいない場合、シニアウェブ開発者の仕事をする経験はありません。
  > [!NOTE]
  > 応募すべきかどうか迷っている場合や、その企業に他の適した求人があるかどうか知りたい場合は、採用担当者の連絡先を確認し、問い合わせてください。現在ではなくても、将来役立つ連絡先になる可能性があります。
- その会社で働いている人を探し、例えばコミュニティネットワークや LinkedIn で連絡を取ってみてください。その会社で働くことについて、その仕事をするために必要なこと、または近い将来に他の適切な求人があるかどうかを尋ねてみてください。同じ会社で働く多くの人々に連絡を取るのではなく、応募する役職に最も近いと思われる1～2人を選択してください。
- 会社についてさらに調査してください。ウェブサイトやブログを読み、歴史や目標について調べてください。製品を使用し、その仕組みに慣れるために時間を費やしてください。例えば、ソフトウェアをインストールしてデモアプリを作成したり、 API を使用して何かを行うウェブサイトを作成したりしてください。
  > [!NOTE]
  > ボーナスポイントとして、彼らが現在解決しようとしている問題や製品のバグを特定し、デモやコードスニペットを添えて修正案を提案してみてください。
- この特定の職務に合ったように履歴書/職務経歴書をカスタマイズしてください。職務内容を確認し、最も関連性の高いスキルや職務経験を強調し、関連性が低いと思われる項目は最小限に抑えるか、削除してください。また、この職務に最も関連性の高いデモ/例を、ポートフォリオで何らかの形で強調表示してください。例えば、リストの一番上に配置するなどです。

### 求人への応募

求人に応募する際は、必ず指定された手順に従ってください。各求人広告には応募方法が記載されています。求人広告がネットワークの連絡先との会話のような形式だった場合は、具体的な応募方法を直接確認してください。

何より重要なものを示します。

- 応募書類がプロフェッショナルでよく書かれ、誤字脱字がないことを確認してください。文章が拙い場合、採用される可能性は低くなります。 Grammarly のようなツールを使用して、文章を確認してください。
- 応募期限を厳守し、期限までに応募書類を提出してください。

すべての求人応募で同じことが求められるわけではありませんが、おそらく以下の手順が必要になるでしょう。

- オンラインフォームに詳細を入力してください。回答は、フォームに入力する前に、ローカル（例: VSCode）またはオンラインドキュメント（例: Google ドキュメント）に記入しておきましょう。これにより、ブラウザーのクラッシュなどにより作業が失われるリスクを回避できます。また、テキストを別の場所に保存しておくと、後で確認やフィードバックが容易になります。
- 履歴書/職務経歴書をアップロードしましょう。
- 添え状を書き、アップロードしてください。効果的な添え状の書き方については、オンラインに多くのガイドがありますが、一般的には、次のような内容を記載する必要があります。
  - その会社で働きたい理由。
  - その会社の文化に合う理由。
  - その仕事に最適である理由。
- 過去の業務実績を別セクションでいくつか例を挙げて記載してください。

### 継続的なプロセス

重要なポイントは、就職活動は毎日少しずつ、目標に向かって絶えず努力し続ける継続的なプロセスであるということです。大量の応募書類を送りつけて、返事を待つだけになってはいけません。

コミュニティネットワークや LinkedIn などで出会った有望な情報については、必ずフォローアップしてください。また、出会った人や応募した企業の一覧、それぞれの連絡先や概要をまとめておきましょう。希望する企業には定期的に連絡を取り、求人状況を確認しましょう。ある求人で不採用になった場合でも、現在または将来的にあなたに適したポジションがあるかもしれません。

応募した企業から返信がない場合は、少し待ってから採用担当者に連絡し、フィードバックや他の求人の有無を確認しましょう。希望する企業が現在適切な正社員の求人を掲載していない場合でも、連絡を取ってみてください。あなたのスキルセットに合った人材を探している可能性があります。フリーランスの契約など、役立つ経験につながるかもしれません。

選択した求人サイトやコミュニティネットワークを通じて、引き続き求人を探し、人々と話し合い、何が利用可能か確認してください。新しい機会は常に生まれます。

本当にやりたい仕事が見つからない場合は、中間ステップを踏む必要があります。

- 時には、希望する仕事に隣接する仕事を得る戦略が有効です。希望する仕事に必要な経験を得るために、最初はランクの低い仕事から始める必要があります。これは、使用する特定の技術スタックや、働きたい業界に特化した経験を得るためです。
- 家賃を支払うために、フリーランスや契約仕事をする必要があるかもしれません。これは、目指す仕事を得るための役立つ経験も得られます。

## 就職の面接の準備

採用担当者や採用マネージャーがあなたの応募書類を気に入った場合、面接やその他の選考プロセスへの参加を招待されます。この節では、その準備に関するアドバイスを提供します。

あらゆる面接に備えて行うべき一般的なポイントがあります。

- 面接を行うための清潔で静かな場所を確保してください。面接中に騒音や混乱で気が散らないようにし、面接中の周囲が整然としていることを確認してください。面接官とビデオ通話を行う場合、あなたのスペースが散乱していると、良い印象を与えません。
- 面接時に着る服装についても考えておきましょう。開発者の仕事ではスーツを着用する必要はありません（求人広告にドレスコードに関する記載があるか確認してください）が、中立的でスマートな服装を選ぶようにしてください。攻撃的なスローガンが書かれた T シャツを着たり、パジャマで現れたりしないでください。
- [コーディング面接のための準備](#コーディング面接)コーディング面接のための準備をしておきましょう。
- [経験に基づく面接の質問](#経験に基づいた質問)に使えるエピソードをいくつか用意しておきましょう。
- 自分の長所と短所について考え、現実的な答えを用意しておきましょう。あらゆる面で完璧であるように見せようとすると、逆に不自然に見えてしまいます。面接官は、あなたの最大の長所と短所についてよく質問します。
- 希望する職種（レベル、地域など）の一般的な給与範囲やその他の福利厚生（健康保険やボーナスなど）について調査してください。これは、[就職のオファーを受け入れるかどうかを決定](#内定を受け入れる)する際に役立ちます。

また、応募する企業ごとに、以下の準備も必要です。

- 前述の通り、企業の歴史、目標、製品について調査してください。
  - 企業について大まかに読み、最近のニュースについてよく理解しておいてください。この調査を行う良い理由のひとつは、その企業について、就職を再考すべきような懸念事項がないかどうかを確認するためです。
  - 企業の市場ポジション、およびその強みと弱みを把握してください。
  - 主な競合他社を調査してください。
  - 製品の特徴や動作を把握してください。例えば、ソフトウェアをインストールしてデモアプリを作成したり、その API を使用して何かを行うウェブサイトを作成したりしてください。企業の状況に関する洞察や改善点を具体的に示せれば、面接官に好印象を与えることができます。面接官は、会社や業界に関する知識をテストするために、具体的な質問をすることがよくあります。
- 上記の点に関連して、会社や面接官の役割について、面接官に尋ねたい質問をいくつか考えておきましょう。面接官は、面接の最後に必ず「何か質問はありますか？」と尋ねます。この質問に対して、「今日の朝食は何を食べましたか？」といった質問よりも、より意味のある質問をいくつか用意しておくと、あなたを思慮深く、探究心のある人物として印象付けることができます。
- もし会社内に知り合いがいる場合（コミュニティネットワークなどから）、連絡を取って仕事について尋ねてください。その仕事についてどう思っているか、特に注目すべき点や避けるべき点はあるか、などです。面接中に「EC チームのボブを知っています」と言及するのも役立つかもしれません。

### 初期審査

会社があなたの応募書類を確認し、あなたが「有望」だと判断した場合、まず最初に、採用担当者または採用責任者との最初の電話面接に招待されるでしょう。これは通常、文化適合性（あなたとチームや組織が合うかどうか）を評価し、あなたが良い従業員にならない可能性を示す直感的な懸念点がないか確認し、その後の面接プロセスについて説明する短い面接です。

この段階では、親しみやすく話しやすい態度で臨むべきですが、話しすぎないように注意し、自然体で振る舞い、面接官が会話の方向性をどう導くかを見極めましょう。今後の面接に参加可能な日程を必ず伝えてください。できるだけ柔軟に対応し、必要に応じてスケジュールを調整する用意をしておきましょう。

ただし、油断は禁物です。この段階では、どのような質問が投げかけられるか確実には分かりませんので、他の段階と同様に真剣に臨むことが重要です。

### 本面接

通常、面接プロセスの大部分は 1 回以上の本面接で構成されます。本面接は、企業によって機能や頻度が大きく異なります。

- 本面接が 1 回だけの場合もありますが、職務のさまざまな側面（実際のコーディング作業、チームワーク、作業プロセス、文書化/コミュニケーション、紛争解決など）をテストする複数の面接が行われる場合がほとんどです。
- 1 人の面接官との面接の場合もあれば、複数の面接官が異なる側面について質問する形式（パネル面接）の場合もあります。
- 面接の内容は、高水準で探求的なもの（例：「X を実装するプロセスを説明してください」や「この解決策の主な問題点は何か」など）と、低水準で詳細な内容（例：「この問題を解決する JavaScript プログラムを書いてください」など）があります。

#### 経験に基づいた質問

面接では、経験に基づく質問が必ず出されます。これらの質問は、その仕事に関連する経験があることを示すエピソードを話すことを求め、次のような内容になります。

- 「同僚のコードについて、否定的なフィードバックを伝えなければならなかった状況を例を挙げて教えてください。どのように対応しましたか？」
- 「プロジェクトの途中で、選択したアプローチが機能しないことに気づいた経験を教えてください。その問題をどのように解決し、どのようなステップを踏みましたか？」
- 「小さなバグの修正は、チームの作業プロセスに組み込むのが難しいことがあります。しばしば、バグの積み重ねが放置され、膨れ上がってしまうことがあります。過去のチームでこのような状況にどう対処しましたか？」

このような質問に答える際に使えるエピソードをいくつか用意しておくと良いでしょう。経験を重ねるにつれ、回答は次第にスムーズになっていきます。

#### コーディング面接

一部の面接はコーディング面接となり、コーディング問題の解決能力をテストします。以下のような問題が出題される場合があります。

- 「x と y を入力として受け取り、z を返す JavaScript 関数を作成してください。」
- 「同僚が以下の React コンポーネントを書きましたが、バグがあります。問題点を特定し、修正したコードを書き直してください。」
- 「以下のコードを見て、その動作を説明してください。」
- 「フィボナッチ数列の最初の n 項を生成するアルゴリズムを記述してください。」

多くの場合、採用担当者は結果そのものではなく、問題へのアプローチ方法に注目しています。コードのコメントの仕方、エラー処理の対応方法、時間があればどのような対応をしたかなどです。

一般的なコーディング面接の質問タイプを調査し、解く練習をし、その内容を説明できるように練習してください。コーディング面接の質問を練習するためのオンラインリソースは多く存在します。

#### 宿題

一部の企業では、面接候補者に課題を出題し、コーディング能力を評価する代替手段または追加手段として活用する場合があります。これは通常、応募する職種に関連する機能を実現するミニプロジェクト形式の課題となります。例えば、企業の JavaScript ライブラリーを使用して API からデータを取得し、情報ページを表示するスタンドアロンデモを作成するよう求められる場合があります。

この方法は適切に行われれば候補者を評価する合理的な方法ですが、多くの人が宿題問題の設定を嫌がる理由があります。

- 時々、本物の仕事のように感じられ、雇用主が無料で仕事をさせようとしているように思えることがあります。
- 範囲が不明確だったり、不合理な時間的コミットメントを要求する場合があります。プロジェクトを 4 時間に限定し、フォローアップの質問は別面接で対応するといった具体的な範囲設定は合理的なものですが、週末を丸ごと費やすことを求めるのは、家族や多忙な生活を送る人にとっては受け入れがたいものです。
- その効果はしばしば議論の的となります。なぜなら、明確に評価するのが難しいからです。

宿題の課題が求められた場合、作業の範囲とスケジュールが明確に定義されていることを確認し、方向性を誤って時間を無駄にするリスクを軽減してください。

### 内定を受け入れる

面接を無事に通過し、内定を得た場合、おめでとうございます。すぐに内定を受け入れたい気持ちになるかもしれませんが、その前に以下の点を確認しておきましょう。

- 提示された給与が、同様の職種の一般的な相場と比べて妥当かどうかを確認してください。大幅に低い場合は、交渉を検討した方が良いでしょう。
- 現在の仕事（または他の活動）を辞めることが、新しい仕事を開始する上で問題ないか確認してください。例えば、現在の雇用主がどの程度の退職予告期間を求めているかを確認し、新しい仕事と調整可能か確認してください。
- 仕事以外の既定の予定（例えば、既に予約済みの休暇など）がある場合は、雇用主に伝えてください。信頼できる雇用主は、そのような予定を考慮し、あなたに合った開始日を調整してくれるでしょう。
- 新しい仕事のために引っ越しが必要な場合は、引っ越しがスムーズに進むよう、必要な手配をすべて済ませておきましょう。新しい仕事を開始するだけでもストレスが溜まるものですから、引っ越しのストレスまで加わるとさらに大変です。

## 挑戦、挑戦、また挑戦

多くの求人に応募してもまだ何も決まっていない場合は、落胆しないようにしましょう。挑戦を続け、ウェブへの情熱を持ち続け、新しい人と交流し、新しいことを学び続けてください。

面接まで進んだものの不採用になった場合は、採用担当者に不採用の理由を尋ね、そのフィードバックを今後の応募に活かすようにしてください。

忍耐強く続けてください。最高の候補者でも、希望する仕事に就くまでに複数の応募で不採用になることがあります。必ず目標に到達できます。
