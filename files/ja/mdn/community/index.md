👋 MDN Web Docs へようこそ! MDN Web Docs は、[HTML](/ja/docs/Web/HTML)、[CSS](/ja/docs/Web/CSS)、[JavaScript](/ja/docs/Web/JavaScript)、[Web API](/ja/docs/Web/API) など、ウェブプラットフォーム技術のオープンソースかつ共同プロジェクトで、その文書を作成しています。また、初期段階の開発者や学生向けに、豊富な[学習リソース](/ja/docs/Learn_web_development)を提供しています。

- [始めよう](/ja/docs/MDN/Community/Getting_started)
  - : この章では、協力を始めるにはどうすればよいか、また、どのような協力が受け入れられるかについて説明します。
    誰でも協力することができ、協力するひとりひとりが数百万人の読者に影響を与えます。
    オープンウェブに寄与し、革新を推進する方法を学びましょう。
- [私たちのリポジトリー](/ja/docs/MDN/Community/Our_repositories)
  - : この記事では、MDN Web Docs に協力する際に必要となる可能性のある GitHub リポジトリ－ (repos) について記述しています。
- [GitHub issue](/ja/docs/MDN/Community/Issues)
  - : issue は、すべてのバグと明確な実行可能な結果を保有する作業を追跡するために使用されています。
    この記事では、 issue の開設と作業に関するガイドラインを記載し、トリアージすべき issue とコンテンツの提案についても取り上げています。
- [プルリクエスト](/ja/docs/MDN/Community/Pull_requests)
  - : この章では、プルリクエストを送信する際のガイドラインと、レビュープロセスで期待されることについて説明します。
- [役割とチーム](/ja/docs/MDN/Community/Roles_teams)
  - : この章では、MDN Web Docs プロジェクトに参加しているユーザーとチームの概要を指定し、チームの一員であるということの意味を詳しく説明します。
- [コンテンツの翻訳](MDN/Community/Translated_content)
  - : MDN Web Docs のローカライズ情報、チーム、コミュニケーションチャネル、参加方法の詳細です。
- [オープンソースのエチケット](/ja/docs/MDN/Community/Open_source_etiquette)
  - : この記事は、私たちのオープンソースプロジェクトに協力する際の振る舞い方について、協力のルール、エチケット、紛争を処理する方法など、さまざまな指針を提供します。
- [コミュニケーションチャンネル](/ja/docs/MDN/Community/Communication_channels)
  - : このページには、 MDN チームとコミュニティが使用しているコミュニケーションチャネルが掲載されており、その中から自分に最適なものを選ぶためのヒントが得られます。

## 行動規範

私たちのプロジェクトや議論に参加し、協力することによって、あなたは [Mozilla コミュニティ参加ガイドライン](https://github.com/mdn/mdn-community/blob/main/CODE_OF_CONDUCT.md) (CPG) を読み、同意したものとみなします。

### 一般的なサポートに関する質問

私たちは小さなチームで、絶えず変化するウェブのエコシステムの文書化の要求に追いつくために、一所懸命作業しています。
残念ながら、私たちは一般的なサポートに関する質問には対応できません。
学習中の方は、[ウェブ開発の学習](/ja/docs/Learn_web_development)の章がよい開始点であり、コーディングの質問や答えの検索は [Stack Overflow](https://stackoverflow.com/questions/) が向いています。

一般的な対応を依頼するリポジトリーで開かれた issue、ディスカッション、プルリクエストはすべてここに向けられ、閉じられたりロックされたりすることがあります。
