ページの冒頭には、モジュールの名前を明記し、その機能について説明する導入段落を記載してください。仕様書で定義されている機能の概要を簡潔に述べ、必要に応じて、関連する仕様書の機能とどのように連携するかを記述してください。この説明は概要を示すものであり、チュートリアルやガイドではありませんので、簡潔にまとめてください。

## NameOfTheModule の実演

この節では、`\{{EmbedLiveSample}}` を使用する例を記載してください（情報については、[ライブサンプル](/ja/docs/MDN/Writing_guidelines/Page_structures/Live_samples)を参照）。これにより、このモジュールが提供するさまざまなプロパティの有用性と威力を示すことができます。
この節の目的は、使用例を実演し、このモジュールについて学んでいる読者の関心や好奇心を喚起することです。使用例を理解するために不可欠な場合（名前空間や入れ子など）を除き、コードは非表示にしてください。

該当する場合は、読者がその例を操作する方法を、短く説明してください。

## リファレンス

関連するプロパティ、関数、データ型などを掲載するための適切な小見出しを作成して、それらを列挙してください。リファレンスには、その単一の仕様書で導入された機能のみを記載するようにしてください。仕様書に記載されているものの、対応していない機能がある場合は、適切な見出しの下の段落で言及してください。他の仕様書で定義されている関連機能は、「関連概念」の項に記載し、この節には含めないでください。
