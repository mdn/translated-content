## 使用 MDN Web Docs 內容

本節說明我們提供的內容類型，以及當你選擇重用這些內容時，適用的著作權與授權條款。

### 文件

> [!NOTE]
> MDN Web Docs 的內容由 Mozilla 內部與外部的作者共同撰寫。除非另有說明，否則內容皆以 [Creative Commons Attribution-ShareAlike license](https://creativecommons.org/licenses/by-sa/2.5/)（CC-BY-SA）第 2.5 版或任何後續版本授權。

你重用這裡的內容時，必須以相同的授權條款（CC-BY-SA v2.5 或任何後續版本）發布。
當你重用 MDN Web Docs 的內容時，必須確保[適當地標示出處](https://creativecommons.org/licenses/by/2.5/deed.en#ref-appropriate-credit)，並註明「Mozilla Contributors」。
良好的標示方式為：**文件標題**，並加上超連結（線上）或網址（印刷品）指向所引用內容的特定頁面，並簡要說明你所做的修改。
例如，若要標示本頁內容，你可以寫：

> [「署名與著作權授權」](/zh-TW/docs/MDN/Writing_guidelines/Attrib_copyright_license) by Mozilla Contributors，依 [CC-BY-SA 2.5](https://creativecommons.org/licenses/by-sa/2.5/) 授權。

你也可以將「Mozilla Contributors」連結到頁腳的 `contributors.txt` 檔案，以便合理地列出作者清單。
詳情請參考 [標示出處的建議做法](https://wiki.creativecommons.org/wiki/Recommended_practices_for_attribution)。

### 程式碼範例

2010 年 8 月 20 日（含）之後新增的程式碼範例皆屬於 [public domain CC0](https://creativecommons.org/publicdomain/zero/1.0/)。不需特別標示授權，但如有需要，可使用：「Any copyright is dedicated to the Public Domain: https://creativecommons.org/publicdomain/zero/1.0/」

2010 年 8 月 20 日之前新增的程式碼範例則採用 [MIT license](https://opensource.org/license/mit)；你應在 MIT 授權範本中填入以下標示資訊：「© \<最後一次 wiki 頁面修訂日期> \<將程式碼加入 wiki 的人名>」。

自 2020 年 12 月 14 日新 Yari MDN 平台上線後，目前尚無法判斷程式碼範例所屬授權。我們正努力改善，並將盡快更新本內容。<!--這段還需要保留嗎？-->

### 你的貢獻

若你希望貢獻內容至 MDN Web Docs，即表示你同意你的文件內容將以 Attribution-ShareAlike 授權條款（或該頁面已指定的其他授權條款）提供，而你的程式碼範例將以 [Creative Commons CC-0](https://creativecommons.org/publicdomain/zero/1.0/)（公眾領域貢獻）授權。

> [!WARNING]
> 不得以其他授權條款建立新頁面。

**貢獻內容的著作權仍歸原作者所有，除非作者另行轉讓。**

如對本頁內容有任何疑問，請聯絡 [MDN Web Docs 團隊](/zh-TW/docs/MDN/Community/Communication_channels)。

### 標誌、商標、服務標章與字標

Mozilla Foundation 的標誌、商標、服務標章，以及本網站的整體設計，皆不在 Creative Commons 授權條款範圍內。若這些標誌與設計屬於著作權作品，則不包含在授權內容中。若你希望在引用文件文字時同時使用這些權利，或對本授權條款有其他疑問，請聯絡 Mozilla Foundation：[licensing@mozilla.org](mailto:licensing@mozilla.org)。

## 使用 MDN Web Docs 其他內容

我們原則上不贊成將其他來源的內容複製到 MDN。
MDN 應盡可能由原創內容組成。
若我們收到 pull request 並發現內容為抄襲，將會關閉該請求，並要求提交者以自行撰寫的內容重新提交。

### 在 MDN 重新使用或重新發布你的內容

> [!NOTE]
> 除非有充分理由重新發布內容，否則我們通常會說「不」。
> MDN 編輯團隊的決定為最終決定。

若有人希望將先前發表於部落格的文章捐贈給 MDN，或有必要將複雜的參考資料表複製到 MDN，這些情況下可考慮重新發布。請事先與 MDN 團隊討論你的計畫：

- [建立 GitHub issue](https://github.com/mdn/mdn/issues/new/choose) 說明你的意圖。
  - 描述你想複製或重新發布的內容。
  - 提供該資源的網址。
  - 解釋你認為合適的理由。

**若內容採封閉授權發布：**

- 若你擁有該內容的權利，請明確說明並同意將其重新發布於 MDN。
- 若你不擁有該內容的權利，請盡可能將作者／出版者加入 issue，或提供聯絡方式，以便我們徵求授權。

**若內容採開放授權發布：**

- 請說明授權類型，並附上授權連結，以便我們確認其是否與[MDN 授權條款](https://github.com/mdn/content/blob/main/LICENSE.md)相容。

## 連結至 MDN Web Docs 文章

我們經常收到使用者詢問是否可以連結至 MDN Web Docs，以及如何連結。簡單來說：**你可以連結至 MDN Web Docs！** 超連結是網路的本質，不僅能為你的用戶指向有價值的資源，也展現你對我們社群成果的信任。
