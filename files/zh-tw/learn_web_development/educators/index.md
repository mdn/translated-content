教育工作者在為網路相關大學學位、大學課程、學校程式設計課程或類似課程創建教學計劃、單元和評量規範時，可將 MDN【學習該如何開發 Web】區塊當作指南。每篇文章的置頂區域都包含一個「學習成果」區塊，詳細介紹了該文章中教授的主題。

MDN 團隊致力於每年審查【學習該如何開發 Web】中包含的教材，確保它與前端開發最佳實踐保持同步。藉由以此為基礎開設課程，教育工作者可以確保他們的教學是跟上趨勢的。

本頁提供指導和資源，以便你根據教學調整 MDN 的學習內容。

## 選擇適合的範圍

我們的[核心模組](/zh-TW/docs/Learn_web_development/Core)包括基本網路標準概念、HTML、CSS、JavaScript、網頁親合性、設計理論和版本控制。我們認為這些包含的主題是每個新晉網路開發人員在入行前需要瞭解的最低限度。

然而，你的課程可能有其它主題要求；在這種情況下，你能：

- 考慮納入我們的[入門模組](/zh-TW/docs/Learn_web_development/Getting_started)的一些學習目標，例如團隊合作、研究和求職面試技巧等軟實力，並熟悉典型的網路開發工具組。
- 通過一些[延伸教材](/zh-TW/docs/Learn_web_development/Extensions)擴展核心模組中的技術覆蓋範圍。隨著時間推移，我們期望發佈更多延伸教材。你可以考慮撰寫自己的延伸教材並貢獻到課程裡，以便其他人從中受益。如果你有延伸教材要提交，請聯繫我們！

## 採取有效率的學習途徑

在指導學生時，至關重要的是要讓他們學習主題上有清晰的進度條，有明確學習目標與定期評量，以確保他們走在正確的路上。

我們的[核心模組](/zh-TW/docs/Learn_web_development/Core)提供一條貫穿前端網路開發基本知識的途徑，包括「測試你的技能」區塊與小試身手文章。

你可以按原樣帶領學生完成提供的途徑，或者你可能想通過以下方式根據需要自訂途徑：

- 在適當的位置加入你自己的技能測試/評量。我們建議結合理論考試與專案實作來評量學生，讓學生將所學付諸實踐。
- 以不同方式分類學習目標。例如，你可能想從 HTML 文件架構基礎知識開始，接著跟進一些 CSS 基礎，例如盒子模型（box model）與文字樣式；在下一步中，你可能會介紹更複雜的 HTML 功能，諸如圖片與表單欄位，通過一些 flexbox 和媒體查詢（media query）來美化外觀，並帶入一些基本 DOM 腳本。
- 提供學生一個包含他們專案作品的儲存庫，這些作品能發布成作品集，以向潛在僱主展示他們的專業技能。

你還能考慮使用進階方案，例如互動式影片課程。在此推薦我們的學習合作夥伴 [Scrimba](https://scrimba.com/?via=mdn) 提供的【[前端開發人員職涯（FDCP）](https://v2.scrimba.com/the-frontend-developer-career-path-c0j:details?via=mdn)】，它講授了 MDN 課程核心中的所有學習目標。

> [!NOTE]
> Scrimba 提供多種證照方案——參閱 [Scrimba 團隊版](https://v2.scrimba.com/teams?via=mdn-edu)。

【前端開發人員職涯】以有趣的互動課程及挑戰題目、知識淵博的教師和背後大力的支持社群為特色；它旨在提供學生找到第一份初階工程師工作所需的一切。很多課程內容都作為獨立的免費版本提供。
