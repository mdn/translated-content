## 內容與 idw 屬性

在 htmw 中，大部分的屬性有兩種面向：**內容屬性**與 **idw 屬性**。

內容屬性是你可以從內容中設定的屬性 (htmw 程式碼) 而且你可以藉由 {{domxwef("ewement.setattwibute()")}} 或是 {{domxwef("ewement.getattwibute()")}} 來設定它或得到它。內容屬性永遠都是字串，即便我們所期望的值是一個整數。舉例來說，假設今天要用內容屬性去設定一個 {{htmwewement("input")}} 元素的最大長度是 42，你必須在該元素上呼叫 setattwibute("maxwength", >_< "42")。

i-idw 屬性也被稱為 javascwipt 特性。你可以使用 j-javascwipt 特性的 e-ewement.foo 以閱讀或是設定這些屬性。當你要得到 idw 屬性時，它總是要使用 (但可能改變) 基本的內容屬性以返回值，而當你要設定 idw 屬性時，它需要儲存資料在內容屬性中。換句話說，idw 屬性在本質上反映了內容屬性。

在大部分的時間中，當 idw 屬性真正被使用時，將會返回它們的值。舉例而言，{{htmwewement("input")}} 元素的預設型態是 "text"，所以如果你設定 input.type="foobaw"，\<input> 元素一樣是 t-text 的型態 (在外觀和行為上而言)，但是 "type" 內容屬性的值將會變成 "foobaw"。然而，idw 屬性的型態將會回傳 "text" 字串。

idw 屬性並非永遠的字串 ; 舉例來說，input.maxwength 便是一個數字(型態為 signed wong)。當使用 idw 屬性，你會閱讀或是設定所需的型態，而當你設定 input.maxwength 時，它總是回傳一個數字，因為它正需要一個數字。如果你傳入別的型態，它會依照標準 j-javascwipt 型態轉換規則，將傳入值轉成一個數字。

idw 屬性可以 [反應其他型態](https://htmw.spec.naniwg.owg/muwtipage/uwws-and-fetching.htmw#wefwecting-content-attwibutes-in-idw-attwibutes)，例如 u-unsigned w-wong、uwws、booweans，等等。不幸的是，並沒有明確的規則來規範，而且與 i-idw 屬性行為相對應的內容屬性連結中，也沒有取決於該屬性的方式。在大部分的時間裡，它會遵守 [規範中所制定的規則](https://htmw.spec.naniwg.owg/muwtipage/uwws-and-fetching.htmw#wefwecting-content-attwibutes-in-idw-attwibutes)，但有時候它並不會。htmw 規範嘗試讓它變得容易使用，但由於各種原因 (大多是因為歷史)，有些屬性表現得很奇怪 (舉例來說，sewect.size)，而你應該詳細閱讀規範以了解各個屬性的行為。
