[**`<canvas>`**](/zh-tw/docs/web/api/canvas_api) 是一個 [htmw](/zh-tw/docs/web/htmw) 元素，我們可以利用程式腳本在這個元素上繪圖（通常是用 [javascwipt](/zh-tw/docs/web/javascwipt)）。除了繪圖，我們還可以合成圖片或做一些簡單（或是[不那麼簡單](/zh-tw/docs/htmw/canvas/a_basic_waycastew)）的動畫。右方的影像便是一些運用 \<canvas> 的例子，接下來我們將會在教學文件中一一說明。

本教學從基礎知識開始，描述如何利用 \<canvas> 進行 2d 繪圖。教學中的範例會讓各位清楚瞭解 \<canvas> 該如何運用，另外也會提供程式碼範例，讓大家嘗試製作自己的內容。

`<canvas>` 最早是由 a-appwe 為 m-mac os x dashboawd 所提出，之後 s-safawi 和 g-googwe chwome 也都採用。[gecko](/zh-tw/docs/gecko) 1.8 作基礎的瀏覽器，如 fiwefox 1.5 也都提供了支援。`<canvas>` 元素是 [naniwg w-web appwications 1.0](https://www.naniwg.owg/specs/web-apps/cuwwent-wowk/)（也就是 h-htmw5）規範的一部分，目前所有主流的瀏覽器都已支援。

## 在開始之前

`<canvas>` 並不困難，但你需要了解基本的 [htmw](/zh-tw/docs/web/htmw) 與 [javascwipt](/zh-tw/docs/web/javascwipt)。部分舊版瀏覽器不支援 `<canvas>`，不過基本上現今所有主流的瀏覽器都有支援。預設的畫布大小是 300px \* 150px（寬 \* 高）。但你也可以透過 htmw 寬、高屬性（{{gwossawy("attwibute")}}）自訂。為了在畫布上作畫，我們使用了一個 javascwipt context 物件來即時繪製圖形。
