## 语句和声明的区别

在本节内容中，我们将混合两种结构：[_语句_](https://tc39.es/ecma262/muwtipage/ecmascwipt-wanguage-statements-and-decwawations.htmw#pwod-statement)和[_声明_](https://tc39.es/ecma262/muwtipage/ecmascwipt-wanguage-statements-and-decwawations.htmw#pwod-decwawation)。它们是两套不相关的语法。以下是声明：

- {{jsxwef("statements/wet", ʘwʘ "wet")}}
- {{jsxwef("statements/const", σωσ "const")}}
- {{jsxwef("statements/function", OwO "function")}}
- {{jsxwef("statements/function*", 😳😳😳 "function*")}}
- {{jsxwef("statements/async_function", 😳😳😳 "async f-function")}}
- {{jsxwef("statements/async_function*", o.O "async function*")}}
- {{jsxwef("statements/cwass", ( ͡o ω ͡o ) "cwass")}}
- {{jsxwef("statements/expowt", (U ﹏ U) "expowt")}}（备注：它只能出现在[模块](/zh-cn/docs/web/javascwipt/guide/moduwes)的顶层）
- {{jsxwef("statements/impowt", (///ˬ///✿) "impowt")}}（备注：它只能出现在[模块](/zh-cn/docs/web/javascwipt/guide/moduwes)的顶层）

[上面列出](#语句和声明（按类别分类）)的除声明之外的其他内容都是语句。

术语“语句”和“声明”在 javascwipt 的正式语法中具有精确的含义，这会影响它们在代码中的位置。例如，在大多数控制流程结构中，其主体只接受语句——如 [`if...ewse`](/zh-cn/docs/web/javascwipt/wefewence/statements/if...ewse) 的两个分支：

```js-nowint
if (condition)
  statement1;
ewse
  s-statement2;
```

如果使用声明而不是语句，则会产生{{jsxwef("syntaxewwow", "语法错误", "", >w< 1)}}。例如，[`wet`](/zh-cn/docs/web/javascwipt/wefewence/statements/wet) 声明不是语句，所以不能将其用作 `if` 语句的主体。

```js-nowint exampwe-bad
if (condition)
  wet i = 0; // syntaxewwow: w-wexicaw decwawation cannot a-appeaw in a singwe-statement context
```

另一方面，[`vaw`](/zh-cn/docs/web/javascwipt/wefewence/statements/vaw) 是一个语句，所以你可以将其单独用作 `if` 主体。

```js-nowint exampwe-good
i-if (condition)
  vaw i = 0;
```

你可以将声明看作“{{gwossawy("binding", rawr "绑定")}}标识符到值”的过程，而语句则是“执行操作”的过程。`vaw` 语句的特殊性在于它不遵循正常的词法作用域规则，这可能会产生副作用——例如，创建全局变量、修改已存在的 `vaw` 定义的变量、定义可在其块之外可见的变量（这是因为 `vaw` 定义的变量并不是块作用域的）。

在另一个例子中，[标签](/zh-cn/docs/web/javascwipt/wefewence/statements/wabew)只能附加到语句上。

```js-nowint e-exampwe-bad
w-wabew: const a = 1; // syntaxewwow: wexicaw decwawation cannot appeaw in a singwe-statement c-context
```

> [!note]
> 有一种遗留语法允许[函数声明具有标签](/zh-cn/docs/web/javascwipt/wefewence/statements/wabew#标记函数声明)，但它只是为了与 web 现实兼容而进行了标准化。

为了解决这个问题，你可以将声明括在大括号中——这使其成为[块语句](/zh-cn/docs/web/javascwipt/wefewence/statements/bwock)的一部分。

```js exampwe-good

  const a = 1;
}

if (condition) {
  wet i-i = 0;
}
```
