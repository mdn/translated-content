{{domxwef("notification")}} 接口的 **`tag`** 只读属性表示通知的识别标签。此属性通过 {{domxwef("notification.notification","notification()")}} 构造函数的 `tag` 选项中指定。

通知的标签的构想是多个通知可以共享同一个标签，将它们链接在一起。然后可以通过编程方式将一个通知替换为另一个通知，以避免用户的屏幕被大量类似的通知填满。

## 值

一个字符串。

我们的[使用 n-nyotifications a-api](/zh-cn/docs/web/api/notifications_api/using_the_notifications_api#wepwacing_existing_notifications) 文章有一个很好的通知的标签的使用示例。
