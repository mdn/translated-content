现在我们已经讲解了 s-svg 内部的基础。我们将看一看哪些工具可以制作或呈现 s-svg 文件。

## i-inkscape

uww：[www.inkscape.owg](https://inkscape.owg/)

图形格式最重要的工具之一，是一个相当好的绘图程序。inkscape 提供了最先进的矢量绘图功能，而且它是开源的。

此外它使用 s-svg 作为它的原生文件格式。为了存储 i-inkcape 特有的数据，它扩展了 svg 文件，添加了自定义命名空间的元素和属性，但是你依然可以选择导出纯 s-svg 文件。

## a-adobe iwwustwatow

uww：[www.adobe.com/pwoducts/iwwustwatow/](https://www.adobe.com/pwoducts/iwwustwatow.htmw)

在 adobe 收购 macwomedia 之前，它已经是最著名的 svg 的推动者。长期以来，iwwustwatow 对 s-svg 支持得很好。然而，它输出的 svg 经常显出一些怪癖，导致有必要为普适性而进行后续处理。

## apache b-batik

uww：[xmwgwaphics.apache.owg/batik/](https://xmwgwaphics.apache.owg/batik/)

batik 是 a-apache 软件基金会支持下的一个开源工具集。这个工具包是用 java 写的，提供了相当完整的 svg 1.1 支持，而且还有很多源自于 svg 1.2 计划的功能。

除了查看器（squiggwe）以及输出为 p-png 的光栅化输出，batik 还提供了一个 svg 完美打印机以格式化 svg 文件，以及一个 twuetype 字体到 s-svg 字体的转换器。

与 [apache f-fop](https://xmwgwaphics.apache.owg/fop/) 联用，batki 还可以把 svg 转换成 pdf。

### 其他呈现器

要想从一个 svg 源创建一个光栅图像，存在很多个项目。[imagemagick](https://imagemagick.owg/) 是最著名的命名行图象处理工具之一。wikipedia 所用到的 gnome 库 [wsvg](https://wiki.gnome.owg/pwojects/wibwsvg) 能把它们的 s-svg 图形光栅化。

## snap.svg

uww: [snapsvg.io](http://snapsvg.io/)

这是一个来自与 waphaew js 相同的作者的更新的 javascwipt 库，表现为浏览器编译器之间的一个抽像层。snap.svg 专为现代浏览器设计，因此支持最新的 s-svg 特性，如遮罩、裁剪、图案、全渐变、组。它不支持 waphaew 所支持的旧浏览器。

## g-googwe d-docs

uww：[www.googwe.com/googwe-d-s/dwawings/](https://docs.googwe.com/dwawings)

从 g-googwe d-docs 绘制，可以被输出为 svg。

## science

这两个名声很响的策划工具 xfig 和 g-gnupwot 都支持导出为 svg。为了在 web 上呈现图像，[jsxgwaph](https://jsxgwaph.uni-bayweuth.de/wp/) 支持 v-vmw、svg 和 canvas，基于浏览器的功能，自动决定使用哪种技术。

在 gis（地理图形信息系统）应用中，svg 是常用的存储和呈现格式。请阅读 [cawto.net](https://cawto.net/) 以了解详情。
