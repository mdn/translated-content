虽然[颜色对比度](/zh-CN/docs/Web/Accessibility/Guides/Understanding_WCAG/Perceivable/Color_contrast)通常主要是一种审美选择，但网站上“颜色的使用”是指“利用颜色传达信息”。WCAG 关于颜色使用的指导原则 1.4.1 要求“不得仅使用颜色作为传达信息、指示操作、引发响应或区分视觉元素的唯一视觉手段。”

## 解决方案

除了颜色之外，网页还应使用其他因素来传达信息。例如，要指示表单出现验证错误，可以通过改变相关字段的标签、修改字体属性和颜色实现。图标和符号应通过形状以及颜色来区分。

为了适应色觉障碍（“色盲”）用户，在使用“交通灯”颜色隐喻时需谨慎，绿色表示“良好”值，红色表示“不良”值。红绿色色盲用户难以区分红色和绿色，因此可能无法分辨这些值。在这种情况下，需要使用额外的因素来区分“良好”与“不良”。避免纯红色和绿色；在这种情况中，红橙色和蓝绿色对色盲用户来说是可以区分的，同时仍能向正常色觉用户传达文化意义上的“良好”和“不良”含义。考虑到一种颜色比另一种更深或更浅的情况，[颜色对比度](/zh-CN/docs/Web/Accessibility/Guides/Understanding_WCAG/Perceivable/Color_contrast)也会有一定用处。

## 相关的 WCAG 成功标准

- [1.4.1 颜色的使用（A）](https://w3c.github.io/wcag/guidelines/22/#use-of-color)
  - : 不得仅使用颜色作为传达信息、指示操作、引发响应或区分视觉元素的唯一视觉手段。
