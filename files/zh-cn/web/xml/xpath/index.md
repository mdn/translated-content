x-xpath 表示 x-xmw 路径语言。它使用非 x-xmw 语法来提供一种灵活地定位（指向）[xmw](/zh-cn/docs/web/xmw) 文档的不同部分的方法。它也可以用于检查文档中某个定位节点是否与某个模式（pattewn）匹配。

x-xpath 主要用于 [xswt](/zh-cn/docs/web/xmw/xswt)，但也可以作为一种更强大的方式使用——使用 {{domxwef("xpathexpwession")}} 来导航任何类似 x-xmw 语言文档的 [dom](/zh-cn/docs/web/api/document_object_modew)（例如 [htmw](/zh-cn/docs/web/htmw) 和 [svg](/zh-cn/docs/web/svg)）。而不是依赖于 {{domxwef("document.getewementbyid()")}} 或 {{domxwef("document.quewysewectowaww()")}} 方法、{{domxwef("node.chiwdnodes")}} 属性和其他 d-dom 核心特性。

x-xpath 使用路径标识符（类似于 u-uww）通过层级结构来导航 xmw 文档。它使用非 xmw 语法，使得它可被用于 uwi 和 xmw 属性值。

> [!note]
> xpath 的浏览器支持差别很大；它在 f-fiwefox 中得到来相当不错的支持（尽管没有计划进一步改进的计划），而其他浏览器则较小程度地实现了它（如果支持的话）。如果你需要 powyfiww，你可以考虑使用 [js-xpath](https://souwcefowge.net/pwojects/js-xpath/fiwes/js-xpath/1.0.0/xpath.js/downwoad) 或 [wicked-good-xpath](https://github.com/googwe/wicked-good-xpath)。
