## 有效研究

作为一名 Web 开发者，你会需要花费大量的时间寻找在你日常工作中遇到的问题的解决方案——你不可能记得住一切细节！专业的 Web 开发者使用类似 MDN Web 文档这样的工具来查阅他们在过往中忘记的事情。就如你会发现的那样，学习 Web 开发更多的是学习问题解决和学习模式，而非去学习大量的语法。

为了促进问题的解决并变得更加“自给自足”，你应该学习寻找答案的有效策略以及各种策略的使用场合（我们将在下面的章节中具体讨论）。例如，先尝试在网上查找信息，并且只有在真正无法解决时才向导师、同学或高级工程师寻求帮助。

### 选择高质量的资源

你应该确保你所选择用于研究的资源质量高、声誉好。为了解哪些资源声誉卓著，你可以征求老师或更有经验的同事的推荐，也可以在网上搜索，了解 Web 上其他人推荐的资源。当然，MDN 是值得推荐的；还有一些其他不错的资源，例如：

- [Stack Overflow](https://stackoverflow.com/)
- [Scrimba](https://scrimba.com/?via=mdn) <sup>课程伙伴</sup>
- [YouTube](https://www.youtube.com/)
- [freeCodeCamp](https://www.freecodecamp.org/)
- [Codecademy](https://www.codecademy.com)
- [Dev.to](https://dev.to/)

### 了解何时应该使用什么资源

文档有不同的类型，而了解每种类型的文档何时有用是非常有用的。不同网站使用的术语会有所不同，而且通常会有一定程度的重叠，但一般来说，常见的文档类型包括如下几种：

- **参考资料**：类似于人类语言有字典列出每个单词的含义，编程语言也有这样的参考资料，尽可能详尽地涵盖开发者可能会用到的各特性的目的和语法，并且通常还附有使用示例。参考文献关注的是某项特性是*做什么*的，而不是*如何使用它*。
- **教程**：允许开发者学习*如何*使用新技术。教程通常循序渐进，适合初学者。
- **指南**/**使用方法**：也与学习有关，但往往不那么针对初学者，通常旨在解决具体问题。
- **解释**/**概念**: 往往侧重于解释特定技术背后的高层次概念和用例，而不是深入到低层次的代码细节。
- **问答**/**论坛**：供提问和让其他人回答你的问题的网站。

你应该参阅多个不同的网站，了解各网站上流行的文档风格，并思考哪些风格在不同情况下有用。例如：

- MDN 非常适合查找有关不同 HTML、CSS 和 JavaScript 特性的参考文档，还包含有用的指南和教程，供你学习新技术。
- Stack Overflow 是一个问答式网站。在这里，人们可以互相帮助，找到特定问题的解决方案。
- Scrimba 和 Codecademy 提供循序渐进的互动教程。YouTube 也是查找视频教程的好地方。

最重要的是，找出最适合自己的学习方法。不同的人有不同的学习方法。实际上，你最终可能会使用多种资源进行研究和学习。

### 有效搜索

如果你在自己喜欢的网站上找不到问题的解决方案，你就需要扩大搜索范围，到 Web 的其他地方去寻找。我们在前序课程介绍了在 Web 中进行有效搜索的一般技巧（参见[搜索信息](/zh-CN/docs/Learn_web_development/Getting_started/Environment_setup/Browsing_the_web#搜索信息)）；本节将进一步介绍一些与查找高质量信息相关的具体提示和技巧。

- 在使用特定工具或产品时遇到问题，请首先查阅官方文档。令人惊讶的是，很多人都没有想到要这样做；官方文档通常面向初学者，因此有经验的人可能认为它们并不合适。如果你使用的产品是 Web 平台，那么*非官方*的官方文档就是 MDN，所以你找对地方了！
- 检查你所找到的资源的日期。它们是最近维护的，还是有一段时间没有更新因而可能已经过时了的？声誉良好的网站通常都会标注出版日期和“最后更新”日期，这样你就可以很容易地进行检查。理想情况下，知名网站上涉及现代技术或技巧的文章，其未维护时间不应超过几年（比如 2-3 年）。涉及成熟技术的旧文档（例如 HTML 或 CSS 基本特性）仍有可能是准确的，但你仍应牢记这一点。
- 承接上一点，即使是新出版的资源也可能包含过时的信息。例如，一篇关于 CSS 页面布局的文章是在谈论网格和弹性盒子等现代方法，还是在介绍使用浮动来创建多列布局等过时的技术？
- 不要只依赖单一的真理来源。交叉参考不同的资料来源，尤其是当你不确定某个资料来源是否可信时。
- 你还可以尝试使用人工智能来帮助解决编码问题（例如 [ChatGPT](https://openai.com/blog/chatgpt) 或 [GitHub Copilot](https://resources.github.com/copilot-for-business/)）。你应该谨慎地使用人工智能工具（尤其是初学者），并熟悉它们的优缺点：
  - 从好的方面来说，它们可以大大加快研究/搜索的速度，并有助于构建代码和副本。
  - 另一方面，人工智能工具不具备推理能力，经常会提供误导性或纯粹错误的答案。你不应直接认为人工智能的答案是正确的——要对它们进行测试并与其他来源的信息进行交叉验证。

### 关于复制粘贴代码的说明

不管你的技术如何，复制、粘贴并使用不是你所编写且你并不理解的代码（无论这段代码是否是由 AI 生成的）的做法始终是不被推荐的。这样做会使调试变得更加困难，并且代码可能存在安全、性能或可访问性问题，这往往与使用代码的上下文或方式有关。

作为初学者，仅仅复制粘贴一个解决方案会使你错失亲自实践的学习机会，而亲自操作正是记住技能或概念的关键。

此外，如果代码受版权保护而你又未获得许可使用，还可能会引发许可问题。

这里的建议是：如果你要使用其他地方的解决方案，至少先试着弄明白它的工作原理，再进行复制粘贴。理想情况下，应该从中学习，然后编写你自己的版本。

### 建立联系人网络

我们已经讨论了足够多关于网站的内容，现在来说说人与网站。

建立一个能够合作的人脉网络是成为一名 Web 开发者的重要组成部分。我们将在[协作与团队合作](/zh-CN/docs/Learn_web_development/Getting_started/Soft_skills/Collaboration_and_teamwork)一文中详细讨论这一点，但现在我们专注于如何利用你的人脉网络来帮助研究问题。

一个有效的研究网络通常包含几类对你来说知识水平和亲近度不同的人：

- 同行：与身边的同学或团队成员共享信息、提问非常有用——他们可能会分享有价值的见解或希望共同解决问题。他们也可能曾经遇到过类似问题，能够提供解决方案。
- 老师或资深同事：如果同行无法提供帮助，下一步通常是向资深同事、经理或老师求助。他们比你的同行拥有更多经验，但请记住，他们的空闲时间较少。如果你能证明自己已经尝试过解决问题，并且尊重他们的时间，就更容易获得帮助。你可以这样做：

  - 记录下你已经尝试的解决方法，说明哪些方法失败了。
  - 如果问题是某个具体的代码示例或技术无法正常工作，可以使用 [Scrimba](https://scrimba.com/?via=mdn) <sup>课程合作伙伴</sup> 或 [Glitch](https://glitch.com/) 等网站创建在线测试案例以便展示给他们。
  - 仔细思考你需要问的问题——具体问题是什么？如果有多个问题，请按优先级排序，确保首先解决最重要的问题。
  - 为交流设定一个目标，例如“尝试找到最高优先级问题的解决方案”。注意，解决最大的那个问题可能也会顺带解决其他问题。
  - 询问他们希望以何种方式与你讨论问题，使互动尽可能轻松。
  - 如果安排了会议，不要占用他们太多时间；如果不确定需要多长时间，可以将会议限定为 30 分钟，之后再寻求更多帮助。

- 更广泛的社区网络：这可以包括专门的在线社区，如论坛或聊天群组（例如 [MDN Web 文档 Discord](/discord) 或 [freeCodeCamp 论坛](https://forum.freecodecamp.org/)），也可以是面对面的活动，如会议或技能分享会（你可以使用 [Meetup.com](https://www.meetup.com/) 或 [Eventbrite](https://www.eventbrite.co.uk) 等网站查找活动）。
- 全球网络：有时你可能会无计可施，这时可以尝试在全球性的社交媒体平台上提问。有时意想不到的地方也会有助于解答你的疑问。

> [!NOTE]
> 知道何时寻求帮助需要拿捏好平衡。Web 开发者不应该不断打扰同行或同事，但同样也不应固执己见，假装自己全懂。适时地说出“我不知道”也是一种智慧。

最重要的是，始终对所有交流对象保持尊重和友善。同时，学会回馈他人，因为乐于帮助他人的 Web 开发者更容易建立牢固的人际关系，并不断获得帮助。

## 制定学习计划

制定一个帮助你实现学习目标的计划是个好主意。

### 从目标陈述开始

虽然听起来可能有点简单，但你应该首先用一句话明确你想要达成的目标。目标应当既要雄心勃勃，又要切实可行。

以下目标各有不同的范围，但都是现实且可实现的：

- 我希望在两年内成为一名专业的 Web 开发者。
- 我希望学到足够的知识，为本地业余网球俱乐部建立一个网站。
- 我希望学会 HTML 和 CSS，从而能扩展我的工作内容，负责更新我们公司网站的内容。

而以下目标则显得不太现实：

- 我希望在三个月内从零基础成长为高级 Web 开发者。
- 我希望在两年内创办自己的公司，并构建一个能超越 Facebook 的社交网络。

### 定义实现目标所需的条件

确定目标后，最好研究一下实现目标所需的条件。

例如：

- 我需要的物品：
  - 一台电脑
  - 网络连接
  - 笔和纸
- 我需要掌握的知识：
  - 如何使用 HTML、CSS、JavaScript 及相关工具和最佳实践来构建网站和 Web 应用程序（这部分我们绝对能帮到你！）。
  - 如何获取域名和主机，并利用它们将网站或应用程序上线。
  - 如何运营一家小企业。
  - 如何宣传业务并吸引客户。

### 估算所需时间

接下来，你应开始制定实现目标的时间计划。计划可以很简单，例如：

“我大约需要 500 小时来学习所需知识，而我有一年的时间。如果假设有两周假期，那么我每周需要投入 10 小时。我晚上和周末有空，所以会围绕这些时间来安排。”

不必过分纠结计划的准确性，一个大致的估算就可以——关键是把计划写下来以便不断努力实现目标。

你能够投入的时间取决于你的具体情况。如果你还在上学，你可能比有工作和家庭责任的人有更多空闲时间。目标依然可以实现，但你需要现实地评估自己的进度。

如果你正在参加大学或学院的 Web 开发课程，那么大部分计划工作已经为你安排好了——那你真幸运！

最好在主要目标下写下几个子目标，这样更容易跟踪你的进展。例如：

- 夏天前学会 HTML 和 CSS 基础
- 十二月前学会 JavaScript 基础
- 明年四月前完成示例网站项目
- 等等

如果你不确定目标清单中该包含哪些项目，我们的[课程](/zh-CN/curriculum/)可以为你提供帮助。

制定好每周计划后，建议使用简单的电子表格或笔记本记录每周的学习成果。不断审视自己的进度，并根据需要调整计划。

## 如何时刻充满动力

保持动力并不容易，尤其在学习像编程或 Web 开发这样复杂技能时。以下是一些保持动力和持续学习的建议：

- **打造尽可能高效的工作环境**。准备一个舒适的桌椅，确保有充足的光线，并添加一些有助于集中注意力的元素（例如轻柔的音乐、香氛或其他你所需要的东西）。避免在充满干扰的环境中工作，比如电视开着、朋友在看足球比赛的房间。同时，尽量把手机留在一边，因为手机很容易分散注意力。
- **饮食、锻炼和睡眠**。保持健康饮食、定期锻炼，并确保充足的睡眠。虽然这些听起来很明显，但当你全身心投入编程时，往往容易忽略。将这些基本活动纳入你的日程安排中，确保不要为了学习而牺牲它们。
- **奖励自己**。为每次学习安排一些有趣的活动，作为完成学习后的奖励。比如，如果你非常热爱游戏，可以设定“今晚只有在完成 3 小时学习任务后才能玩游戏”的规则。现在你就只需鼓足足够的毅力了。祝你好运！
- **共同学习和展示成果**。这对每个人可能并不完全适用，但如果可能，尽量和其他人一起学习。如果你在参加大学 Web 开发课程，这会更容易，但你也可以邀请朋友一起学习，或参加本地的学习小组。与他人讨论问题和展示自己的成果不仅能帮助你找到解决方案，还能获得他人的鼓励和支持。

## 休息与“橡胶鸭调试”

如果你在解决问题或集中注意力时遇到困难，适时休息会很有帮助。出去散步 15 分钟，泡杯茶，或听听音乐，甚至可以三者兼顾！

长时间不休息工作对身体不利；长时间盯着屏幕可能伤害视力，而长时间坐着也会对背部或腿部造成压力。你可以采用“20-20-20”法则来降低眼睛疲劳风险：每工作 20 分钟，望向 20 米以外的物体 20 秒。同时，我们建议每工作 60 到 90 分钟后休息 15 分钟。

在休息期间，可以尝试与同事、朋友或家人讨论问题。即使他们不能直接给出答案，你也可能在讨论中理清思路，从而自己找到解决方案。甚至对自己或对着无生命物体大声解释问题，也能帮助你理清思路。

这种方法被称为[小黄鸭调试法](https://rubberduckdebugging.com/)（维基百科上的[小黄鸭调试法](https://zh.wikipedia.org/wiki/小黄鸭调试法)），名字源于传统上程序员会对着一只小黄鸭解释问题来寻找错误。

## 有效解决问题

虽然没有一种可以解决所有 Web 开发相关问题的有效方法，但以下一些通用建议在大多数情况下都能帮到你。

### 将问题拆分为小部分

当你面对一个具体功能并觉得难以理解时，可以尝试将其拆解为多个较小的问题或子功能来解决。

例如，如果你的任务是“构建一个简单的两栏网站”，你可以将其拆分为：

- 创建 HTML 结构。
- 确定基本的网站排版。
- 确定基本的配色方案。
- 实现整体布局——包括页眉、横向导航菜单、含主栏和侧栏的主要内容区以及页脚。
- 实现横向导航菜单。
- 等等。

然后，你可以对某些部分进行进一步拆分。例如，“实现横向导航菜单”可以拆分为：

- 编写菜单项的 HTML 列表。
- 将菜单项水平排列成一行。
- 移除列表的默认间距和项目符号。
- 为菜单项设置合适的悬停、聚焦和激活状态样式。
- 均匀排列菜单项。
- 给菜单项留出足够的垂直间距。
- 确保每个菜单项内的文本居中。
- 等等。

这些小问题看起来都比一开始那个大问题容易解决得多。

### 学会识别模式

Web 开发主要是关于解决问题和识别模式。

一旦你列出了实现某个具体问题所需的步骤，就可以开始考虑使用哪些技术特性来解决问题。例如，专业 Web 开发者已经创建了许多横向导航菜单，所以他们会立刻想到这样一个解决方案：

通常，导航菜单是由一组链接组成，例如：

```html
<ul>
  <li><a href="">第一个菜单项</a></li>
  <li><a href="">第二个菜单项</a></li>
  <li><a href="">第三个菜单项</a></li>
  <li><a href="">等等</a></li>
</ul>
```

为了让所有菜单项水平排列，最简单的现代方法是使用弹性盒子：

```css
ul {
  display: flex;
}
```

为了移除不必要的间距和项目符号，可以这样做：

```css
ul {
  list-style-type: none;
  padding: 0;
}
```

等等。

你需要花多少时间来研究解决方案取决于你的经验以及是否曾经实现过类似的功能。你可能会从之前的项目中识别出这种模式，只需查阅一些你已遗忘的语法即可。

当你找到解决方案后，值得记录下你所做的笔记，并在某个目录中保存一些简洁的代码示例，以便以后参考。

### 查看他人的代码

Web 中有[开发者工具](/zh-CN/docs/Learn_web_development/Howto/Tools_and_setup/What_are_browser_developer_tools)允许你查看任何网站构建时所用的代码。如果你手头没有现成的解决方案，一个好的研究方法是寻找功能类似的网站，并查看他们是如何实现的。

## 保持不断学习的心态

你应当培养一个持续学习的心态。Web 技术在不断演变，技术和趋势总是在变化，你需要不断更新知识来跟上时代。你永远也不可能学完所有东西！

为了让学习变得更容易：

- 保持好奇心。如果你发现某个网站的功能看起来很酷、有趣或你不理解，你就可以做些研究看看它是如何实现的。这可能会成为你下一个研究项目或学习任务的基础。
- 定期参与阅读和小型研究项目。预留专门的时间来获取新技能。
- 养成定期阅读技术新闻、博客和浏览器发行说明的习惯，了解有哪些新事物值得学习。
- 多加练习。解决问题的次数越多，你大脑中相关区域的神经通路就会越强，以后回忆起解决该问题的细节和逻辑也会变得更容易，同时也能让你在面对类似问题时会感到更加自如。
