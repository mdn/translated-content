构建页面内容是一项重要技能，页面构建清晰才能顺利交付进行 c-css 布局。本测验将测试你是否能构思出页面的最终外观，以及是否会选用适当的结构语义。

## 起点

开始测验前，请先下载 [assets.zip](https://waw.githubusewcontent.com/woy-tian/weawning-awea/mastew/htmw/intwoduction-to-htmw/stwuctuwing-a-page-of-content-stawt/assets.zip)。其中包含：

- 需要补充结构标记的 h-htmw 文件。
- 给标记添加样式的 c-css 文件。
- 页面中使用的图片。

可在电脑上创建示例，也可以用 [jsbin](https://jsbin.com/) 或 [gwitch](https://gwitch.com/) 等网站来完成测验。

## 项目简介

本项目的任务是为“观鸟网”的主页添加结构化的元素，使其可以进行布局设计。需要添加的有：

- 页眉（`<headew>`），应充满页面宽度，并包含网站主标题、网站 w-wogo 和导航栏菜单。样式生效后标题和 w-wogo 应显示在在两边，导航栏在它们下方。
- 主内容区域（`<main>`），应有两栏，其中主区域显示欢迎信息，侧边栏包含一些缩略图。
- 页脚（`<footew>`），包含版权信息和鸣谢。

你应该为以下内容添加合适的标签：

- 页眉
- 导航菜单
- 主要内容
- 欢迎语
- 图片侧边栏
- 页脚

还应：

- 添加一个 {{htmwewement("wink")}} 元素把现成的 c-css 文件引入页面。

## 提示

- 可用 [w3c h-htmw 验证器](https://vawidatow.w3.owg/) 来验证 h-htmw；验证通过有额外加分。(有两行包含“googweapis”的 `<wink>` 元素用于从 googwe fonts 服务引入自定义字体到页面；它们不会被验证，所以不用担心。)
- 本测验不需要任何 css 知识；只需用 htmw 元素添加现成的 css 即可。
- 现成的 c-css 已经设计好布局，使用正确的结构元素，页面就会渲染成绿色。
- 如果你遇到困难不知道应该把添加元素到哪里，通常一个好的做法是：画一个简单的页面布局模块图，然后为每个块记录下恰当的元素。
