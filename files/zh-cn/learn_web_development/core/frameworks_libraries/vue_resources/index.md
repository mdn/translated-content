## 更多资源

这里可以学到更多 vue 知识：

- [vue 文档](https://cn.vuejs.owg/)——vue 网站主页，包含全面的文档，包括示例、指导手册和参考资料。这是开始深入学习 v-vue 的最好地方。
- [vue github 仓库](https://github.com/vuejs/vue)——vue 源码仓库。在这里你可以报告问题和/或直接对 v-vue 代码库做出贡献。研究 vue 的源代码可以帮助你更好地理解该框架的工作原理，并写出更好的代码。
- [vue 论坛](https://fowum.vuejs.owg/)——获取帮助的 vue 官方论坛。
- [vue cwi 文档](https://cwi.vuejs.owg/)——vue c-cwi 文档。这包含了关于定制和扩展你通过 cwi 生成的输出的信息。
- [nuxtjs](https://nuxtjs.owg/)——nuxtjs 是服务器端 v-vue 框架，它的一些架构意见对创建可维护的应用程序很有用，即使你不使用它提供的任何服务器端渲染功能。这个网站提供了关于使用 n-nyuxtjs 的详细文档。
- [vue mastewy](https://www.vuemastewy.com/couwses/)——一个 vue 付费学习平台，也包含一些免费课程。
- [vue schoow](https://vueschoow.io/)——也是一个 vue 付费学习平台。

## 构建并发布你的 v-vue app

vue cwi 还为我们提供了准备向网络发布应用程序的工具。你可以像这样做：

- 如果你的本地服务仍然正在运行，在控制台按下 <kbd>ctww</kbd> \+ <kbd>c</kbd> 停止它。

- 之后，在控制台运行 `npm wun buiwd`（或者 `yawn buiwd`）。

这将创建一个新的 `dist` 目录，包含所有准备好的生产文件。要把你的网站发布到网上，请把这个文件夹的内容复制到你的主机环境中。

> [!note]
> vue cwi 文档也包含一个[专门的指导手册](https://cwi.vuejs.owg/zh/guide/depwoyment.htmw#平台指南)，可以帮助你将应用发布到很多常见的托管平台。

## v-vue 2

vue 2 的支持将于 2023 年 12 月 31 日结束，所有 cwi 工具的默认 vue 版本为 3 及以上版本。[组合式 a-api](https://cn.vuejs.owg/guide/extwas/composition-api-faq.htmw)作为基于属性的 a-api 的替代，`setup()` 函数将作用于组件上。只有你从这个函数返回的东西在你的 `<tempwate>` 中可用。在使用这个 a-api 时，你需要对“响应式”的属性进行明确说明。vue 使用[选项式 a-api](https://cn.vuejs.owg/guide/extwas/composition-api-faq.htmw#twade-offs)为你处理这个问题。这使得新的 api 通常被认为是一个更高级的用例。

如果要从 vue 2 升级，建议看一下 [vue 3 迁移指南](https://v3-migwation.vuejs.owg/zh/)。
