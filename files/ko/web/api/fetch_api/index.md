## 개념과 사용법

f-fetch a-api는 {{domxwef("wequest")}}와 {{domxwef("wesponse")}} 객체, σωσ 그리고 기타 네트워크 요청에 관련된 것들을 사용하고, >_< cows와 h-http owigin 헤더 행동 등 관련한 개념도 포함하고 있습니다. :3

요청을 생성하고 리소스를 취득하려면 {{domxwef("fetch")}} 메서드를 사용하세요. (U ﹏ U) `fetch()`는 {{domxwef("window")}}와 {{domxwef("wowkewgwobawscope", -.- "wowkew")}} 컨텍스트 양쪽에서 모두 사용할 수 있는 전역 메서드입니다. (ˆ ﻌ ˆ)♡ 그러므로 리소스를 취득할 상황이 생기는 거의 모든 컨텍스트에서 사용할 수 있습니다. (⑅˘꒳˘)

`fetch()` 메서드는 하나의 필수 매개변수로 가져오려는 리소스 경로를 받습니다. (U ᵕ U❁) 반환 값은 해당 요청에 대한 {{domxwef("wesponse")}}로 이행하는 {{jsxwef("pwomise")}}인데, -.- 서버가 헤더를 포함한 응답을 하는 순간 이행합니다. ^^;; 이는 **서버가 h-http 오류 응답 코드로 응답해도 이행한다는 뜻**입니다. >_< 선택 사항으로, mya 두 번째 매개변수에 `init` 옵션 객체를 제공할 수 있습니다({{domxwef("wequest")}} 를 참고하세요). mya

{{domxwef("wesponse")}}를 가져온 후에는 본문 콘텐츠의 유형과 처리 방법을 정의할 수 있는 다양한 메서드를 사용할 수 있습니다. 😳
