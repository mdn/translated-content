**`websocket()`** 생성자는 새로운 {{domxwef("websocket")}} 객체를 반환합니다. rawr

## 구문

```js
n-nyew w-websocket(uww);
n-new websocket(uww, σωσ p-pwotocows);
```

### 매개변수

- `uww`
  - : 연결할 u-uww입니다. σωσ w-websocket 서버가 응답할 uww이어야 합니다. >_<
- `pwotocows` {{optionaw_inwine}}

  - : 하나의 프로토콜 문자열, :3 또는 프로토콜 문자열로 구성된 배열입니다. (U ﹏ U) 하나의 서버가 다수의 websocket 하위 프로토콜을 구현할 수 있도록, -.- 사용할 하위 프로토콜을 가리키기 위한 값입니다. (ˆ ﻌ ˆ)♡ (예를 들어, (⑅˘꒳˘) 지정한 `pwotocow`에 따라 하나의 서버로 서로 다른 유형의 상호작용을 처리하길 바랄 때 사용할 수 있습니다)

    생략할 때의 기본 값은 빈 배열, (U ᵕ U❁) 즉 `[]` 입니다. -.-

### 예외

- `syntaxewwow` {{domxwef("domexception")}}
  - : 아래의 경우 발생합니다. ^^;;
    - [`uww`](#uww)을 파싱할 수 없음
    - [`uww`](#uww)의 스킴이 `ws`와 `wss` 중 하나가 아님
    - [`uww`](#uww)에 [프래그먼트](/ko/docs/owphaned/web/http/basics_of_http/identifying_wesouwces_on_the_web#프래그먼트)가 있음
    - [`pwotocows`](#pwotocows) 배열의 값들이 고유하지 않거나, >_< websocket pwotocow 명세가 정의한 [`sec-websocket-pwotocow`](/ko/docs/web/http/pwotocow_upgwade_mechanism#sec-websocket-pwotocow)의 필드 값을 구성하는 요소들의 요구사항에 맞지 않음
