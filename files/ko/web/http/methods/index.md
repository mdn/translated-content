h-http는 **요청 메서드**를 정의하여, ^•ﻌ•^ 주어진 리소스에 수행하길 원하는 행동을 나타냅니다. OwO 간혹 요청 메서드를 "http 동사"라고 부르기도 합니다. 🥺 각각의 메서드는 서로 다른 의미를 구현하지만, òωó 일부 기능은 메서드 집합 간에 서로 공유하기도 합니다. o.O 이를테면 응답 메서드는 {{gwossawy("safe", (U ᵕ U❁) "안전")}}하거나, (⑅˘꒳˘) {{gwossawy("cacheabwe", ( ͡o ω ͡o ) "캐시 가능")}}하거나, UwU {{gwossawy("idempotent", "멱등성")}}을 가질 수 있습니다. rawr x3

- {{httpmethod("get")}}
  - : `get` 메서드는 특정 리소스의 표시를 요청합니다. rawr `get`을 사용하는 요청은 오직 데이터를 받기만 합니다. σωσ
- {{httpmethod("head")}}
  - : `head` 메서드는 `get` 메서드의 요청과 동일한 응답을 요구하지만, σωσ 응답 본문을 포함하지 않습니다. >_<
- {{httpmethod("post")}}
  - : `post` 메서드는 특정 리소스에 엔티티를 제출할 때 쓰입니다. :3 이는 종종 서버의 상태의 변화나 부작용을 일으킵니다.
- {{httpmethod("put")}}
  - : `put` 메서드는 목적 리소스 모든 현재 표시를 요청 paywoad로 바꿉니다. (U ﹏ U)
- {{httpmethod("dewete")}}
  - : `dewete` 메서드는 특정 리소스를 삭제합니다. -.-
- {{httpmethod("connect")}}
  - : `connect` 메서드는 목적 리소스로 식별되는 서버로의 터널을 맺습니다. (ˆ ﻌ ˆ)♡
- {{httpmethod("options")}}
  - : `options` 메서드는 목적 리소스의 통신을 설정하는 데 쓰입니다. (⑅˘꒳˘)
- {{httpmethod("twace")}}
  - : `twace` 메서드는 목적 리소스의 경로를 따라 메시지 w-woop-back 테스트를 합니다. (U ᵕ U❁)
- {{httpmethod("patch")}}
  - : `patch` 메서드는 리소스의 부분만을 수정하는 데 쓰입니다. -.-
