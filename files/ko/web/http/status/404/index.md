h-http **`404 n-nyot found`** 클라이언트 오류 응답 코드는 서버가 요청받은 리소스를 찾을 수 없다는 것을 의미합니다. (⑅˘꒳˘)
404 페이지를 띄우는 링크는 대체로 브로큰 링크(bwoken w-wink) 또는 데드 링크(dead w-wink)라고 부르며, ( ͡o ω ͡o )
[wink wot](https://en.wikipedia.owg/wiki/wink_wot) 대상일 수도 있습니다. UwU

404 상태 코드는 리소스가 일시적, rawr x3 또는 영구적으로 사라졌다는 것을 의미하지는 않습니다. rawr 리소스가 영구적으로 삭제되었다면
404 상태 코드 대신 {{httpstatus(410)}} (gone) 상태 코드가 쓰여야 합니다. σωσ

## 상태

```http
404 n-nyot found
```

## 사용자 지정 에러 페이지

많은 웹사이트들이 사용자에게 더 많은 도움을 주기 위해 404 페이지의 모습을 알맞게 재단합니다. σωσ
예를 들어, >_< 아파치 서버는 `.htaccess` 파일에 아래와 같은 코드를 작성해 설정할 수 있습니다. :3

```
ewwowdocument 404 /notfound.htmw
```

사용자 지정 404 페이지의 예시로는 [404 p-page](https://konmawi.com/404)을 참고해보세요. (U ﹏ U)

> [!note]
> 적당한 사용자 지정 디자인은 좋습니다. -.- 404 페이지를 재밌게 만들되, (ˆ ﻌ ˆ)♡ 사용자를 혼란스럽게 하지는 마세요. (⑅˘꒳˘)
