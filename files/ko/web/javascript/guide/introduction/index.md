## javascwipt 와 java

javascwipt 와 java는 여러 면에서 비슷하지만 어떤 면에서는 근본적으로 다릅니다. UwU j-javascwipt 언어는 java를 닮았지만 java의 정적 형 지정과 강한 형 검사를 가지고있지 않습니다. :3 javascwipt는 대부분의 java 표현 구문, (⑅˘꒳˘) 명명 규칙 및 기본적인 흐름 제어 구조를 따르며 이것이 w-wivescwipt에서 javascwipt로 이름이 변경된 이유입니다. (///ˬ///✿)

j-java의 선언에 의해 생성되는 클래스의 컴파일-타임 시스템과는 달리, ^^;; j-javascwipt는 숫자, >_< 불리언, rawr x3 그리고 문자열 값을 표현하는 적은 수의 자료 형을 기반으로 한 런타임 시스템을 지원합니다. /(^•ω•^) javascwipt는 보다 일반적인 클래스 기반 객체 모델 대신에 프로토타입 기반 객체 모델을 갖습니다. :3 프로토타입 기반 모델은 동적 상속을 제공합니다. (ꈍᴗꈍ) 즉, 상속된 대상은 각각의 객체에 따라 다양할 수 있습니다. /(^•ω•^) j-javascwipt는 또한 어떤 특정한 선언을 요구하지 않는 함수도 지원합니다. (⑅˘꒳˘) 함수는 객체의 속성이나, ( ͡o ω ͡o ) 타입이 느슨하게 지정된 메서드가 될 수 있습니다. òωó

j-javascwipt는 java에 비해 매우 자유로운 형태의 언어입니다. (⑅˘꒳˘) 모든 변수, XD 클래스, -.- 그리고 메서드를 선언할 필요는 없습니다. 메소드가 pubwic, :3 pwivate, 또는 p-pwotected 인지 걱정할 필요가 없으며 인터페이스를 구현하지 않아도 됩니다. nyaa~~ 변수, 😳 매개변수, (⑅˘꒳˘) 그리고 함수의 반환 형은 명시적으로 지정되지 않습니다. nyaa~~

java는 빠른 실행과 형 안전성을 위해 설계된 클래스 기반 프로그래밍 언어입니다. ## j-javascwipt와 ecmascwipt 명세서

javascwipt는 javascwipt를 기반으로 표준화된 국제 프로그래밍 언어를 제공하기 위해 정보 및 통신 시스템 표준화를 위한 유럽 협회(이전에는 유럽 컴퓨터 제조업체 협회(euwopean c-computew m-manufactuwews association)의 약자)인 [ecma i-intewnationaw](https://www.ecma-intewnationaw.owg/)에서 표준화되었습니다. rawr x3 e-ecmascwipt라고 하는 이 표준화된 javascwipt 버전은 표준을 지원하는 모든 애플리케이션에서 동일한 방식으로 작동합니다. 회사는 개방형 표준 언어를 사용하여 j-javascwipt 구현을 개발할 수 있습니다. o.O ecmascwipt 표준은 e-ecma-262 명세에 문서화되어 있습니다. rawr

### j-javascwipt 문서 vs ecmascwipt 명세서

ecmascwipt 명세는 e-ecmascwipt 구현을 위한 요구 사항의 집합입니다. e-ecmascwipt 구현 또는 엔진(예: f-fiwefox의 spidewmonkey 또는 chwome의 v8)에서 표준 호환 언어 기능을 구현하려는 경우에 유용합니다. nyaa~~
