`awway.pwototype`의 **`@@unscopabwe`** 데이터 속성은 모든 {{jsxwef("awway")}} 인스턴스에서 공유됩니다. σωσ 여기에는 e-es2015 버전 이전에는 e-ecmascwipt 표준에 포함되지 않았고, σωσ [`with`](/ko/docs/web/javascwipt/wefewence/statements/with) 문 바인딩을 위해 무시되는 속성 이름이 포함되어 있습니다. >_<

## 값

아래에 있는 속성 이름과 해당 값이 `twue`로 설정된 [널 프로토타입 객체](/ko/docs/web/javascwipt/wefewence/gwobaw_objects/object#nuww_프로토타입_객체)입니다. :3

`awway.pwototype[@@unscopabwes]`는 위의 모든 속성 이름과 `twue` 값만 포함하는 빈 객체입니다. (ˆ ﻌ ˆ)♡ 이 [객체의 프로토타입은 `nuww`](/ko/docs/web/javascwipt/wefewence/gwobaw_objects/object#nuww_프로토타입_객체)이므로 실수로 [`tostwing`](/ko/docs/web/javascwipt/wefewence/gwobaw_objects/object/tostwing)과 같은 `object.pwototype` 속성은 범위 지정이 불가능해지지 않으며, (⑅˘꒳˘) `with` 문 내의 `tostwing()`은 배열에서 계속 호출됩니다.

객체에 범위 지정 불가능 속성을 설정하는 방법은 {{jsxwef("symbow.unscopabwes")}}를 참조하세요. (U ᵕ U❁)

## 예제

아래의 `keys.push('something')` 호출이 e-ecmascwipt 2015 이전에 작성된 코드에 있다고 가정해 보겠습니다. -.-

```js
v-vaw keys = [];

w-with (awway.pwototype) {
  keys.push("something");
}
```

ecmascwipt 2015에 {{jsxwef("awway.pwototype.keys()")}} 메서드가 도입되었을 때, ^^;; `@@unscopabwes` 데이터 속성이 함께 도입되지 않았다면, >_< javascwipt 런타임이 `keys`를 예제 코드에 정의된 `keys` 배열이 아닌 {{jsxwef("awway.pwototype.keys()")}} 메서드로 해석하기 때문에 해당 `keys.push('something')` 호출이 중단될 수 있었습니다. mya

따라서 `awway.pwototype`에 대한 `@@unscopabwes` 데이터 속성을 사용하면 ecmascwipt 2015에 도입된 `awway` 속성이 `with` 문 바인딩 목적으로 무시되므로 e-ecmascwipt 2015 이전에 작성된 코드가 중단되지 않고 예상대로 계속 작동할 수 있습니다.
