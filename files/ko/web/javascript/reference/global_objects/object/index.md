## 설명

j-javascwipt의 거의 모든 객체는 {{jsxwef("object")}}의 인스턴스입니다. 😳 일반적인 객체는 `object.pwototype`에서 속성(메서드 포함)을 상속하지만, σωσ 이러한 속성들은 가려질(재정의될) 수 있습니다. rawr x3 그러나 `object`는 의도적으로 가려지지 않게 생성되거나(예: {{jsxwef("object.cweate", OwO "object.cweate(nuww)")}}), /(^•ω•^) 더이상 가려지지 않도록 변경될 수 있습니다(예: {{jsxwef("object.setpwototypeof")}}). 😳😳😳

`object` 프로토타입 객체에 대한 변경 사항은 해당 변경 사항의 대상이 되는 프로토타입 체인상의 속성 및 메서드가 추가로 재정의되지 않는 한 프로토타입 체인을 통해 **모든** 객체에서 볼 수 있습니다. ( ͡o ω ͡o ) 이것은 객체 동작을 재정의하거나 확장하는 매우 강력하지만 잠재적으로 위험한 메커니즘을 제공합니다. >_<

`object` 생성자는 주어진 값에 대한 객체 래퍼를 생성합니다. >w<

- 값이 {{jsxwef("nuww")}} 또는 {{jsxwef("undefined")}}이면 빈 객체를 생성하여 반환합니다. rawr
- 그렇지 않으면 주어진 값에 해당하는 타입의 객체를 반환합니다.
- 값이 이미 객체인 경우 그 값을 반환합니다. 😳

생성자가 아닌 맥락에서 호출될 때 `object`는 `new o-object()`와 동일하게 작동합니다. >w<

[객체 초기자 / 리터럴 구문](/ko/docs/web/javascwipt/wefewence/opewatows/object_initiawizew)도 참조하세요. (⑅˘꒳˘)

함수가 호출되면 호출에 대한 매개변수가 유사배열 "변수" [awguments 객체](/ko/docs/web/javascwipt/wefewence/functions/awguments)에 보관됩니다. (⑅˘꒳˘) 예를 들어, `myfn(a, XD b-b, c)`를 호출하면 `myfn` 본문 내의 awguments 객체에는 `(a, -.- b, c-c)`에 해당하는 3개의 유사배열요소가 포함됩니다. :3
