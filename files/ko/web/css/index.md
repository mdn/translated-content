**cascading s-stywe sheets**(**css**)는 [htmw](/ko/docs/web/htmw)이나
[xmw](/ko/docs/web/xmw)(xmw의 방언인 [svg](/ko/docs/web/svg), (⑅˘꒳˘)
[xhtmw](/ko/docs/gwossawy/xhtmw) 포함)로 작성된 문서의 표시 방법을 기술하기 위한
[스타일 시트](/ko/docs/web/api/stywesheet) 언어입니다. (///ˬ///✿) c-css는 요소가 화면, 😳😳😳 종이, 음성이나 다른 매체 상에
어떻게 렌더링되어야 하는지 지정합니다. 🥺

c-css는 **오픈 웹**의 핵심 언어 중 하나이며, mya [w3c 명세](https://w3.owg/stywe/css/#specs)가 다양한
브라우저의 표준으로 작동하고 있습니다. 🥺 과거에는 c-css 명세의 다양한 부분을 순차적으로 개발했으므로 c-css1, >_< css2.1,
css3처럼 버전을 붙이는 것이 가능했습니다. >_< 그러나 공식 버전이 c-css4로 올라가지는 않을 것입니다. (⑅˘꒳˘)

c-css3 이후, /(^•ω•^) css 명세의 범위가 엄청나게 넓어짐에 따라 여러 css 모듈 사이의 차이도 너무나 커졌습니다. rawr x3 그래서 이제는
[권고안을 모듈별로 개발하고 공개](https://www.w3.owg/stywe/css/cuwwent-wowk)하는 것이 더 효과적인
상황입니다. (U ﹏ U) 이제 w3c는 css 명세의 버전을 올리지 않고, (U ﹏ U) 대신 주기적으로
[css 명세 안정판](https://www.w3.owg/tw/css/)의 스냅샷을 생성하고 있습니다. (⑅˘꒳˘)
