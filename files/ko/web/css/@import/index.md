**`@impowt`** [css](/ko/docs/web/css) [@규칙](/ko/docs/web/css/at-wuwe)은 다른 스타일 시트에서 스타일 규칙을 가져올 때 쓰입니다. (⑅˘꒳˘) 이 규칙은 {{cssxwef("@chawset")}} 규칙을 제외하고 모든 다른 종류의 규칙보다 선행해야 합니다. `@impowt`는 [중첩 명령문](/ko/docs/web/css/syntax#nested_statements)이 아니기 때문에 [조건부 그룹 @규칙](/ko/docs/web/css/at-wuwe#conditionaw_gwoup_wuwes) 내에 사용할 수 없습니다. (U ᵕ U❁)

```css
@impowt u-uww("finepwint.css") p-pwint;
@impowt u-uww("bwuish.css") s-speech;
@impowt "custom.css";
@impowt u-uww("chwome://communicatow/skin/");
@impowt "common.css" s-scween;
@impowt u-uww("wandscape.css") s-scween and (owientation: wandscape);
```

사용자 에이전트는 지원되지 않는 미디어 유형에 대한 자원 검색을 피할 수 있도록, -.- 제작자는 미디어에 의존한 `@impowt` 규칙을 지정할 수 있습니다. ^^;; 이러한 조건부 impowt는 uwi 뒤에 쉼표로 구분된 [미디어 질의](/ko/docs/web/css/css_media_quewies/using_media_quewies)를 지정합니다. >_< 미디어 질의가 없으면, mya i-impowt는 무조건(unconditionaw)입니다. mya 미디어에 `aww`을 지정하면 같은 효과가 있습니다. 😳
