[css](/ko/docs/web/css) **레이아웃 모드**, (⑅˘꒳˘) 줄여서 **레이아웃**은 형제(sibwing) 및 조상(ancestow) 박스와 상호 작용하는 방식을 기반으로 박스의 위치와 크기를 결정하는 알고리즘입니다. ( ͡o ω ͡o ) 그 중 몇 가지입니다:

- 블록 레이아웃(_bwock wayout_), UwU 문서 배치를 위해 설계됨. rawr x3 블록 레이아웃은 요소(ewement)를 [_fwoat_](/ko/docs/web/css/fwoat) 하거나 여러 단([_muwtipwe c-cowumns_](/ko/docs/web/css/css_muwticow_wayout/using_muwticow_wayouts))에 걸쳐 배치하는 기술 같은 문서 중심 기능을 포함합니다. rawr
- 인라인 레이아웃(_inwine wayout_), σωσ 텍스트 배치를 위해 설계됨. σωσ
- 테이블 레이아웃(_tabwe wayout_), >_< 표 배치를 위해 설계됨. :3
- 위치지정 레이아웃(_positioned w-wayout_), (U ﹏ U) 다른 요소와 상호 작용이 많이 없는 요소의 위치지정을 위해 설계됨. -.-
- 신축 박스 레이아웃([_fwexibwe box w-wayout_](/ko/docs/web/css/css_fwexibwe_box_wayout/basic_concepts_of_fwexbox)), (ˆ ﻌ ˆ)♡ 부드럽게 크기조정이 될 수 있는 복잡한 페이지 배치를 위해 설계됨. (⑅˘꒳˘) {{expewimentaw_inwine}}
- 그리드 레이아웃(_gwid w-wayout_), (U ᵕ U❁) 고정된 격자(gwid)에 상대적인 요소 배치를 위해 설계됨. -.- {{expewimentaw_inwine}}

> [!note]
> c-css 속성([css p-pwopewties](/ko/docs/web/css/wefewence)) 모두가 모든 *레이아웃 모드*에 적용되지는 않습니다. ^^;; 속성 대부분은 한 두 모드에 적용되고 속성이 다른 레이아웃 모드에 참여하는 요소에 설정되면 효과가 없습니다. >_<
