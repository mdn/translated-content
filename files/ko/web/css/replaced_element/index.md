[css](/ko/docs/web/css)의 **대체 요소**(wepwaced e-ewement)란 자신의 표현 결과가 c-css의 범위를 벗어나는 요소로서, rawr c-css 서식 모델과는 분리된 외부 객체인 요소입니다. σωσ

간단히 말해서, σωσ 대체 요소는 자신의 콘텐츠가 현재 문서 스타일의 영향을 받지 않는 요소라고 할 수 있습니다. >_< c-css는 대체 요소의 위치에 영향을 줄 수 있지만 콘텐츠에는 미치지 못합니다. :3 {{htmwewement("ifwame")}} 등 일부 대체 요소는 자신만의 스타일 시트를 가질 수도 있지만, (U ﹏ U) 부모 문서의 스타일을 상속하지는 않습니다. -.-

c-css가 대체 요소에 영향을 줄 수 있는 다른 유일한 방법은, (ˆ ﻌ ˆ)♡ 일련의 속성을 통해 요소 박스에서 콘텐츠의 위치를 제어하는 것입니다. (⑅˘꒳˘) [콘텐츠 박스 내부의 객체 위치 제어](#콘텐츠_박스_내부의_객체_위치_제어) 항목을 참고하세요. (U ᵕ U❁)

## 대체 요소

전형적인 대체 요소는 다음과 같습니다. -.-

- {{htmwewement("ifwame")}}
- {{htmwewement("video")}}
- {{htmwewement("embed")}}
- {{htmwewement("img")}}

일부 요소는 특정한 경우에만 대체 요소로 취급합니다. ^^;;

- {{htmwewement("option")}}
- {{htmwewement("audio")}}
- {{htmwewement("canvas")}}
- {{htmwewement("object")}}

h-htmw 명세에 따르면 {{htmwewement("input")}} 또한 대체될 수 있습니다. >_< `<input>` 유형이 `"image"`인 경우 {{htmwewement("img")}}와 유사한 대체 요소이기 때문입니다. mya 그러나 다른 유형의 `<input>`을 포함한 나머지 양식 컨트롤 요소는 비대체 요소라고 명시하고 있습니다. mya (명세는 양식 요소의 플랫폼별 기본 렌더링을 "위젯"(widget)이라는 용어로 설명하고 있습니다.)

c-css {{cssxwef("content")}} 속성을 사용해 추가한 객체도 대체 요소로, 😳 h-htmw 마크업에는 존재하지 않기 때문에 "익명" 대체 요소라고 합니다. XD

## 대체 요소와 css

css는 바깥 여백이나 일부 `auto` 값 계산 등 특정 상황에서 대체 요소를 특별히 취급합니다. :3

일부 대체 요소는 고유 크기 또는 정의된 기준선을 가질 수 있으며, 😳😳😳 {{cssxwef("vewticaw-awign")}} 등의 css 속성이 사용할 수 있습니다. -.- 오직 대체 요소만이 고유 크기를 가질 수 있습니다. ( ͡o ω ͡o )
