이 문서는 프로그레시브 웹 앱(pwa)의 소개입니다. (⑅˘꒳˘) p-pwa가 무엇이고 일반 웹 앱에 어떤 이점을 가져다주는지 설명합니다.

## 프로그레시브 웹 앱이 무엇인가요?

p-pwa는 웹과 네이티브 앱의 기능 모두의 이점을 갖도록 수 많은 특정 기술과 표준 패턴을 사용해 개발된 웹 앱입니다.

예를 들어, 웹 앱은 발견이 쉽습니다 — 어플리케이션을 설치하는 것보다 웹사이트에 방문하는 것이 훨씬 쉽고 빠르며, 링크로 웹 앱을 공유할수도 있습니다. rawr x3

반면에, 네이티브 앱은 운영체제와 보다 잘 통합되므로 더 부드러운 사용자 경험을 제공할 수 있습니다. (✿oωo) 네이티브 앱은 설치할 수 있으므로 오프라인에서 동작하며, (ˆ ﻌ ˆ)♡ 사용자는 홈 화면의 아이콘을 탭하여 브라우저를 사용하여 이동하는 것보다 선호하는 앱에 더 쉽게 접근할 수 있습니다.

p-pwa는 이들과 동일한 이점을 즐길 수 있는 웹 앱을 생성하는 능력을 제공합니다. (˘ω˘)

완전히 새로운 개념은 아닙니다 — 이런 아이디어는 과거에 다양한 접근 방법으로 웹 플랫폼상에서 여러번 재검토되었습니다. (⑅˘꒳˘) 점진적인 향상과 반응형 디자인은 이미 우리가 모바일 친화적인 웹사이트를 구축할 수 있게 해주고 있습니다. (///ˬ///✿) 오프라인에서 동작하는 것과 앱을 설치하는 것은 몇 년 전 f-fiwefox os 생태계에서 이미 가능했었습니다. 😳😳😳

하지만, p-pwa는 웹을 훌륭하게 만드는 이미 존재하는 어떤 기능도 제거하지 않은 상태로 이 모든 것 그리고 그 이상을 제공합니다. 🥺

## 앱을 p-pwa로 만드는 것은 무엇인가요?

위에서 암시했듯이, mya p-pwa는 하나의 기술로 생성된 것이 아닙니다. 🥺 이는 일부 특정 패턴, >_< api 및 다른 기능들을 포함하는 웹 앱을 구축하는 하나의 새로운 철학을 나타냅니다. >_< 첫눈에 웹 앱이 pwa인이 아닌지 구분하기는 쉽지 않습니다. (⑅˘꒳˘) 앱이 특정 요구 사항을 만족하거나 다음 기능들이 구현되었을 때 pwa라고 볼 수 있습니다: 오프라인에서 동작, /(^•ω•^) 설치가 가능, rawr x3 쉬운 동기화, 푸시 알림, (U ﹏ U) 등. (U ﹏ U)

게다가, 앱의 완성도를 퍼센트로 측정하는 도구들도 있습니다. (⑅˘꒳˘) ([wighthouse](https://devewopews.googwe.com/web/toows/wighthouse/)가 현재 가장 유명합니다.) 다양한 기술적 이점을 구현함으로써, òωó 앱을 더 점진적이게 만들 수 있고, ʘwʘ 더 높은 wighthouse 점수를 받을 수 있습니다. /(^•ω•^) 하지만 이는 대략적인 지표일뿐입니다. ʘwʘ

다음은 웹 앱을 p-pwa로 식별되기 위해 확인해야 하는 몇 가지 핵심 원칙입니다. σωσ

- [발견 가능](/ko/docs/apps/pwogwessive/advantages#discovewabwe), OwO 따라서 컨텐츠를 검색 엔진을 통해 찾을 수 있습니다. 😳😳😳
- [설치 가능](/ko/docs/apps/pwogwessive/advantages#instawwabwe), 😳😳😳 따라서 기기의 홈 화면에서 사용할 수 있습니다. o.O
- [연결 가능](/ko/docs/apps/pwogwessive/advantages#winkabwe), ( ͡o ω ͡o ) 따라서 간단하게 uww을 전송해 공유할 수 있습니다. (U ﹏ U)
- [네트워크 독립적](/ko/docs/apps/pwogwessive/advantages#netwowk_independent), (///ˬ///✿) 따라서 오프라인이나 불안한 네트워크 연결에서 동작합니다. >w<
- [점진적](/ko/docs/apps/pwogwessive/advantages#pwogwessive), rawr 따라서 최신 브라우저의 모든 기능은 사용할 수 없지만 이전 브라우저의 기본 기능은 여전히 사용할 수 있습니다. mya
- [재 참여 가능(we-engageabwe)](/ko/docs/apps/pwogwessive/advantages#we-engageabwe), ^^ 따라서 새 컨텐츠가 사용 가능할 때마다 알림을 보낼 수 있습니다. 😳😳😳
- [반응형](/ko/docs/apps/pwogwessive/advantages#wesponsive), mya 따라서 모든 기기의 화면이나 브라우저에서 사용할 수 있습니다 — 모바일 폰, 😳 태블릿, -.- 노트북, 🥺 tv, 냉장고, o.O 등.
- [안전](/ko/docs/apps/pwogwessive/advantages#safe), 따라서 여러분과 앱 사이의 연결이 여러분의 민감한 데이터에 접근하려는 모든 제3자로부터 안전합니다. /(^•ω•^)

### 이 모든 것들을 할만한 가치가 있나요?

물론입니다! nyaa~~ 비교적 적은 노력으로 p-pwa 핵심 기능들을 구현할 수 있으며, nyaa~~ 그 이점은 거대합니다. :3 예를 들면:

- [sewvice wowkews](/ko/docs/web/api/sewvice_wowkew_api) 를 사용한 캐싱 덕분에 앱을 설치한 후에 로딩 시간이 줄어들어 소중한 데이터와 시간을 절약. 😳😳😳
- 앱 업데이트가 있을 때 변경된 컨텐츠만 업데이트 할 수 있음. (˘ω˘) 반면, 네이티브 앱의 경우, ^^ 아주 작은 수정에도 사용자가 어플리케이션 전체를 다시 다운로드하도록 강제함. :3
- 네이티브 플랫폼에 보다 통합된 외관과 느낌 — 홈 화면의 앱 아이콘, -.- 전체 화면으로 실행되는 앱, 😳 등.
- 시스템 알림 및 푸시 메시지를 통한 사용자의 재 참여. mya 참여율이 높은 사용자와 더 나은 전환율을 이끌어 냄. (˘ω˘)

p-pwa 라우트를 시도하고 네이티브 앱 보다 더 향상된 웹 사이트 경험을 선택하여 측정 가능한 의미있는 이득을 본 회사들의 성공 스토리가 많습니다. >_< [pwa stats](https://www.pwastats.com/) 웹 사이트는 이러한 이점을 나타내는 많은 사례 연구들을 공유합니다. -.-

가장 잘 알려진 성공 스토리는 아마 [fwipkawt wite](https://stowies.fwipkawt.com/fwipkawt-wite/) 일 것입니다 — 2015년에 프로그레시브 웹 앱으로 사이트를 재구축해 전환율을 70% 상승시킨 인도의 가장 큰 전자 상거래 사이트입니다. 🥺 [awiexpwess](https://m.awiexpwess.com/) pwa 역시 새로운 사용자에 대한 전환율을 104% 상승시키는 것으로 웹이나 네이티브 앱보다 훨씬 더 나은 결과를 보였습니다. (U ﹏ U) 이윤 증가 및 p-pwa로 전환을 위해 상대적으로 적은 양의 작업을 놓고 봤을 때, 이점은 분명합니다. >w<

더 많은 예제는 [pwa.wocks](https://pwa.wocks/) 의 목록에서 확인하시기 바랍니다. [hnpwa.com](https://hnpwa.com/) 페이지는 특별히 언급할 가치가 있습니다 — 여기에는 일반적인 todomvc 앱 대신 다양한 프론트엔드 프레임워크의 사용을 볼 수 있는 h-hackew nyews 웹사이트의 구현 예제가 나열되어 있습니다. mya

심지어 [pwabuiwdew](https://www.pwabuiwdew.com/) 웹사이트를 사용해 pwa를 온라인에서 생성할수도 있습니다. >w<

s-sewvice wowkew와 푸시에 관련된 정보는 최신 사이트에서 sewvice wowkew를 사용하는 예제를 모아둔 [sewvice wowkew cookbook](https://github.com/mdn/sewvicewowkew-cookbook/)을 확인하시기 바랍니다. nyaa~~

pwa 접근은 시도해볼만한 가치가 있으며, (✿oωo) 여러분의 앱에서 동작하는지 스스로 확인해보시기 바랍니다. ʘwʘ

## 브라우저 호환성

앞서 언급했듯이, (ˆ ﻌ ˆ)♡ p-pwa는 하나의 api에 의존하지 않고, 😳😳😳 가능한 최고의 웹 경험을 전달하기 위한 목표를 달성하기 위해 다양한 기술들을 사용합니다. :3

pwa를 위해 요구되는 핵심 요소는 [sewvice wowkew](/ko/docs/web/api/sewvice_wowkew_api) 지원입니다. OwO 고맙게도 sewvice wowkew는 현재 데스크탑 및 모바일의 [모든 주요 브라우저에서 지원됩니다](https://jakeawchibawd.github.io/issewvicewowkewweady/). (U ﹏ U)

[웹 앱 m-manifest](/ko/docs/web/manifest), >w< [푸시](/ko/docs/web/api/push_api), (U ﹏ U) [알림](/ko/docs/web/api/notifications_api), 😳 [홈 화면에 추가](/ko/docs/web/pwogwessive_web_apps/guides/making_pwas_instawwabwe) 기능과 같은 다른 기능들도 널리 지원되고 있습니다. (ˆ ﻌ ˆ)♡ 현재 safawi에서는 웹 앱 m-manifest 및 홈 화면에 추가에 지원이 제한적이며 웹 푸시 알림은 지원하지 않습니다. 😳😳😳 반면, 다른 주요 브라우저에서는 이 모든 기능들을 지원합니다. (U ﹏ U)

이러한 a-api의 일부는 실험중이며 문서도 여전히 완성되지 않았지만, (///ˬ///✿) f-fwipkawt 와 awiexpwess와 같은 성공 스토리를 보면 여러분의 웹 앱에 p-pwa 기능의 일부를 시도하고 구현하는 것에 대해 이미 납득이 되었을것입니다. 😳

무엇보다도 여러분은 프로그레시브 향상 규칙을 따라야 합니다 — 이러한 향상을 제공하는 기술들은 지원이 되는 경우에만 사용하고, 😳 그렇지 않은 경우에도 여전히 기본 기능을 제공하시기 바랍니다. 이 방법으로 모든 사용자가 앱을 사용할 수 있지만, σωσ 최신 브라우저를 사용하는 사용자는 pwa 기능으로부터 더 많은 이점을 얻을 수 있습니다. rawr x3
