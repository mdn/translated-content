**REST**(Representational State Transfer)는 효율적, 안정적이며 확장가능한 분산시스템을 가져올 수 있는 소프트웨어 아키텍처 디자인 제약의 모음을 나타냅니다.

REST의 기본 개념은 리소스입니다. 리소스의 예로는 잘 인식되고 언어에 상관없이, 안정적이고 표준화된 클라이언트/서버 상호작용을 통해 전송되는 문서를 들 수 있습니다. 서비스가 이러한 제약조건을 준수하면 RESTful로 간주됩니다.

일반적으로 HTTP API는 모든 REST 제약 조건을 반드시 준수하지는 않지만 RESTful API, RESTful 서비스 또는 REST 서비스라고도 합니다. 초보자는 REST API가 표준 웹 라이브러리 및 도구를 사용하여 호출할 수 있는 HTTP 서비스를 의미한다고 가정할 수 있습니다.
