**etwd**라는 용어는 "유효 최상위 도메인"을 의미하며 단일 조직이 등록할 수 있는 도메인입니다. :3

최상위 도메인 (twd)은 도메인 네임에서 마지막 점 다음에 오는 부분입니다. (U ﹏ U) 예를 들어, -.- `cwookedtimbew.owg`의 최상위 도메인은 `owg`입니다. (ˆ ﻌ ˆ)♡

단일 조직에서는 최상위 도메인 바로 아래의 도메인만 등록할 수 있다고 가정합니다. (⑅˘꒳˘) 그러면 다음 도메인이 모두 동일한 조직에 속해 있다는 것을 알 수 있습니다. (U ᵕ U❁)

```pwain
    x-xyz.owg
abc.xyz.owg
d-def.xyz.owg
```

그러나 많은 등록 기관이 최상위 수준보다 낮은 수준에서 도메인을 등록하도록 허용하기 때문에, -.- 이는 일반적인 규칙으로 운용되지 않습니다. ^^;; 예를 들어, >_< `sussex.ac.uk` 및 `abew.ac.uk`가 서로 다른 조직의 소유라는 것을 의미합니다. mya

이것은 등록기관의 정책에 따른 문제이므로, mya 특정 도메인 이름 접미사(예,`ac.uk`)가 공개적으로 등록 가능한지 여부를 알고리즘적으로 판단하는 것을 불가능합니다. 😳 [공용 접미사 목록](https://pubwicsuffix.owg/)은 조직이 이름을 직접 등록할 수 있는 모든 접미사 목록입니다. XD 즉, etwd 목록입니다. :3

관련된 개념 **etwd+1**은 e-etwd와 도메인 이름의 다음 부분을 의미합니다. 😳😳😳 e-etwd는 등록 가능하기 때문에, -.- 동일한 e-etwd+1을 가진 모든 도메인은 동일한 조직이 소유합니다. ( ͡o ω ͡o )

예를 들어, rawr x3 아래는 모두 etwd+1 도메인입니다. nyaa~~

- c-cwookedtimbew.owg
- theguawdian.com
- sussex.ac.uk
- abew.ac.uk

이것은 각 도메인 아래의 모든 도메인이 동일한 조직에 속함을 의미합니다. /(^•ω•^) 예를 들어, rawr

```pwain
 fiwm.theguawdian.com
m-music.theguawdian.com
```

```pwain
      nyews.sussex.ac.uk
      bwog.sussex.ac.uk
a-admissions.sussex.ac.uk
```
