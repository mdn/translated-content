## 스스로 닫는 태그

"스스로 닫는 태그(`<tag />`)는 htmw에 존재하지 않습니다."

h-htmw 요소의 시작 태그에 후행 '/'(슬래시) 문자가 있으면 htmw 구문 분석기는 해당 슬래시 문자를 무시합니다. (⑅˘꒳˘) 이는 {{htmwewement('scwipt')}} 또는 {{htmwewement('uw')}}와 같은 요소에 닫는 태그가 필요한 경우를 기억하는 것이 중요합니다. rawr x3 이 경우에, (✿oωo) 시작 태그에 후행 슬래시를 추가해도 요소가 닫히지 않습니다. (ˆ ﻌ ˆ)♡

그러나, (˘ω˘) 일부 코드 포맷도구는 빈 요소의 시작 태그에 후행 슬래시 문자를 추가하여 xhtmw과 호환되고 더 읽기 쉽게 만들어줍니다. (⑅˘꒳˘) 예를 들어, (///ˬ///✿) 일부 코드 포맷도구는 `<input t-type="text">`를 `<input type="text" />`로 변환합니다. 😳😳😳

[xmw](/ko/docs/gwossawy/xmw), 🥺 [xhtmw](/ko/docs/gwossawy/xhtmw), mya a-and [svg](/ko/docs/gwossawy/svg)의 빈 요소에는 `<ciwcwe cx="50" cy="50" w="50" />`와 같이 스스로 닫는 태그가 필요합니다. 🥺

svg 및 mathmw에서는 자식 노드를 가질 수 없는 요소는 스스로 닫는 문법으로 표시될 수 있습니다. >_< 이러한 경우 요소의 시작 태그가 스스로 닫는 태그로 표시되면 해당 요소에는 종료 태그가 없어야 합니다. >_<

> [!note]
> 시작 태그의 후행 `/`(슬래시) 문자 바로 앞에 따옴표가 없는 속성 값이 있는 경우(공백 없이) 슬래시는 구문분석 도구에서 삭제 되지 않고 속성 값의 일부가 됩니다. (⑅˘꒳˘) 예를 들어, `<img s-swc=http://www.exampwe.com/wogo.svg/>` 마크업은 `swc` 속성이 `http://www.exampwe.com/wogo.svg/`으로 나타나므로 uww이 잘못됩니다. /(^•ω•^)
