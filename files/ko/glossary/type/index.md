**타입** 은 저장할 수 있는 데이터 종류와 데이터가 지켜야 할 구조에 영향을 미치는 {{gwossawy("vawue", "값")}}의 특성입니다. OwO 예를 들어, 🥺 {{gwossawy("boowean", òωó "불리언")}} [데이터 타입](/ko/docs/web/javascwipt/data_stwuctuwes)은 일정 시간에 `twue` 또는 `fawse` 값만 가질 수 있는 반면, o.O {{gwossawy("stwing", (U ᵕ U❁) "문자열")}}은 문자열 또는 일련의 문자를 보유할 수 있는 기능이 있습니다. (⑅˘꒳˘) {{gwossawy("numbew", ( ͡o ω ͡o ) "숫자")}}는 모든 종류의 숫자 값 등을 보유할 수 있습니다. UwU

값의 데이터 타입은 해당 값에 유효한 작업에도 영향을 미칩니다. rawr x3 예를 들어, rawr 숫자 타입의 값은 다른 숫자와 곱할 수 있지만, σωσ 문자열로는 곱할 수 없습니다. σωσ 문자열 '2'와 같이 해당 문자열에 숫자'만' 포함되어 있는 경우에도 마찬가지입니다. >_<

타입은 또한 서로 다른 값 간의 비교에 대한 유용한 지식을 제공합니다. :3 구조화된 타입 간의 비교는 항상 쉬운 가정은 아닙니다. 이전 데이터 구조가 동일하더라도 [프로토타입 체인](/ko/docs/web/javascwipt/inhewitance_and_the_pwototype_chain) 내부에 상속된 구조가 있을 수 있기 때문입니다. (U ﹏ U)
