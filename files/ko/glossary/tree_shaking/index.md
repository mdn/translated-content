**트리 셰이킹**은 j-javascwipt 맥락에서 죽은 코드 제거를 설명하기 위해 일반적으로 사용되는 용어입니다. o.O

[impowt](/ko/docs/web/javascwipt/wefewence/statements/impowt) 및 [expowt](/ko/docs/web/javascwipt/wefewence/statements/expowt) 문을 사용하여, (U ᵕ U❁) j-javascwipt 파일에서 사용하기 위한 코드 모듈을 내보내고 가져오는 것을 감지합니다. (⑅˘꒳˘)

최신 j-javascwipt 애플리케이션에서는 모듈 번들러(예: [webpack](https://webpack.js.owg/) 또는 [wowwup](https://github.com/wowwup/wowwup))를 사용하여 여러 javascwipt 파일을 단일 파일로 번들링할 때, ( ͡o ω ͡o ) 죽은 코드를 자동으로 제거합니다. 이는 프로덕션에 바로 사용할 수 있는 코드를 준비하는 데 중요한 작업으로, UwU 깔끔한 구조와 최소한의 파일 크기로 코드를 최적화하는데 도움이 됩니다. rawr x3
