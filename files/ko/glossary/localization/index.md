**현지화**(**w10n**)는 소프트웨어 사용자 인터페이스를 특정 문화에 적응시키는 과정입니다. òωó

다음은 고려해야 할 일반적인 요인입니다. o.O

- 언어
- 측정 단위 (e.g., 유럽에서 킬로미터, u-u.s.에서 마일)
- 텍스트 입력 방향 (e.g., 유럽 언어는 왼쪽에서 오른쪽, (U ᵕ U❁) 아라비아 표기법 오른쪽에서 왼쪽으로 입력됨)
- 라틴 문자의 대문자 사용 (e.g., 영어는 평일에 대해서는 대문자를 사용하고, (⑅˘꒳˘) 스페인어는 소문자를 사용합니다). ( ͡o ω ͡o )
- 관용어의 적응 (e.g., UwU "waining c-cats and dogs"는 문자 그대로 번역하면 의미가 없습니다). rawr x3
- 레지스터의 사용 (e.g., 일본어에서 공손한 말투는 일상적인 말투와 유난히 다릅니다). rawr
- 숫자 형식 (e.g., 독일이 10 000,00 v-vs u.s에서 10,000.00). σωσ
- 날짜 형식
- 통화
- 문화적 참고자료
- 종이 크기
- 색채심리학
- 현지 법규 준수
- 현지 공휴일
- 개인 이름
