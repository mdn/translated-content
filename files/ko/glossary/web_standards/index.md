웹 표준은 여러 국제 표준 기구가 정한 규칙으로서 {{Glossary("World Wide Web", "웹")}}이 작동하는 방식을 정의합니다. (때로는 {{Glossary("Internet", "인터넷")}}을 제어하기도 합니다).

각 표준 기구는 웹의 서로 다른 측면을 정의할 책임을 지니고 있으며 모든 표준은 웹의 사용성과 접근성을 극대화하도록 조정되어야 합니다. 웹 표준은 또한 현재 상태를 개선하고 새로운 환경에 적응하도록 진화해야 합니다.

다음 대략적인 목록은 웹사이트와 네트워크 시스템이 준수해야 하는 표준에 대한 아이디어를 제공합니다.

- **IETF** (Internet Engineering Task Force): {{Glossary("URI", "URI")}}, {{Glossary("HTTP")}}, {{Glossary("MIME")}}의 설정과 사용 등에 관한 인터넷 표준(STD)
- **{{Glossary("W3C")}}**: 마크업 언어의 명세(예, {{Glossary("HTML")}}), 스타일 정의(예, {{Glossary("CSS")}}), {{Glossary("DOM")}}, {{Glossary("Accessibility", "접근성")}}
- **IANA** (Internet Assigned Numbers Authority): 이름과 숫자 레지스트리
- **Ecma Intl.:** {{Glossary("JavaScript")}}로 잘 알려진 스크립팅 표준
- **{{Glossary("ISO")}}** (International Organization for Standardization): 문자 인코딩, 웹사이트 관리, 사용자 인터페이스 디자인 등 다양한 면을 다루는 표준
