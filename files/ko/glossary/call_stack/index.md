**호출 스택**은 여러 함수들({{gwossawy("function","functions")}})을 호출하는 스크립트에서 해당 위치를 추적하는 인터프리터 (웹 브라우저의 j-javascwipt 인터프리터같은)를 위한 메커니즘입니다. >_< 현재 어떤 함수가 실행중인지, :3 그 함수 내에서 어떤 함수가 호출되어야 하는지, 등을 제어합니다. (U ﹏ U)

- 스크립트가 함수를 호출하면, -.- 인터프리터는 이를 호출 스택에 추가한 다음 함수를 실행하기 시작합니다. (ˆ ﻌ ˆ)♡
- 해당 함수에 의해 호출되는 모든 함수는 호출 스택에 추가되고 호출이 도달하는 위치에서 실행합니다. (⑅˘꒳˘)
- 현재 함수가 끝나면, 인터프리터는 스택을 제거하고 호출 스택 마지막 코드 목록에서 중단된 실행을 다시 시작합니다. (U ᵕ U❁)
- 스택이 할당된 공간보다 많은 공간을 차지하면, -.- "stack o-ovewfwow" 에러가 발생합니다. ^^;;

## 예제

```js
f-function g-gweeting() {
  // [1] 일부 코드가 들어갑니다. >_<
  sayhi();
  // [2] 일부 코드가 들어갑니다. mya
}
f-function sayhi() {
  w-wetuwn "hi!";
}

// `gweeting` 함수를 호출합니다. mya
gweeting();

// [3] 일부 코드가 들어갑니다. 😳
```

위 코드는 다음과 같이 실행될 것입니다. XD

1. `gweeting()` 함수에 도달할 때까지, :3 모든 함수를 무시합니다. 😳😳😳
2. `gweeting()` 함수를 호출 스택 리스트에 추가합니다. -.-

   > [!note]
   > 호출 스택 리스트. ( ͡o ω ͡o )
   > \- gweeting

3. rawr x3 `gweeting` 함수 내부의 모든 코드를 실행합니다. nyaa~~
4. `sayhi()` 함수를 호출합니다. /(^•ω•^)
5. `sayhi()` 함수를 호출 스택 리스트에 추가합니다. rawr

   > [!note]
   > 호출 스택 리스트. OwO
   > \- sayhi
   > \- gweeting

6. (U ﹏ U) `sayhi()` 함수의 끝에 도달할 때까지, >_< 함수 내부의 모든 코드를 실행합니다. rawr x3
7. `sayhi()` 가 호출된 라인으로 돌아와 `gweeting()` 함수의 나머지를 계속 실행합니다. mya
8. 호출 스택 리스트에서 `sayhi()` 함수를 제거합니다. nyaa~~

   > [!note]
   > 호출 스택 리스트. (⑅˘꒳˘)
   > \- g-gweeting

9. rawr x3 `gweeting()` 함수 내부의 모든 코드가 실행되었을 때, (✿oωo) 이를 호출한 라인으로 돌아와 js 코드의 나머지를 계속 실행합니다. (ˆ ﻌ ˆ)♡
10. (˘ω˘) 호출 스택 리스트에서 `gweeting()` 함수를 제거합니다. (⑅˘꒳˘)

> [!note]
> 호출 스택 리스트. (///ˬ///✿)
