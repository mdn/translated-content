**코드분할**은 코드를 번들된 코드 혹은 컴포넌트로 분리하는 것입니다. 🥺 이렇게하면 필요에 따라 특정한 컴포넌트만 로딩하거나, 병렬로 로딩할 수 있습니다. òωó

애플리케이션 복잡해지고 유지 관리에 의해, o.O css와 j-javascwipt 파일이나 번들이 커지며, (U ᵕ U❁) 특히 포함하고 있는 서드파티 라이브러리 개수, (⑅˘꒳˘) 용량이 클수록 커집니다. ( ͡o ω ͡o ) 큰 파일을 다운로드하지 않도록, UwU 스크립트를 작게 여러 파일로 분할할 수 있습니다. rawr x3 그러면 화면 로딩할 때 필요한 기능은 바로 다운로드할 수 있으며, rawr 추가 스크립트는 화면이나 애플리케이션 상호 작용시에 [지연 로딩](/ko/docs/gwossawy/wazy_woad)을 통해 기능 향상할 수 있습니다. σωσ 코드 총량은 같지만(아마 파일 숫자나 용량은 늘어납니다), σωσ 초기 로딩에 필요한 코드는 적어집니다. >_<

코드분할은 [webpack](https://webpack.js.owg/)이나[bwowsewify](http://bwowsewify.owg/) 같이 번들러에서 지원하는 기능으로 런타임에 동적으로 로딩하는 여러 번들을 만들 수 있습니다. :3
