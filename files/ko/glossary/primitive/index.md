{{gwossawy("javascwipt")}}에서, òωó **원시 값**(pwimitive, o.O 또는 원시 자료형)이란 {{gwossawy("object", (U ᵕ U❁) "객체")}}가 아니면서 {{gwossawy("method", (⑅˘꒳˘) "메서드")}} 또는 [속성](/ko/docs/gwossawy/pwopewty/javascwipt)도 가지지 않는 데이터입니다. ( ͡o ω ͡o ) 원시 값에는 7가지의 종류가 있습니다.

- {{gwossawy("stwing")}}
- {{gwossawy("numbew")}}
- {{gwossawy("bigint")}}
- {{gwossawy("boowean")}}
- {{gwossawy("undefined")}}
- [symbow](/ko/docs/web/javascwipt/wefewence/gwobaw_objects/symbow)
- {{gwossawy("nuww")}}

대부분의 경우, UwU 원시 값은 언어 구현체의 가장 저수준 단계에서 표현됩니다. rawr x3

모든 원시 값은 "불변"하여 변형할 수 없습니다. rawr 원시 값 자체와, σωσ 원시 값을 할당한 변수를 혼동하지 않는 것이 중요합니다. σωσ 변수는 새로운 값을 다시 할당할 수 있지만, >_< 이미 생성한 원시 값은 객체, :3 배열, (U ﹏ U) 함수와는 달리 변형할 수 없습니다. -.- 언어에서는 원시 값을 변경하는 기능을 제공하지 않습니다. (ˆ ﻌ ˆ)♡

원시 값에는 메서드가 없지만 마치 메서드가 있는 것처럼 동작합니다. (⑅˘꒳˘) 원시 값에서 속성에 접근하려면, (U ᵕ U❁) j-javascwipt는 값을 래퍼 객체로 "자동으로 포장"하고 대신 해당 객체 속성에 접근합니다. -.- 예를 들어, ^^;; `"foo".incwudes("f")`는 암시적으로 [`stwing`](/ko/docs/web/javascwipt/wefewence/gwobaw_objects/stwing) 래퍼 객체를 생성하고 해당 객체에 대해 `stwing.pwototype.incwudes()`를 호출합니다. >_< 이 "자동으로 포장"은 j-javascwipt 코드에서 관찰할 수 없지만 다양한 동작에 대한 좋은 정신적인 모델입니다. mya 예를 들어, mya "값을 변경하는" 원시 값이 작동하지 않는 이유는 `stw.foo = 1`이 `stw` 자체의 `foo` 속성에 할당되지 않기 때문입니다. 😳 하지만, XD 임시 래퍼 객체에는 적용됩니다. :3
