MDN 웹 문서는 웹 개발자들이 브라우저에 구애받지 않는 코드를 작성할 수 있도록 하는 브라우저 중립적인 문서를 포함합니다. 이 문서에서는 특정 주제나 콘텐츠의 유형이 MDN 웹 문서에 포함되어야 하는지에 대한 정보를 제공합니다.

## 편집 정책

이 섹션에서는 MDN 웹 문서의 콘텐츠를 관리하기 위해 Mozilla MDN 직원이 설정한 정책을 설명합니다. MDN 웹 문서의 모든 기여자는 이러한 정책을 준수해야 합니다.

### 연관성

MDN 웹 문서의 모든 콘텐츠는 해당 콘텐츠가 속한 기술 섹션과 관련이 있어야 합니다. 스팸 (상업 광고) 과 기타 관련 없는 콘텐츠는 사이트에 절대 허용되지 않습니다. 계속해서 스팸을 제출하려는 기여자는 Mozilla MDN 직원의 재량에 따라 MDN에서 차단될 수 있습니다.

연결된 주제와 관련이 있는 상업 사이트로의 외부 링크는 사례별로 판단됩니다. 웹 개발자를 돕는 데 있어서의 가치가 연결된 사이트의 상업적 이익보다 커야 합니다.

> [!NOTE]
> [웹 개발 학습하기](/ko/docs/Learn_web_development) 섹션에서 상업 사이트의 링크를 확인할 수 있을 것입니다만, 이는 제한적으로만 사용되며 신뢰할 수 있는 교육 파트너에게만 제공됩니다. 이에 관한 자세한 내용은 [웹 개발 학습하기 작성 지침 > 파트너 링크와 임베드](/ko/docs/MDN/Writing_guidelines/Learning_content#partner_links_and_embeds) 에서 확인할 수 있습니다.

### 중립성

MDN 웹 문서 내의 문서들은 반드시 [중립적 관점](https://en.wikipedia.org/wiki/Wikipedia:Neutral_point_of_view)을 유지해야 하며, 편집에서는 편견 없이 브라우저의 차이점을 보고해야 합니다. 어떤 브라우저나 사용자 에이전트에 대한 경멸적인 언급은 허용하지 않습니다.

### 표준화

MDN 웹 문서에서 문서화될 웹 기술은 표준 트랙에 있어야 하며, 최소한 하나의 렌더링 엔진에서 구현되어야 합니다. 브라우저 지원의 차이는 문서의 [브라우저 호환성](/ko/docs/MDN/Writing_guidelines/Page_structures/Compatibility_tables) 섹션에 문서화됩니다.

## 콘텐츠 제안

MDN 웹 문서에 대한 콘텐츠를 제안하고 싶다면, 제출하기 전에 이 페이지를 읽어 제안하시는 내용이 적절한지 확인하시기 바랍니다.

새로운 참조 페이지나 가이드의 경우, 어떤 콘텐츠를 제안하는지와 그 이유를 설명하는 [새 이슈](https://github.com/mdn/mdn/issues/new/choose)를 생성합니다. (가능한 한 명확하게 작성해 주세요)

새로운 콘텐츠 섹션을 포함하는 더 큰 프로젝트를 제안하려면, 신청 프로세스도 설명하는 [포함 기준](/ko/docs/MDN/Writing_guidelines/Criteria_for_inclusion) 페이지를 참조하세요.

## MDN 문서에 속하는 주제

일반적으로 오픈 웹 기술이라면 MDN 웹 문서에서 문서화합니다. 여기에는 웹 개발자가 현재와 가까운 미래에 웹사이트와 애플리케이션을 만드는 데 사용할 수 있는 모든 기능이 포함됩니다.

어떤 기능이 여러 브라우저에서 구현되고 표준으로 받아들여졌거나 표준화를 향해 진행 중이라면, 우리는 확실히 여기에 문서화합니다. 기능이 여전히 매우 실험적이고 여러 브라우저에서 구현되지 않았거나 변경될 가능성이 있다면, 여전히 포함하기에 적합하지만 작성팀이 작업할 우선순위로 보지 않을 수 있습니다.

다시 말해, MDN 웹 문서에서 문서화할 웹 기술은 다음 기준을 모두 충족해야 합니다.

- 표준 트랙에 있어야 합니다.
- 신뢰할 수 있는 표준 기구에서 발행한 사양에 명시되어 있어야 합니다.
- 최소한 하나의 렌더링 엔진에서 구현되어야 합니다.
- 안정적인 브라우저 버전에서 출시되어야 합니다.

우리의 주요 초점은 다음 프런트엔드 웹 기술에 대해 작성하는 것입니다.

- [HTML](/ko/docs/Web/HTML)
- [CSS](/ko/docs/Web/CSS)
- [JavaScript](/ko/docs/Web/JavaScript)
- [Web APIs](/ko/docs/Web/API)
- [HTTP](/ko/docs/Web/HTTP)

또한 [SVG](/ko/docs/Web/SVG), [XML](/ko/docs/Web/XML), [WebAssembly](/ko/docs/WebAssembly), [Accessibility](/ko/docs/Learn_web_development/Core/Accessibility) 과 같은 더 넓은 주제도 문서화합니다. 추가로, 이러한 기술에 대한 광범위한 [학습 가이드](/ko/docs/Learn_web_development)와 [용어집](/ko/docs/Glossary) 도 제공합니다.

> [!NOTE]
> 백엔드 기술은 보통 다른 곳에 자체 문서가 있으며 MDN Web Docs는 이를 대체하려 하지 않습니다. 다만 [몇 가지 예외는 있습니다](/ko/docs/Learn_web_development/Extensions/Server-side).

MDN 웹 문서의 모든 콘텐츠는 해당 콘텐츠가 속한 기술 섹션과 관련이 있어야 합니다. 기여자들은 작성 스타일, 코드 샘플 및 기타 주제에 대해 [MDN 작성 지침](/ko/docs/MDN/Writing_guidelines) 을 따라야 합니다.

기술이 MDN 웹 문서에서 문서화될 수 있는지에 대한 기준에 대한 자세한 내용은 [포함 기준](/ko/docs/MDN/Writing_guidelines/Criteria_for_inclusion) 페이지를 참조하세요.

### 새로운 기술을 문서화하는 시기

MDN 웹 문서에서는 적절한 경우 새로운 웹 표준 기술을 지속적으로 문서화하려고 합니다.
개발자가 새로운 기능을 필요로 할 때 즉시 배울 수 있도록 문서를 충분히 일찍 게시하는 것과 기술이 성숙하고 안정적이어서 문서가 지속적인 업데이트나 빠른 제거가 필요하지 않도록 충분히 늦게 게시하는 것 사이의 균형을 맞추려고 합니다.

일반적으로, 새로운 기술 문서화를 고려하는 가장 이른 시점에 대한 우리의 정의는, 기능이 표준 트랙에 있고 어딘가에 구현되어 있을 때 입니다.

다음과 같은 경우 새로운 기술 문서화를 고려합니다.

- 신뢰할 수 있는 표준 조직(W3C, WHATWG, Khronos, IETF 와 같은)에서 발행한 사양 문서에 명시되어 있고 합리적인 수준의 안정성에 도달한 경우(예: W3C 작업 초안이나 후보 권고안 또는 제기된 이슈의 흐름으로 판단할 때 사양이 상당히 안정적으로 보이는 경우), 그리고
- 최소한 하나의 브라우저에서 일관되게 구현되었고, 다른 브라우저 개발자가 관심을 보이는 경우(활성 티켓이나 "구현 의도" 프로세스가 진행 중인 경우)

다음과 같은 경우 새로운 기술을 문서화하지 않습니다.

- 사양이 없거나 사양이 변경될 가능성이 있는 대략적인 노트인 경우
- 하나 이하의 브라우저가 현재 구현했고 지원하지 않는 브라우저가 구현에 관심을 보이지 않는 경우. 해당 브라우저에서 작업하는 엔지니어에게 문의하고 브라우저 버그 추적기와 메일링 리스트 등을 살펴보면 이를 판단할 수 있습니다.
- 웹에 노출되지 않는 기술이거나 완전히 독점적인 경우
- 이미 비슷한 기능으로 대체되거나 사용 중단되는 징후를 보이는 경우

## MDN 웹 문서에 속하지 않는 주제

일반적으로 오픈 웹 표준이 아닌 것은 MDN 웹 문서에 속하지 않습니다. 스팸 (상업 광고) 과 기타 관련 없는 콘텐츠는 사이트에 절대 허용되지 않습니다. 계속해서 스팸을 제출하려는 기여자는 Mozilla MDN 직원의 재량에 따라 MDN에서 차단될 수 있습니다.

MDN Web Docs에 부적절한 주제의 예시입니다.

- 웹에 노출되지 않고 특정 브라우저에 특화된 기술
- 웹과 관련이 없는 기술
- 최종 사용자를 위한 문서. 예를 들어 Mozilla 제품의 경우, 이러한 문서는 [Mozilla 지원 사이트](https://support.mozilla.org/)에 속합니다.
- 자기 링크 또는 자기 홍보 외부 링크. 외부 링크를 추가하기 전에 [작성 스타일 지침](/ko/docs/MDN/Writing_guidelines/Writing_style_guide#external_links)을 확인하세요.

### 문서를 제거하는 시기

페이지가 더 이상 어떤 방식으로도 유용한 정보를 포함하지 않거나, 충분히 오래되었거나, 유지하면 오해의 소지가 있을 정도로 부정확할 수 있는 경우 MDN 웹 문서에서 삭제됩니다.

다음 예시는 페이지/콘텐츠가 삭제될 수 있는 상황에 대한 예시입니다.

- 문서에 모든 브라우저에서 구현되지 않고 나중에 철회된 기능에 대한 정보가 포함된 경우(보통 접두사 기능과 같은 실험적 기능)
- 참조 페이지가 브라우저에서 구현되기 전에 사양에서 제거된 기능을 설명하는 경우
- 문서가 나중에 나쁜 관행으로 밝혀지고 더 나은 기술로 대체된 기법을 다루는 경우
- 문서에 나중에 더 나은 품질의 다른 문서로 대체된 정보가 포함된 경우
- 문서에 MDN 웹 문서에 부적절한 콘텐츠가 포함된 경우
- MDN 웹 문서의 섹션이 오픈 웹 기술에 초점을 맞추지 않고 유지 관리 부담이 되는 경우

문서를 삭제하는 방법에 대한 자세한 정보는 [페이지 생성, 이동 및 삭제](/ko/docs/MDN/Writing_guidelines/Howto/Creating_moving_deleting) 가이드를 참조하세요.

## MDN 웹 문서에서 허용되는 문서 유형

일반적으로 우리의 문서는 다음 범주에 속합니다.

- 참조
- 가이드
- 용어집
- 학습/튜토리얼

일반적으로 MDN 웹 문서는 프로젝트 또는 프로세스 문서가 아닌 제품 문서를 위한 것입니다. 따라서 문서가 "어떤 것을 사용하는 방법" 또는 "어떤 것이 작동하는 방법"에 관한 것이라면(여기서 "것"은 위에서 언급한 주제 범주 중 하나에 속함) MDN 웹 문서에 포함될 수 있습니다.

문서가 "누가 어떤 것을 개발하고 있는지" 또는 "어떤 것을 개발하기 위한 계획"에 관한 것이라면 MDN 웹 문서에 포함되어서는 안 됩니다.

다음은 MDN 웹 문서에 배치되어서는 안 되는 문서 유형의 예시입니다.

- 계획 문서
- 설계 문서
- 프로젝트 제안서
- 사양 또는 표준
- 홍보 자료, 광고 또는 개인 정보
