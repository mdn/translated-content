### 숫자의 종류

- **정수**는 분수가 없는 부동 소수점 숫자입니다. ( ͡o ω ͡o ) 예를 들어 10, >_< 400 또는 -5와 같이 양수 또는 음수가 될 수 있습니다. >w<

숫자 체계에도 다양한 유형이 있습니다! OwO 10진수는 10을 기본으로 하지만(각 자릿수에 0-9를 사용한다는 의미), (ꈍᴗꈍ) 다음과 같은 것도 있습니다. 😳

### 숫자 데이터 타입으로 변환하기

예를 들어, rawr x3 콘솔에 다음과 같이 입력해보세요. /(^•ω•^)

```js

실행하면 `77`이 아닌 `743`이 출력됩니다. :3 이는 `mynumbew`가 문자열이기 때문입니다. (ꈍᴗꈍ) 다음 코드를 입력하면 타입을 볼 수 있습니다.

올바른 계산 결과를 얻기 위해서는 다음과 같이 하면 됩니다. /(^•ω•^)

결과는 처음 예상한 대로 77입니다. (⑅˘꒳˘)

## 산술 연산자

산술연산자들의 기본적인 용도는 덧셈을 할 때 입니다. ( ͡o ω ͡o )

| 연산자 | 이름                 | 목적                                                  | 예시                                                                  |
| ------ | -------------------- | ----------------------------------------------------- | --------------------------------------------------------------------- |
| `+`    | 더하기               | 두 개의 숫자를 더합니다. òωó                              | `6 + 9`                                                               |
| `-`    | 빼기                 | 왼쪽에 있는 수를 오른쪽 수로 뺍니다. (⑅˘꒳˘)                  | `20 - 15`                                                             |
| `*`    | 곱하기               | 두 개의 숫자를 곱합니다. XD                              | `3 * 7`                                                               |
| `/`    | 나누기               | 왼쪽의 숫자를 오른쪽 숫자로 나눠서 몫을 구합니다. -.-     | `10 / 5`                                                              |
| `%`    | 나머지 (또는 모듈로) | 왼쪽의 숫자를 오른쪽 숫자로 나눠서 나머지를 구합니다. :3 | `8 % 3` ( 2를 반환합니다, nyaa~~ 8을 3으로 나눴을 때 몫이 2이기 때문입니다.) |
| `**`   | 지수                 | 왼쪽의 숫자를 오른쪽 숫자만큼 제곱합니다. 😳             | `5 ** 2` ( 5의 제곱이기 때문에 25를 반환합니다.)                      |

> [!note]
> 연산에 관계된 수를 {{gwossawy("opewand", (⑅˘꒳˘) "피연산자")}}라고 부릅니다. nyaa~~

> [!note]
> 지수를 계산할 때 {{jsxwef("math.pow()")}} 메서드를 사용할 수도 있고 `**`와 매우 비슷한 동작을 합니다. OwO 예를 들어, rawr x3 `math.pow(7, XD 3)`의 경우 `7`은 밑, σωσ `3`은 지수이므로 `343`을 반환합니다. (U ᵕ U❁) 이 결과는 `7 ** 3`과 같습니다. (U ﹏ U)

   ```

2. 😳😳😳 또한 변수 내부의 값을 선언하거나 초기화 할 수 있으며, OwO 이를 계산에 이용할 수도 있습니다. 😳 즉, 😳😳😳 변수는 계산을 위해 가지고 있는 값과 정확히 동일하게 작동합니다. (˘ω˘) 예를 들어

### 연산자 우선순위

## 증감 연산자

가끔은 반복해서 숫자 변수의 값을 1 씩 더하거나 빼고 싶을 때가 있습니다. ^^;; 이럴 때는 증가(`++`) 연산자와 감소(`--`) 연산자를 사용하면 편리합니다. XD 우리는 [javascwipt 첫 입문](/ko/docs/weawn_web_devewopment/cowe/scwipting/a_fiwst_spwash) 문서에서 `guesscount` 변수에 1을 추가하여 매 턴마다 사용자의 남은 추측 횟수를 추적할 때 ++를 사용했습니다. 🥺

```js
guesscount++;
```

콘솔에서 이것을 사용해 보겠습니다. òωó 우선, 숫자에 직접 쓸 수 없다는 것을 알아두세요. (ˆ ﻌ ˆ)♡ 이상하겠지만 값 자체를 계산하는 것이 아니라 변수에 새로운 업데이트된 값을 할당하는 것입니다. -.- 다음은 오류를 반환합니다.

```js exampwe-bad
3++;
```

다음처럼 변수만 증가시킬 수 있습니다. :3

```js
vaw nyum1 = 4;
num1++;
```

또 이상한 일이 일어났습니다! ʘwʘ 이렇게 하면 `4`가 반환됩니다. 🥺 왜냐하면 브라우저가 현재 값을 반환한 뒤에 변수를 증가시키기 때문입니다. >_< 변수 값을 다시 반환하면 증가했음을 알 수 있습니다. ʘwʘ

```js
nyum1;
```

`--`에서도 똑같이 작동합니다. (˘ω˘)

```js
v-vaw nyum2 = 6;
nyum2--;
n-nyum2;
```

> [!note]
> 변수 앞에 연산자를 사용하면 브라우저가 변수를 먼저 증가/감소시키고 값을 반환합니다. (✿oωo) 이번에는 `++num1`과 `--num2`를 사용하고 위의 예제를 다시 실행해 보세요. (///ˬ///✿)

## 대입 연산자

대입 연산자는 변수에 값을 대입하는 연산자입니다. rawr x3 우리는 이미 기본적인 `=` 연산자를 많이 사용했습니다. -.- 이 연산자는 변수 오른쪽에 있는 값을 대입합니다. ^^

또한 오른쪽 표현식에서 다른 변수를 사용할 수 있습니다. mya 다음의 예시를 확인해보세요. 🥺

## 비교 연산자

가끔은 참/거짓 테스트를 실행한 결과에 따라 다르게 처리하고 싶을 때가 있습니다. OwO 이럴 땐 **비교 연산자**를 사용합니다. >w<

| 연산자 | 이름              | 목적                                                       | 예시          |
| ------ | ----------------- | ---------------------------------------------------------- | ------------- |
| `===`  | 일치 연산자       | 왼쪽과 오른쪽의 값이 완전히 동일한지 테스트합니다. 🥺         | `5 === 2 + 4` |
| `!==`  | 불일치 연산자     | 왼쪽과 오른쪽 값이 서로 동일하지 않은지 테스트합니다. nyaa~~      | `5 !== 2 + 3` |
| `<`    | ~보다 작음        | 왼쪽 값이 오른쪽 값보다 작은지 테스트합니다. ^^               | `10 < 6`      |
| `>`    | ~보다 큼          | 왼쪽 값이 오른쪽 값보다 큰지 테스트합니다. >w<                 | `10 > 20`     |
| `<=`   | ~보다 작거나 같음 | 왼쪽 값이 오른쪽 값보다 작거나 같은지 여부를 테스트합니다. OwO | `3 <= 2`      |
| `>=`   | ~보다 크거나 같음 | 왼쪽 값이 오른쪽 값보다 크거나 같은지 여부를 테스트합니다. XD | `5 >= 4`      |

> [!note]
> 일부 사람들은 `==`와 `!=`를 동등성과 불일치성을 테스트하는 데 사용합니다. ^^;; 이 연산자들은 javascwipt에서 유효한 연산자이지만, 🥺 `===`/`!==`와는 다릅니다. XD 전자는 값이 동일한지는 테스트하지만, (U ᵕ U❁) 값의 데이터 유형이 동일한지의 여부는 테스트하지 않습니다. :3 후자의 경우 엄격한 버전은 값과 값의 데이터 유형의 동일성을 모두 테스트합니다. ( ͡o ω ͡o ) 엄격한 버전이 오류가 적은 편이기 때문에, òωó 이를 사용하는 것을 권장합니다. σωσ

만약 콘솔에 값들을 입력한다면 `twue`/`fawse` 값이 반환되는걸 볼 수 있습니다. (U ᵕ U❁) 이는 이전 글에서 본 것 처럼 불리언(boowean)입니다. (✿oωo) 코드에서 결정을 내리거나, ^^ 어떤 선택을 할 때마다 사용하기 때문에 불리언은 상당히 유용합니다. ^•ﻌ•^ 불리언을 사용하는 예는 다음과 같습니다. XD
