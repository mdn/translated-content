Le rôle abstrait **`sectionhead`** est une superclasse pour les étiquettes ou résumés du sujet de sa section associée.

> [!WARNING]
> Le rôle `sectionhead` est un [rôle abstrait](/fr/docs/Web/Accessibility/ARIA/Reference/Roles#6._rôles_abstraits). Il est inclus ici pour compléter la documentation. Il ne doit pas être utilisé par les développeur·euse·s web.

## Description

Le rôle structurel `sectionhead` est un rôle abstrait pour les sous-rôles qui identifient les étiquettes ou résumés des sections qu'ils étiquettent. Le rôle ne doit pas être utilisé. Les quatre sous-rôles — [`columnheader`](/fr/docs/Web/Accessibility/ARIA/Reference/Roles/columnheader_role), [`heading`](/fr/docs/Web/Accessibility/ARIA/Reference/Roles/heading_role), [`rowheader`](/fr/docs/Web/Accessibility/ARIA/Reference/Roles/rowheader_role) et [`tab`](/fr/docs/Web/Accessibility/ARIA/Reference/Roles/tab_role). Il existe des équivalents d'éléments HTML, comme, {{HTMLElement('th', '<code>&lt;th scope="col"&gt;</code>')}} pour `columnheader`, {{HTMLElement('th', '<code>&lt;th scope="row"&gt;</code>')}} pour `rowheader`, et n'importe lequel des éléments de titre HTML, {{HTMLElement("Heading_Elements", "h1")}} à {{HTMLElement("Heading_Elements", "h6")}} pour `heading`. Le rôle `tab` n'a actuellement pas d'équivalent HTML.
