En ce qui concerne les données ou un objet **{{Glossary("Wrapper", "enveloppant")}}** cette donnée, la valeur est la **{{Glossary("Primitive", "valeur primitive")}}** que cet objet wrapper contient. Pour les **{{Glossary("Variable", "variables")}}** ou les **{{Glossary("Property", "propriétés")}}**, la valeur peut être soit une primitive, soit une **{{Glossary("Object reference", "référence d'objet")}}**.

Dans le contexte des valeurs des propriétés CSS, il existe des valeurs spécifiées, calculées et réelles. La valeur finale de chaque propriété CSS appliquée à chaque élément et pseudo-élément résulte d'un calcul en quatre étapes&nbsp;: la valeur est déterminée par la spécification (la «&nbsp;[valeur définie](/fr/docs/Web/CSS/Guides/Cascade/Property_value_processing#valeur_définie)&nbsp;»), puis résolue en une valeur utilisée pour l'héritage (la «&nbsp;[valeur calculée](/fr/docs/Web/CSS/Guides/Cascade/Property_value_processing#valeur_calculée)&nbsp;»), puis convertie en une valeur absolue si nécessaire (la «&nbsp;[valeur utilisée](/fr/docs/Web/CSS/Guides/Cascade/Property_value_processing#valeur_utilisée)&nbsp;»), et enfin transformée selon les contraintes de l'environnement local (la «&nbsp;[valeur réelle](/fr/docs/Web/CSS/Guides/Cascade/Property_value_processing#valeur_réelle)&nbsp;»).

Pour les types de données CSS, voir [Valeurs et unités CSS](/fr/docs/Web/CSS/Guides/Values_and_units).
