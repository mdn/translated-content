Un **générateur de site statique** (SSG) est un logiciel utilisé pour générer des sites _statiques_. Un site statique est composé de fichiers {{Glossary("HTML")}}, {{Glossary("CSS")}} et {{Glossary("JavaScript")}}. Surtout, les sites statiques n'ont pas de [logique côté serveur](/fr/docs/Learn_web_development/Extensions/Server-side), donc pour une URL donnée, tous les utilisateur·ice·s recevront le même contenu. Les auteur·ice·s rédigent le contenu dans n'importe quel format accepté par le générateur, comme Markdown, reStructuredText, HTML (et parfois même [React](/fr/docs/Learn_web_development/Core/Frameworks_libraries/React_getting_started), etc.), et le générateur les compile en un ensemble de fichiers statiques optimisés pouvant être affichés par le navigateur.

Les sites statiques sont couramment utilisés pour les blogs, la documentation et d'autres sites axés sur le contenu, qui n'ont pas de données à récupérer ou à générer côté serveur. Ils sont rapides, sécurisés et plus faciles à déployer, car ils peuvent être servis depuis un {{Glossary("CDN")}}.
